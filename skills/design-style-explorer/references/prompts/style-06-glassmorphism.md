# STYLE 06 — 글래스모피즘 (매력 축: 투명감)

매력 가설: 만지고 싶은 유리. 매력은 유리 자체가 아니라 "유리 너머로 비치는 것"에서 온다.
고도화 = 배경의 삶 + 유리의 광학. 마스터 프롬프트 뒤에 붙여 사용한다.

```
[스타일 지시 — 글래스모피즘 v2]

■ 무드
iOS 제어센터. 컬러풀한 배경 위에 떠 있는 반투명 유리 레이어.

■ 팔레트
배경: #4F46E5 → #9333EA → #EC4899 대각 그라디언트
유리: rgba(255,255,255,.14) / 유리 보더: rgba(255,255,255,.3)
텍스트: #FFFFFF / 보조: rgba(255,255,255,.65)

■ 타이포
Pretendard 500~700. 유리 위 가독성이 우선 — 얇은 굵기 금지.

■ 형태·질감 — 유리의 광학
backdrop-filter: blur(20px)의 유리 카드, radius 20px,
카드 상단 1px 흰 하이라이트(빛 반사), 유리는 최대 2겹까지만.

■ 히어로 연출
1) 배경 그라디언트가 매우 느리게(20초 이상 주기) 색상 위치를 이동한다
   — 유리 너머의 세계가 살아있다는 감각. 어지럽지 않게, 알아채기 직전의 속도로
2) 카드들은 400ms 일괄 페이드인 (스태거 불필요 — 주인공은 배경과 유리)

■ 마이크로 인터랙션
1) 카드 hover: 유리가 밝아진다 (배경 투명도 .14→.22) + 보더 하이라이트 강화
2) 차트 막대: 유리 재질로 — hover 시 상단에 흰 캡 점등
3) 상태 태그: 유리 알약 (반투명 + 상태색 텍스트)

■ 시그니처
흐르는 배경 + 유리 재질 차트 — 막대 너머로 그라디언트가 비쳐 보인다.

■ 접근성·안전장치
- prefers-reduced-motion: 배경 이동 정지, 페이드인 제거
- 흰 텍스트 대비 4.5:1 확보 — 부족하면 해당 유리의 배경 투명도를 올린다
- backdrop-filter 미지원 브라우저 대비 유리에 반투명 배경색 폴백

■ 금지
불투명 흰 카드, 검정 텍스트, 직각, 격자 보더, 유리 3겹 이상 중첩,
빠른 배경 애니메이션, 유리 위의 강한 그림자

■ 자체 크리틱 루프
1) 유리 너머가 실제로 비쳐 보이는가? (blur가 과하면 그냥 회색 카드다)
2) 흰 텍스트가 배경 밝은 지점에서도 읽히는가?
3) 유리가 3겹 이상 겹친 곳이 있는가? → 제거
4) 배경 이동이 인지되는가? → 인지되면 더 느리게
```
