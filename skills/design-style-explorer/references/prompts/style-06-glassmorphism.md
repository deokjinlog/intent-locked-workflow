# STYLE 06 — 글래스모피즘 (매력 축: 투명감)

매력 가설: 만지고 싶은 유리. 매력은 유리 자체가 아니라 "유리 너머로 비치는 것"에서 온다.
고도화 = 배경의 삶 + 유리의 광학. 핵심 단 하나 — 카드는 불투명한 물체가 아니라 뒤가 비치는 창(窓)이다.
뒤에서 흐르는 컬러 그라디언트가 backdrop-blur 를 통해 우윳빛으로 번져 올라오는 그 순간이 이 스타일의 전부다.
마스터 프롬프트 뒤에 붙여 사용한다.

```
[스타일 지시 — 글래스모피즘 v2]

■ 무드
iOS 제어센터 / macOS Big Sur 사이드바 / visionOS 유리 패널.
컬러풀하게 살아 움직이는 배경 위에 떠 있는 반투명 유리 레이어들.
서리 낀 창에 손을 대면 뒤의 빛이 번지는 감각 — 재질이 아니라 광학이 주인공.

■ 팔레트 — "뒤에서 흐르는 빛"
배경(주인공): #4F46E5 → #7C3AED → #9333EA → #EC4899 대각(135deg) 4-stop 그라디언트,
  추가로 화면 모서리마다 radial-gradient 컬러 blob 2~3개(#22D3EE, #F472B6, #A855F7)를
  낮은 투명도로 겹쳐 "오로라"처럼 뭉게진 광원을 만든다 — 이 blob 들이 유리 뒤로 비쳐야 한다.
유리 필: rgba(255,255,255,.10~.16) — 절대 .5 이상 불투명으로 올리지 않는다(뒤가 안 비치면 실패).
유리 보더: rgba(255,255,255,.28~.34) 1px — 유리 가장자리의 빛 굴절선.
텍스트: #FFFFFF / 보조: rgba(255,255,255,.62) / 강조 수치: #F0ABFC(연보라핑크, 유리 위에서 빛남).
그림자는 색을 쓰지 않는다 — 유리는 표면이 아니라 창이므로 무채색 확산광만.

■ 타이포
Pretendard 500~700. 유리 위 가독성이 최우선 — 200/300 얇은 굵기 절대 금지.
숫자·라벨은 흰색, 배경 밝은 지점과 겹칠 땐 text-shadow: 0 1px 2px rgba(0,0,0,.25) 로 띄운다.

■ 형태·질감 — 유리의 광학 (이 스타일의 심장)
- backdrop-filter: blur(20~28px) saturate(140%) — blur 없는 카드는 이 스타일이 아니다.
- radius 20~24px, 유리 카드 상단 안쪽 1px 흰 하이라이트(inset 0 1px 0 rgba(255,255,255,.5)) = 빛 반사선.
- 유리 겹침은 최대 2겹까지 — 겹칠수록 뒤가 안 보이니 겹을수록 얇게(투명도 더 낮게).
- 카드끼리 미세하게 겹쳐 배치해 "유리판이 포개진" 깊이를 만든다 — 단, 3겹부턴 회색 죽으로 붕괴.
- 유리 표면은 완벽히 평평하다 — 볼록/오목/두께감 있는 입체 덩어리가 아니다. 창이지 물체가 아니다.

■ 히어로 연출
1) 배경 그라디언트 + 오로라 blob 이 매우 느리게(20~40초 주기) 색상 위치를 이동한다
   — 유리 너머의 세계가 살아있다는 감각. 어지럽지 않게, 알아채기 직전의 속도로.
   blob 은 background-position / transform: translate 로 서로 다른 속도로 흐르게 해 시차(parallax)를 준다.
2) 카드들은 400ms 일괄 페이드인 + 아주 살짝 blur-in(backdrop-filter 가 0→20px 로) — 주인공은 배경과 유리.

■ 마이크로 인터랙션
1) 카드 hover: 유리가 밝아진다(필 .12→.20) + 보더 하이라이트 강화 + backdrop blur 살짝 증가(20→24px).
   이동/그림자 팽창이 아니라 "빛이 더 든다"로 반응 — 물체가 아니라 창이니까.
2) 차트 막대: 유리 재질(반투명 + backdrop-blur)로 — 막대 뒤로 배경 그라디언트가 비친다.
   hover 시 상단에 흰 캡(1px 하이라이트) 점등.
3) 상태 태그: 유리 알약 — 반투명 필 + 상태색은 텍스트/보더로만(면을 불투명 색으로 칠하지 않는다).

■ 시그니처
흐르는 오로라 배경 + 유리 재질 차트 — 막대 너머로 그라디언트가 그대로 비쳐 보인다.
같은 화면을 스샷 찍으면, 반투명 유리를 통해 뒤의 컬러가 우윳빛으로 번진 것만으로 "아 유리다" 하고 즉시 안다.

■ 접근성·안전장치
- prefers-reduced-motion: 배경/blob 이동 정지, 페이드인·blur-in 제거(정지 상태 그라디언트는 유지).
- 흰 텍스트 대비 4.5:1 확보 — 부족하면 그 유리 뒤에 어두운 스크림(rgba(0,0,0,.18)) 한 겹만 깔고 필 투명도는 유지.
  (필 자체를 불투명하게 올려 대비를 확보하는 건 금지 — 투명감이 죽는다. 스크림으로 푼다.)
- backdrop-filter 미지원 브라우저 대비 유리에 rgba(255,255,255,.22) 반투명 배경색 폴백(여전히 반투명).

■ 금지 (클러스터메이트 12 뉴모·15 클레이와의 방어선)
- 불투명 카드 절대 금지 — rgba alpha .5 이상, opaque #FFFFFF/#F5F5F5 면, 뒤가 안 비치는 카드는 전부 실패.
- backdrop-filter/blur 없는 카드 금지 — blur 가 없으면 그냥 반투명 색면일 뿐 유리가 아니다.
- emboss/deboss·볼록·오목·요철·이중 그림자(밝은쪽+어두운쪽) 금지 — 이건 뉴모피즘(12)이다. 유리는 표면 요철이 없다.
- 두께감 있는 통통·물렁한 클레이 덩어리·큰 radius(32px+)의 젤리 블롭 금지 — 이건 클레이(15)다. 유리는 얇고 평평하다.
- flat 파스텔 색면(불투명 민트/베이비블루/살구 단색 채움) 금지 — 뒤가 안 비치면 파스텔 카드지 유리가 아니다.
- 유리 필을 상태색으로 불투명하게 채우기 금지 — 색은 텍스트/보더로만, 면은 항상 투명.
- 검정 텍스트, 직각, 격자 보더, 유리 3겹 이상 중첩, 빠른 배경 애니메이션, 유리 위의 짙은/컬러 드롭섀도우 금지.

■ 자체 크리틱 루프
1) 유리 너머로 배경 그라디언트가 실제로 비쳐 보이는가? (blur가 과하거나 필이 짙으면 그냥 회색 카드다)
2) 카드를 뉴모(12)처럼 요철·이중그림자로 만들지 않았나? → 표면은 평평한 창이어야 한다, 제거.
3) 카드를 클레이(15)처럼 통통·불투명하게 만들지 않았나? → 얇고 반투명하게, 뒤를 비춰라.
4) 흰 텍스트가 배경 밝은 지점에서도 읽히는가? → 스크림 한 겹으로, 필 불투명화로는 말고.
5) 유리가 3겹 이상 겹친 곳이 있는가? → 제거(겹칠수록 뒤가 죽는다).
6) 배경/blob 이동이 인지되는가? → 인지되면 더 느리게, 안 보이면 blob 대비를 살짝만 올려라.
```
