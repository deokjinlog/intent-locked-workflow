# STYLE 26 — 다크 프로 미니멀 (매력 축: 절제된 프로)

매력 가설: 개발자 도구를 매일 여는 사람의 눈에 편안한 화면. 매력은 화려함이 아니라 절제 — 어두운 배경에 헤어라인 한 줄, 저채도 액센트 하나만 남기고 나머지를 다 덜어낸 결과의 신뢰감.
고도화 = 보더 대비를 1~2 단계로만 조율하는 감각과 여백을 아끼지 않는 배짱. 마스터 프롬프트 뒤에 붙여 사용한다.

```
[스타일 지시 — 다크 프로 미니멀 v2]

■ 무드
Linear·Vercel 계열의 절제된 다크. 개발자 대시보드를 오래 봐도 피로하지 않은 저자극 화면.
어두운 표면 위에 얇은 헤어라인 보더로 영역을 나누고, 저채도 액센트 하나만 신호로 쓴다.
차분하고 프로페셔널하며, 말수가 적다. 장식을 빼서 콘텐츠가 유일한 주인공이 되게 한다.

■ 팔레트 — 저채도 다크 (배경 톤 + 헤어라인 + 액센트 1)
배경 base #0E0E11 / surface #131317 / surface-raised #17171C / surface-hover #1C1C22
헤어라인 border #232326 / border-strong #2E2E33 (구분이 꼭 필요한 곳만 한 단계 위)
텍스트 primary #EDEDEF / secondary #A0A0A8 / tertiary #6E6E76 (3단 위계로만 운용)
액센트 primary #8B8BF5 (저채도 인디고 — 링크·선택·주요 CTA·차트 라인 1개)
상태색은 저채도로 억제 — 성공 #4E9A76 / 경고 #C7A15A / 문제 #C96A6A (원색 금지, 텍스트+아이콘 병행)
규칙: 한 화면에서 채색 면적은 5% 이하. 나머지는 배경·헤어라인·회색 텍스트로 구성.

■ 타이포 — Inter 타이트
본문 한글은 Pretendard 가정. 영문·숫자·라벨은 Inter (구글폰트) 명시 로드.
Display 30/38 620 tracking -0.02em · Title 20/28 600 -0.01em · Section 15/22 600 ·
Body 14/22 400 · Small 13/18 · Label 12/16 500 uppercase tracking 0.04em(테이블 헤더·메뉴 구분).
숫자는 tabular-nums 고정폭 — KPI·테이블 수치 정렬용. 전반적으로 letter-spacing 음수, 줄간격 여유.

■ 형태
radius 작게 일관 — 카드·패널 10px, 버튼·인풋·칩 8px, 아바타·아이콘 버튼 6px, 툴팁 6px.
경계는 그림자 대신 1px 헤어라인(border #232326)이 기본. 그림자는 팝오버·드롭다운에만
0 8px 24px rgba(0,0,0,.4)로 아주 얕게. 카드 내부 여백 20~24px, 섹션 간 간격 32px 이상 — 넉넉하게.
구분선은 full-bleed 헤어라인 1px. 채운 배경 블록보다 선 하나로 나누는 것을 우선.

■ 히어로 연출 — 조용히 자리잡기 (로드 시퀀스)
0.0s 전체가 opacity 0→1로 220ms 페이드 (튀는 등장 없음, 한 번에 차분히)
0.1s 좌측 사이드바 항목이 위→아래 20ms stagger로 텍스트만 또렷해짐 (색 tertiary→primary)
0.25s 상단바 아래 KPI 4장이 y+6px→0 + 페이드, 각 180ms 순차 (그림자 없이 헤어라인만)
0.45s 차트 라인 1개가 좌→우로 path draw 600ms ease-out (면 채움 없이 선만)
0.6s 테이블 행이 상단부터 24ms 간격 페이드 (움직임 최소, 밝기만 올라옴)
연출 원칙: 모든 등장은 페이드+미세 이동뿐. scale 튐·bounce·번쩍임 없음.

■ 마이크로 인터랙션 (이 스타일의 실체)
1) 행·항목 hover: 배경이 surface→surface-hover(#1C1C22)로 120ms 부드럽게, 좌측에 2px 액센트
   바가 스윽 나타남(width 0→2px). 클릭음 없는 조용한 피드백.
2) 보더 하이라이트: 인풋·카드 focus 시 border #232326→액센트 #8B8BF5로 색만 전환(150ms),
   두께·크기 변화 없음. 발광 대신 색 이동 하나로 상태를 말한다.
3) 버튼 press: 배경 밝기 4% 하강 + 96% scale(80ms) 후 복귀 — 눌린 감만 미세하게.
4) 툴팁: 대상 위 6px에서 opacity+y-4px→0 페이드(120ms), 얕은 그림자 1겹.
5) 사이드바 접기: 폭 240→64px 슬라이드(220ms), 라벨은 페이드아웃하며 아이콘만 남음.

■ 시그니처
어두운 저채도 표면 + 영역마다 1px 헤어라인 보더 + 저채도 인디고 액센트 하나.
장식을 최대한 덜어낸 넉넉한 여백과 tabular 숫자. 이 조합이 화면을 즉시
"절제된 프로 다크 도구"로 읽히게 하는 각인 — 색이 아니라 여백과 선이 말한다.

■ 접근성·안전장치
- prefers-reduced-motion: 페이드·path draw·stagger 제거, 상태 전환만 즉시 반영
- JS 실패 폴백: hover 배경·focus 보더 색은 CSS만으로 동작, 차트 라인은 정적 SVG로 표시
- focus-visible: 액센트 #8B8BF5 2px 아웃라인 + offset 2px (헤어라인 색 전환만으로 포커스 표시 금지)
- 대비: secondary 텍스트 #A0A0A8는 배경 대비 4.5:1 확보, tertiary는 비필수 보조 정보에만
- 상태색은 색만으로 구분 금지 — 성공/경고/문제 모두 아이콘·라벨 병행

■ 금지 (정체성의 절반)
발광 네온·글로우·box-shadow로 만든 빛번짐, 다색 그라디언트 폭발·오로라 배경,
밀집 analytics 위젯(작은 카드 안 미니차트 다발·스파크라인 도배), 골드·샴페인 등 럭셔리 금속색,
HUD 프레임·코너 브래킷·스캔라인 연출, 채도 높은 원색 남발, 두꺼운 컬러 보더(1px 헤어라인 이탈),
그림자로 층 쌓기(elevation 위계 — 경계는 선으로), 밝은 배경 강제, radius 16px 초과, 세리프 서체.

■ 자체 크리틱 루프
1) 채색 면적이 화면의 5% 이하이고, 액센트가 인디고 하나로 통일됐는가? — 잉여 색 색출
2) 영역 구분이 그림자가 아니라 1px 헤어라인으로 이뤄졌는가? — box-shadow 층 쌓기 잔재 검사
3) 텍스트가 primary/secondary/tertiary 3단 위계 안에서만 움직이는가? — 임의 회색값 이탈 점검
4) 발광·글로우·미니차트 다발·HUD 프레임 등 금지 요소가 한 조각도 섞이지 않았는가?
```
