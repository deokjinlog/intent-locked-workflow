# STYLE 19 — 맥시멀리즘 (매력 축: 과잉/폭발)

매력 가설: 절제가 아니라 폭발이 시선을 붙든다. 매력은 "더는 못 넣겠다" 직전의 밀도.
고도화 = 무질서처럼 보이되 실은 통제된 충돌. 형광이 서로를 잡아먹으며 소리를 지르는 화면.
색이 하나라도 얌전해지거나 배경이 한 번이라도 조용해지면 이 스타일은 죽는다. 마스터 프롬프트 뒤에 붙여 사용한다.

```
[스타일 지시 — 맥시멀리즘 v2]

■ 무드
안티 미니멀. 형광 다색이 동시에 터지고 패턴 배경이 쉴 틈 없이 깔린 브랜드 랜딩/포스터.
Memphis·anti-design·스티커봄(sticker bomb) 계보 — 화면이 조용할 틈을 1px도 주지 않는다.
"비어 있으면 채운다"가 규칙. 단, 채움에는 리듬이 있다 — 카오스는 연출이지 방치가 아니다.
키워드는 장식적 카오스(decorative chaos): 형광의 충돌 + 패턴/텍스처 + 겹침·회전 + 스티커 남발.
색이 소리를 지르고, 도형이 서로를 밀치고, 스티커가 아무 데나 삐져나오는 축제 상태가 기본값.

■ 팔레트 — 형광 다색 폭발 (5색+ 동시 강제)
배경조차 무패턴 크림으로 두지 않는다 — 크림 #FFF8E7 바탕에 도트/지그재그/체크 패턴 텍스처를
낮은 채도 톤온톤(#FFE9B0 등)으로 항상 깔고, 그 위에 형광 색면 레이어가 폭발한다.
핵심 4형광은 항상 전부 등장: 마젠타 #FF2E93 / 라임 #C6FF00 / 오렌지 #FF6B00 / 전기파랑 #0038FF.
보조: 형광 노랑 #FFE500 / 시안 #00E5FF / 핫핑크 #FF5CC8 — 필요 시 6~7색까지 증식.
텍스트 잉크블랙 #0A0A0A 는 본문 가독 전용 앵커일 뿐, 절대 주역 색이 아니다.
5색 이상 동시 사용 강제 — 한 화면에 형광이 4색 이하로 떨어지면 회귀. 6색이면 더 좋다.
색면은 서로 인접 배치해 대비 극대 (마젠타 옆 라임, 파랑 옆 오렌지, 시안 옆 핫핑크).
회색·베이지·네이비 같은 "쉬어가는 중립색"으로 톤을 낮추지 않는다 — 채도는 항상 최고점 부근.
색은 소수의 강조가 아니라 화면 전체를 뒤덮는 주재료다 (색 인색함 = 즉시 회귀).

■ 타이포 — 폰트 믹스 + 컬러 폭발
헤드라인 Archivo Black 또는 Anton (초대형, 900) — 글자마다 형광색이 다르게 튄다 /
서브·라벨 Space Grotesk 500~700 (기하학적 대비) /
한글 본문 Pretendard 400~700.
최소 2개 디스플레이 폰트 혼용 — 크기 편차 극단 (h1 72px+ vs 라벨 11~12px).
헤드라인은 -6°~8° 회전, 글자별 색을 마젠타·라임·오렌지·파랑으로 번갈아, 일부만 아웃라인(text-stroke).
헤드라인 뒤로 형광 하이라이터 색면·별표·밑줄 낙서가 삐뚤게 겹쳐 깔린다 (글자와 장식이 한 덩어리).
숫자·라벨에도 형광 배경 스티커를 붙여 무채색으로 남는 텍스트가 없게 한다.

■ 형태 — 도형·패턴·겹침·회전 (전부 동시)
배경에 도형 산포: 원·삼각형·지그재그·물결·별·스마일·체크무늬·꽃 SVG 레이어를 다층으로.
배경은 항상 패턴/텍스처가 살아 있음 — 단색 여백 구간을 만들지 않는다.
카드는 각기 다른 회전(-5°~5°) + 굵은 검정 보더 3~4px + 하드 오프셋 형광 이중 그림자
(box-shadow: 8px 8px 0 #FF2E93, 14px 14px 0 #0A0A0A) — 블러 0, 그림자에도 형광을 섞는다.
요소끼리 의도적으로 겹침(z-index 교차) — KPI 카드 모서리가 차트를 물고,
스티커 도형(별·번개·하트·말풍선)이 헤더 글자 위로 삐져나오고, 태그가 카드 경계를 넘어 걸친다.
정렬은 항상 어긋나 있음 — 완벽 수평 그리드는 이 스타일의 반대말이다.

■ 히어로 연출 — 폭발 시퀀스 (로드 0~1.5s)
0.0s 패턴 크림 배경만 (텍스처는 이미 깔려 있음).
0.1s 배경 형광 도형들이 서로 다른 방향에서 튕겨 들어옴 (각 80~140ms 스태거, overshoot 강하게).
0.5s 초대형 다색 헤드라인이 좌측에서 밀려들며 회전 오버슈트 후 삐딱하게 정착.
0.8s KPI 4장이 팡팡 팝인 (scale 0.6→1.12→1, 각 60ms 간격, 각자 다른 회전각으로 착지).
1.1s 스티커/도형 남발 레이어가 마지막에 "탁탁" 찍히며 등장 (회전 진동 + 살짝 튀는 바운스).
차트 막대는 형광색으로 아래에서 차오름 (600ms, cubic-bezier 튕김) — 막대마다 다른 형광색.

■ 마이크로 인터랙션 — 3개+ 번호로
1) 카드 hover: 하드 형광 그림자 거리 8px→14px + 회전 각 반전(살짝 반대로 기울며 떠오름)
2) KPI 숫자 hover: 배경 색면이 다음 형광으로 스냅 교체 (150ms, 톤 튐 — 마젠타→라임→파랑 순환)
3) 버튼/태그 active: 그림자가 0으로 붕괴하며 요소가 오른아래로 8px 눌림 (도장 찍기)
4) 스크롤 시 배경 형광 도형 레이어가 서로 다른 속도로 패럴럭스 (앞 도형 빠르게, 뒤 느리게)
5) 차트 막대 hover: 형광 보색 테두리 점멸 + 값 말풍선이 회전된 스티커로 튀어나옴

■ 시그니처
형광 다색 충돌 + 패턴/텍스처 배경 + 스티커/도형 남발 + 겹치는 회전 레이어.
서로 다른 형광 블록이 경계선 없이 부딪히고, 배경엔 도트·지그재그·체크 텍스처가 늘 살아 있으며,
그 위로 회전된 도형 스티커(별·번개·하트·스마일)가 아무렇게나 붙은 듯 흩뿌려지고,
카드·차트·헤더가 z축으로 겹쳐 물리며 저마다 다른 각도로 삐딱하게 정착한다.
이 네 가지(형광 충돌·패턴 배경·스티커 남발·겹침 회전)가 한 화면에 동시에 없으면 맥시멀리즘이 아니다.

■ 접근성·안전장치
- prefers-reduced-motion: 폭발 등장·패럴럭스·점멸 전부 제거, 최종 정착 상태로 즉시 표시.
  누름(active) 반응만 유지. 정지 상태에서도 형광 충돌·패턴·겹침·회전은 그대로 보인다.
- 본문 가독 사수: 텍스트는 반드시 크림/흰 색면 위 잉크블랙 배치 —
  형광 배경 위 본문 텍스트 금지 (형광은 도형·강조·색면·스티커 전용).
  KPI 숫자 등 형광 위 텍스트는 검정 + text-stroke 로 대비 확보.
- JS 실패 폴백: 애니메이션·패럴럭스가 안 돌아도 정적 레이아웃에서 형광 충돌과
  패턴 배경·겹침·회전이 그대로 보이도록 CSS transform 초기값을 최종 상태로 설정.
- focus-visible: 형광 배경에 묻히지 않도록 검정 3px + 흰 1px 이중 아웃라인.
- 회전은 시각 장식만 — 실제 클릭 히트영역은 회전 전 사각으로 유지.

■ 금지 (정체성의 절반)
여백 위주 절제, 차분함·미니멀리즘, 얌전한 그리드 정렬(모든 카드가 각 없이 수평 정렬),
블러 있는 소프트 섀도, 파스텔 저채도, 형광 4색 이하 팔레트, 겹침 없는 평면 배치,
회전 0°만 사용, 보더 없는 카드, 배경 단색·무패턴(패턴/텍스처 없는 조용한 배경 금지).
색을 소수 강조로만 쓰고 화면 대부분을 중립색으로 채우는 인색한 컬러 운용 금지.

  — 클러스터메이트(05 브루탈리즘·17 Y2K)와 겹치지 않도록 아래를 명시적으로 금지 —
  · [05 브루탈리즘 회피] 흑백·무채색·그레이스케일 화면 금지, 단색(모노크롬) 팔레트 금지,
    구조적 절제·시스템 폰트 단일 사용·앙상한 뼈대 미학 금지, 색 없는 검정 보더+흰 배경만의 구성 금지.
    검정은 오직 본문 텍스트·보더 앵커로만 — 검정이 화면의 주역 색이 되면 브루탈리즘으로 새므로 회귀.
  · [17 Y2K 회피] 크롬/메탈릭 그라디언트·은빛 베벨 금지, 신스웨이브 원근 격자(perspective grid) 금지,
    파랑→보라 네온 그라디언트 하늘·홀로그램 광택·글로시 리플렉션 금지, Y2K 특유의 차가운
    사이버 팔레트(실버·일렉트릭 퍼플 단독) 금지. 형광은 매트한 색면으로 부딪히게 두고 금속 광택을 입히지 않는다.
  · 단일 폰트 사용 금지 — 반드시 디스플레이 2종+ 혼용(브루탈리즘의 시스템 단일 폰트와 구분).

■ 자체 크리틱 루프
1) 한 화면에 형광이 5색 이상 동시에 살아 있는가? → 4색 이하면 색 추가(6색 지향).
2) 형광 충돌·패턴 배경·스티커 남발·겹침 회전 4요소가 모두 존재하는가? → 빠진 것 보강.
3) 배경이 조용한 단색 구간을 만들었는가? → 도트·지그재그·체크 텍스처로 즉시 채움.
4) 정렬이 너무 얌전해졌는가(회전·오프셋 소실)? → 카드 각·z축 겹침 재도입.
5) 화면이 흑백·크롬·모노크롬으로 새고 있는가(클러스터메이트 침범)? → 매트 형광 다색으로 되돌림.
6) 폭발 속에서도 본문 텍스트가 검정·고대비로 읽히는가(형광 위 본문 없는가)? → 색면 교정.
```
