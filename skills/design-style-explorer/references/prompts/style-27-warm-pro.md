# STYLE 27 — 웜 프로페셔널 (매력 축: 따뜻한 품격)

매력 가설: 신뢰는 차가움에서만 오지 않는다. 크림빛 종이 위에 테라코타와 올리브가 절제되어 놓이면, 사람은 긴장을 풀면서도 "제대로 만든 것"이라 느낀다.
고도화 = 웜 뉴트럴 계조의 미세한 온도차와 여백의 넉넉함. 마스터 프롬프트 뒤에 붙여 사용한다.

```
[스타일 지시 — 웜 프로페셔널 v2]

■ 무드
오후 햇살이 든 잘 정돈된 사무실. 크림빛 종이 위에 흙 계열 색이 조용히 얹혀 있다.
따뜻하지만 들뜨지 않는다 — 편안함과 품격이 동시에 성립하는 지점.
과시하지 않는 전문성. 색을 아껴 쓸수록 신뢰가 오른다는 감각.

■ 팔레트 — 크림 배경 + 흙 계열 강조 (웜 뉴트럴)
배경 크림 #FAF6F0 / 카드 표면 #FFFDF9 / 한 단 낮은 표면 #F3ECE2 / 경계선 #E7DDCF
본문 텍스트 #3A342E / 보조 텍스트 #6B6157 / 흐린 라벨 #9A8F82
주색 테라코타 #C97B5A / 진한 테라코타(hover) #B4633F / 테라코타 연한 배경 #F3E0D6
보조색 올리브 #8A9A5B / 진한 올리브 #6E7D45 / 올리브 연한 배경 #E7EBD8
상태색: 성공 #6E7D45(올리브 계열 재사용) / 주의 #D8973C(머스터드) / 문제 #B15A48(벽돌, 순빨강 금지)
차트는 테라코타·올리브·머스터드·모래(#C9B79C)의 흙 4색조로만 구성 — 온도 통일.

■ 타이포 — 휴먼 산세리프 (부드러운 인상)
본문 한글은 Pretendard 가정. 영문·숫자·헤드라인은 Nunito Sans 또는 Mulish (구글폰트) 명시 로드.
Display 34/42 700 · H1 26/34 700 · H2 20/28 600 · H3 17/24 600 ·
본문 15/24 400 · 보조 13/20 400 · 라벨·KPI 캡션 12/16 600 letter-spacing 0.4px 대문자.
KPI 수치는 28~40px 700, 살짝 넓은 자간으로 여유. letter-spacing 본문 0.1px로 숨 트이게.

■ 형태
radius 부드럽게 — 카드 14px, 버튼·인풋 10px, 칩·뱃지 pill(999px), 아바타 원형.
경계는 1px #E7DDCF 얇은 선 또는 아주 옅은 그림자 하나 (택1, 남발 금지):
  카드 그림자 0 1px 2px rgba(80,60,40,.05), 0 8px 24px rgba(120,90,60,.06) — 흙빛 저채도 그림자.
구분선은 실선 대신 배경 톤차(크림 vs 한 단 낮은 표면)로 처리해 딱딱함 제거.
여백 넉넉히: 카드 패딩 24~28px, 섹션 간 32px 이상. 밀도보다 호흡.

■ 히어로 연출 — 종이에 잉크가 스미듯 (로드 시퀀스)
0.0s 상단바가 크림 배경에 자연스레 나타남 (페이드 200ms, 이동 없음 — 차분하게)
0.15s KPI 카드 4장이 y+10px→0 + 페이드로 왼→오 60ms씩 stagger (각 260ms, ease-out)
0.5s KPI 수치가 0에서 실제 값까지 카운트업 (700ms, ease-out) — 채워지는 만족감
0.7s 차트 막대/영역이 baseline에서 테라코타로 차오름 (600ms), 라인은 왼→오 draw
1.0s 테이블 행이 상단부터 40ms 간격 페이드인, 첫 행에 옅은 테라코타 배경이 잠깐 번졌다 사라짐

■ 마이크로 인터랙션 (3개+)
1) 버튼 hover: 테라코타 배경이 진한 테라코타로 240ms 전환 + y-1px 미세 상승, 그림자 살짝 짙어짐
2) 카드 hover: 표면이 #FFFDF9→더 밝게 + 흙빛 그림자 확장 (200ms), 경계선 톤 한 단 진해짐
3) 테이블 행 hover: 행 배경에 테라코타 연한 배경(#F3E0D6) 스며듦 (160ms), 좌측에 3px 올리브 인디케이터 슬라이드 인
4) 탭·세그먼트: 선택 표시가 pill 형태로 뒤에서 슬라이드 (250ms ease), 텍스트는 보조→본문 색으로 진해짐
5) 인풋 focus: 경계가 테라코타로 물들고 바깥에 4px #F3E0D6 halo 부드럽게 확장 (180ms)

■ 시그니처
크림빛 종이 위 테라코타+올리브의 흙 계열 절제 팔레트 그 자체. 차가운 파랑을 하나도 쓰지 않고도
KPI·차트·상태색까지 웜 뉴트럴 4색조로 통일한 온도의 일관성이 첫눈에 "따뜻한 프로페셔널"로 읽힌다.

■ 접근성·안전장치
- prefers-reduced-motion: 카운트업·stagger·번짐 제거, 최종 상태 즉시 표시 (색 전환만 유지)
- JS 실패 폴백: 카운트업 없이 최종 수치 정적 표시, hover 효과는 CSS :hover만으로 동작
- focus-visible: 테라코타 2px 아웃라인 + offset 2px + 옅은 halo (그림자 단독 식별 금지)
- 대비: 크림(#FAF6F0) 위 본문 #3A342E는 4.5:1 이상 확보, 테라코타 텍스트는 큰 크기/굵기에서만 사용
- 상태색은 색만으로 구분 금지 — 성공/주의/문제에 아이콘·라벨 병행, 흙 4색조는 명도차로도 구별

■ 금지 (정체성의 절반)
차가운 파랑·시안 계열을 주색이나 강조로 사용(흙 온도 이탈), 다크 배경 강제(크림 기본 유지),
형광·비비드·네온 채도, 하이테크/사이버 느낌(글로우·격자·HUD·모노스페이스 남발),
순검정(#000) 텍스트·1px 순검정 하드 보더, 순빨강(#F00)·순초록 원색 상태색,
그림자 남발(모든 요소에 그림자) 또는 딱딱한 다중 실선, radius 0 또는 20px 초과,
차가운 회색(중립 그레이) 텍스트/보더로 온도 깨기, 세리프 헤드라인(휴먼 산세리프 유지).

■ 자체 크리틱 루프
1) 화면 어디에도 차가운 파랑/시안이 주색·강조·차트색으로 섞이지 않았는가? — 이탈 색 색출
2) 배경·표면·텍스트·보더가 모두 웜 뉴트럴(황·갈 기미)인가, 중립 그레이가 스며들지 않았는가?
3) 색을 아꼈는가? — 테라코타·올리브가 강조로만 쓰이고 크림 여백이 화면의 주인인지 점검
4) 형광·다크·글로우·모노스페이스 등 하이테크 잔재나 순원색 상태색이 없는가?
```
