# STYLE 20 — 다크 대시보드 (매력 축: 데이터 밀도/관제)

매력 가설: 한 화면에서 시스템 전체를 장악한다. 매력은 여백이 아니라 밀도 —
숫자·차트·상태가 촘촘히 모여 있을수록 "관제하고 있다"는 통제감이 커진다.
고도화 = 밀도를 늘리면서도 임계 색으로 눈을 즉시 유도하는 질서. 마스터 프롬프트 뒤에 붙여 사용한다.

```
[스타일 지시 — 다크 대시보드 v2]

■ 무드
Grafana / Datadog 식 실시간 관제 콘솔. 어두운 벽면에 걸린 대형 모니터.
촘촘한 위젯 그리드가 쉬지 않고 갱신되는, 살아 있는 관제실. 장식은 없다, 정보만 있다.

■ 팔레트 (다크 + 데이터 임계색)
배경 3단 레이어 — 페이지 #0d1117 / 위젯 카드 #161b22 / 카드 내부 트랙 #21262d /
얇은 그리드·보더 라인 #30363d (1px, 절대 두껍게 X) /
본문 텍스트 #c9d1d9 / 보조·라벨 #8b949e (작게, 흐리게) /
데이터 시리즈: 청록 #2dd4bf(primary) · 파랑 #58a6ff · 보라 #bc8cff (구분용) /
임계 색 (상태 신호): 정상 #3fb950(green) · 주의 #d29922(yellow) · 경고 #db6d28(orange) · 위험 #f85149(red)
— 색은 예쁘라고 쓰는 게 아니라 임계값을 말하려고 쓴다. 무채색 위에 임계색만 튀게.

■ 타이포 (산세리프 + tabular-nums)
라벨·본문: Inter 또는 Pretendard 400~500, 12~13px 작게 촘촘하게 (line-height 1.35).
숫자 (KPI 값 · 표 수치 · 축 눈금): 반드시 font-variant-numeric: tabular-nums + font-feature-settings 'tnum' 1
— 자릿수 흔들림 없이 세로로 정렬되는 게 관제의 기본기.
KPI 대표 숫자만 20~28px, 나머지는 전부 작게. 큰 타이틀 남발 금지.
등폭 톤이 필요한 로그·ID 열은 JetBrains Mono 12px.

■ 형태
- 위젯 카드: radius 6px, #161b22, 1px #30363d 보더. 그림자 거의 없음(관제는 플랫).
- 위젯 헤더: 11px 대문자 라벨(letter-spacing 0.04em) #8b949e + 우측 작은 상태 배지/시간.
- 촘촘한 그리드: CSS grid 12컬럼, gap 8~12px. 한 화면에 위젯 8개 이상 들어가도록 밀도 유지.
- 상태 배지: 아주 작은 pill (11px), 임계색 배경 12% opacity + 임계색 텍스트 100%.
- 스파크라인: 카드 우측/하단에 40px 높이 미니 라인. 축·라벨 없이 추세만.
- 게이지: 반원 또는 얇은 도넛, 트랙 #21262d + 값 구간을 임계색으로.

■ 히어로 연출 (관제답게 — 거창한 등장 금지)
로드 0~150ms: 위젯 카드들이 좌상단→우하단 순서로 40ms 간격 stagger 페이드인(opacity 0→1, translateY 4px). 큰 슬라이드·확대 X.
150~900ms: 차트 라인은 좌→우로 stroke-dashoffset 그려지고, KPI 숫자는 count-up(0→실제값, 700ms, easeOutExpo)으로 tabular-nums 자리 유지하며 올라감.
로드 후: 상단에 "LIVE ●" 인디케이터가 1.2s 주기로 은은히 점멸. 관제가 "돌고 있다"는 신호.

■ 마이크로 인터랙션 (3개+)
1) 위젯 hover: 보더 #30363d→#58a6ff 로 점등 + 카드 배경 미세 +4% 명도. 100ms.
2) 차트 hover: 세로 크로스헤어 라인 등장 + 그 지점 값 툴팁(다크 팝오버, tabular-nums). 데이터 포인트 dot 확대.
3) 표 행 hover: 행 배경 #161b22→#1c2230 + 좌측 2px 임계색 액센트 바. 상태 열 값에 따라 색 결정.
4) 상태 배지 위험(red): 1.5s 주기 opacity 0.6↔1 은은한 pulse (주의 환기 — 단, 과격한 깜빡임 X).
5) 임계값 돌파 순간: 해당 KPI 숫자 배경이 임계색으로 200ms 플래시 후 fade.

■ 시그니처
차트·게이지·스파크라인·상태 배지가 빈틈없이 들어찬 고밀도 위젯 그리드 —
그 위를 임계 색(초록·노랑·주황·빨강)만이 신호등처럼 관통한다. 이 밀도+임계색 조합이 정체성.

■ 접근성·안전장치
- prefers-reduced-motion: stagger·count-up·crosshead·pulse 전부 제거, 최종 상태 즉시 렌더. LIVE 점멸도 정지(정적 dot).
- 색맹 대비: 임계 상태를 색으로만 구분 금지 — 배지에 텍스트 라벨(OK/WARN/CRIT) 또는 아이콘(●▲■) 병기. red/green 동시 의존 X.
- JS 실패 폴백: 차트·count-up이 JS라면 <noscript> 또는 초기 HTML에 최종 수치·정적 SVG 라인 렌더. 데이터가 사라지지 않게.
- focus-visible: 위젯·행·인터랙티브 요소에 #58a6ff 2px 아웃라인(다크 배경 대비 확보). 키보드 순회 순서 = 그리드 읽기 순서.
- 대비 확보: 본문 #c9d1d9 / 보조 #8b949e 는 #0d1117 위 4.5:1 이상 유지. 임계색 텍스트도 배경 대비 검증.

■ 금지 (정체성의 절반)
넓은 여백·성긴 레이아웃, 큰 히어로 배너·풀블리드 이미지, 밝은/화이트 배경,
큰 장식 요소·일러스트·그라디언트 배경, 둥근 pill 카드·큰 radius(12px+),
두꺼운 보더(2px+)·강한 그림자, 큰 타이틀 남발, 숫자에 비례 폰트(자릿수 흔들림),
카드 하나가 화면 절반 차지하는 성긴 배치, 파스텔·저채도 감성 색(임계 신호 흐려짐).

■ 자체 크리틱 루프
1) 한 화면에 위젯이 충분히 밀집(8개+)했는가, 아니면 성겨서 관제감이 죽었는가? → 그리드 gap·컬럼 재조정
2) 모든 숫자가 tabular-nums 로 세로 정렬되는가? → 표·KPI·축 눈금 전수 검사
3) 임계색이 상태 신호로만 절제되어 쓰였는가, 장식으로 남발됐는가? → 무채색 비중 재확인
4) 색맹 사용자가 상태(정상/위험)를 텍스트·아이콘 없이 색만으로 구분해야 하는 곳은? → 라벨 병기 보정
```
