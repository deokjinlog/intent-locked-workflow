# STYLE 20 — 다크 대시보드 (매력 축: 데이터 시각 관제)

매력 가설: 한 화면에서 시스템 전체를 눈으로 장악한다. 매력은 여백도 아니고 텍스트도 아니라
"시각화 밀도" — 차트·게이지·스파크라인 위젯이 촘촘한 그리드로 들어차고, 그 위를 임계 색이
신호등처럼 관통할수록 "관제하고 있다"는 통제감이 커진다. 이 스타일은 글자로 상태를 읽는 게 아니라
차트 모양과 임계 색으로 상태를 즉시 감지하게 만든다. 고도화 = 위젯 밀도를 늘리면서도 임계 색이
장식이 아니라 신호로만 절제되어 눈을 정확히 유도하는 질서. 마스터 프롬프트 뒤에 붙여 사용한다.

```
[스타일 지시 — 다크 대시보드 v2]

■ 무드
Grafana / Datadog / Prometheus 콘솔 식 실시간 데이터 시각 관제.
어두운 벽면에 걸린 대형 모니터, 촘촘한 차트 위젯 그리드가 쉬지 않고 갱신되는 살아 있는 관제실.
이 스타일의 언어는 "발광 연출"도 "텍스트 로그 스트림"도 아니라 "시각화 위젯" 그 자체다 —
숫자를 글자로 나열하는 게 아니라 라인 차트·게이지·스파크라인·히트맵으로 그려서 보여준다.
장식은 없다. 오직 데이터의 시각적 형태와 임계 색만 있다.

■ 팔레트 (다크 + 데이터 임계색)
배경 3단 레이어 — 페이지 #0d1117 / 위젯 카드 #161b22 / 카드 내부 차트 트랙·플롯 영역 #21262d /
얇은 그리드·보더 라인 #30363d (1px, 절대 두껍게 X — 차트 축 눈금 라인도 이 색으로 아주 흐리게) /
본문 텍스트 #c9d1d9 / 보조·라벨 #8b949e (작게, 흐리게) /
데이터 시리즈 (차트 라인·바 구분): 청록 #2dd4bf(primary) · 파랑 #58a6ff · 보라 #bc8cff · 자홍 #f778ba (4계열까지 구분) /
차트 면적 채우기(area fill): 시리즈 색 12~18% opacity 그라디언트로 라인 아래만 (배경 그라디언트 X) /
임계 색 (상태 신호): 정상 #3fb950(green) · 주의 #d29922(yellow) · 경고 #db6d28(orange) · 위험 #f85149(red)
— 색은 예쁘라고 쓰는 게 아니라 임계값을 말하려고 쓴다. 무채색 위에 임계색만 튀게.
발광(glow/neon bloom)이나 네온 채도는 쓰지 않는다 — 임계 색은 선명하되 빛나지 않는다.

■ 타이포 (산세리프 + tabular-nums, 단 텍스트는 차트의 보조)
라벨·본문: Inter 또는 Pretendard 400~500, 12~13px 작게 촘촘하게 (line-height 1.35).
숫자 (KPI 값 · 표 수치 · 차트 축 눈금 · 게이지 중앙 값): 반드시 font-variant-numeric: tabular-nums + font-feature-settings 'tnum' 1
— 자릿수 흔들림 없이 세로로 정렬되는 게 관제의 기본기.
KPI 대표 숫자만 20~28px, 나머지는 전부 작게. 큰 타이틀 남발 금지.
등폭 폰트(JetBrains Mono 12px)는 짧은 ID·해시 열에만 국소적으로 — 화면을 채우는 로그 텍스트 스트림 용도 X.
글자는 어디까지나 차트를 보조하는 라벨·수치일 뿐, 화면의 주인공은 시각화 위젯이다.

■ 형태
- 위젯 카드: radius 6px, #161b22, 1px #30363d 보더. 그림자 거의 없음(관제는 플랫). 발광 테두리·코너 브래킷 없음.
- 위젯 헤더: 11px 대문자 라벨(letter-spacing 0.04em) #8b949e + 우측 작은 상태 배지/갱신 시각.
- 촘촘한 그리드: CSS grid 12컬럼, gap 8~12px. 한 화면에 시각화 위젯 8개 이상, 최소 절반은 차트·게이지·스파크라인.
- 라인/에어리어 차트: 얇은 stroke(1.5px) 시리즈 색 + 라인 아래 12~18% area fill. 축 눈금 라인 #30363d 아주 흐리게, 4~5칸만.
- 바 차트: 얇은 세로 바(폭 6~10px, radius 1~2px), 값 구간이 임계 밴드 넘으면 그 바만 임계색.
- 게이지: 반원 또는 얇은 도넛(트랙 폭 6~8px), 트랙 #21262d + 값 아크를 임계 구간 색으로. 중앙에 tabular-nums 값.
- 스파크라인: 카드 우측/하단에 32~40px 높이 미니 라인, 축·라벨 없이 추세만. 마지막 점만 임계색 dot.
- 히트맵/상태 그리드: 작은 셀(10~14px) 격자, 셀 색으로 값 강도 표현(무채색 저강도 → 임계색 고강도).
- 상태 배지: 아주 작은 pill (11px), 임계색 배경 12% opacity + 임계색 텍스트 100%.

■ 히어로 연출 (관제답게 — 거창한 등장 금지, 발광·스캔라인 절대 X)
로드 0~150ms: 위젯 카드들이 좌상단→우하단 순서로 40ms 간격 stagger 페이드인(opacity 0→1, translateY 4px). 큰 슬라이드·확대 X.
150~900ms: 차트 라인은 좌→우로 stroke-dashoffset 그려지고, area fill은 그 뒤를 따라 아래에서 위로 채워지며,
게이지 아크는 0→값까지 스윕(sweep)으로 돌고, KPI 숫자는 count-up(0→실제값, 700ms, easeOutExpo)으로 tabular-nums 자리 유지하며 올라감.
로드 후: 상단에 "LIVE ●" 인디케이터가 1.2s 주기로 은은히 점멸(발광 아닌 opacity 페이드). 관제가 "돌고 있다"는 신호.
스캔라인 오버레이·CRT 왜곡·글로우 블룸·코너 HUD 프레임 애니메이션은 이 스타일에 존재하지 않는다.

■ 마이크로 인터랙션 (3개+)
1) 위젯 hover: 보더 #30363d→#58a6ff 로 점등(발광 아닌 색 전환) + 카드 배경 미세 +4% 명도. 100ms.
2) 차트 hover: 세로 크로스헤어 라인 등장 + 그 지점 값 툴팁(다크 팝오버, tabular-nums) + 해당 데이터 포인트 dot 확대. area fill은 그대로.
3) 게이지 hover: 아크 위 현재 구간의 임계 라벨(OK/WARN/CRIT)과 정확 값 툴팁 표시.
4) 표 행 hover: 행 배경 #161b22→#1c2230 + 좌측 2px 임계색 액센트 바. 상태 열 값에 따라 색 결정.
5) 상태 배지 위험(red): 1.5s 주기 opacity 0.6↔1 은은한 pulse (주의 환기 — 단, 과격한 깜빡임·네온 발광 X).
6) 임계값 돌파 순간: 해당 KPI 숫자·차트 마지막 점 배경이 임계색으로 200ms 플래시 후 fade.

■ 시그니처
차트·게이지·스파크라인·바·히트맵이 빈틈없이 들어찬 고밀도 "시각화" 위젯 그리드 —
글자로 상태를 읽는 게 아니라 그래프 모양으로 추세를, 임계 색(초록·노랑·주황·빨강)으로 위험도를 즉시 감지한다.
이 "시각화 밀도 + 신호등 임계색" 조합이 정체성. 텍스트 로그도, 발광 HUD도 아니다 — 차트가 주인공이다.

■ 접근성·안전장치
- prefers-reduced-motion: stagger·count-up·라인 드로우·게이지 스윕·크로스헤어·pulse 전부 제거, 최종 상태 즉시 렌더. LIVE 점멸도 정지(정적 dot).
- 색맹 대비: 임계 상태를 색으로만 구분 금지 — 배지에 텍스트 라벨(OK/WARN/CRIT) 또는 아이콘(●▲■) 병기. 차트 시리즈도 색+실선/점선/마커 모양으로 이중 구분. red/green 동시 의존 X.
- JS 실패 폴백: 차트·게이지·count-up이 JS라면 <noscript> 또는 초기 HTML에 최종 수치·정적 SVG 라인/아크 렌더. 데이터가 사라지지 않게.
- focus-visible: 위젯·행·차트·인터랙티브 요소에 #58a6ff 2px 아웃라인(다크 배경 대비 확보). 키보드 순회 순서 = 그리드 읽기 순서.
- 대비 확보: 본문 #c9d1d9 / 보조 #8b949e 는 #0d1117 위 4.5:1 이상 유지. 임계색 텍스트·차트 라인도 배경 대비 검증.

■ 금지 (정체성의 절반)
- 넓은 여백·성긴 레이아웃, 큰 히어로 배너·풀블리드 이미지, 밝은/화이트 배경.
- 큰 장식 요소·일러스트·그라디언트 배경, 둥근 pill 카드·큰 radius(12px+).
- 두꺼운 보더(2px+)·강한 그림자, 큰 타이틀 남발, 숫자에 비례 폰트(자릿수 흔들림).
- 카드 하나가 화면 절반 차지하는 성긴 배치, 파스텔·저채도 감성 색(임계 신호 흐려짐).
- [클러스터 분리 — 01 HUD 금지] 코너 브래킷/앵글 프레임, 발광 테두리(box-shadow glow)·네온 bloom·글로우 텍스트,
  스캔라인 오버레이·CRT 곡률·홀로그램 그리드 배경. 임계 색은 선명하되 절대 "빛나지" 않는다.
- [클러스터 분리 — 04 터미널 금지] 화면을 채우는 모노스페이스 텍스트 로그 스트림, 흐르는 콘솔 라인/타이핑 커서 연출,
  ASCII 아트, `>`/`$` 프롬프트 라인, 등폭 폰트로 도배한 텍스트 위주 레이아웃. 숫자는 로그로 흘리지 말고 반드시 차트·게이지로 시각화.
- 위젯 그리드에서 차트·게이지·스파크라인 없이 표와 텍스트만 나열하는 것 (시각화가 절반 미만이면 이 스타일이 아니다).

■ 자체 크리틱 루프
1) 한 화면에 시각화 위젯이 충분히 밀집(8개+)했고, 그 중 최소 절반이 차트·게이지·스파크라인인가? 표·텍스트만 성기게 남았으면 관제감·시각화 정체성이 죽는다 → 그리드 gap·컬럼·위젯 종류 재조정
2) 모든 숫자가 tabular-nums 로 세로 정렬되는가? → 표·KPI·차트 축 눈금·게이지 값 전수 검사
3) 임계색이 상태 신호로만 절제되어 쓰였는가, 장식으로 남발됐는가? → 무채색 비중 재확인
4) 발광·스캔라인(01 HUD) 이나 텍스트 로그 스트림(04 터미널) 냄새가 섞이지 않았는가? → 코너 프레임·글로우·모노스페이스 도배 여부 점검, 발견 시 즉시 차트 위젯으로 대체
5) 색맹 사용자가 상태(정상/위험)·차트 시리즈를 텍스트·아이콘·마커 모양 없이 색만으로 구분해야 하는 곳은? → 라벨·마커 병기 보정
```
