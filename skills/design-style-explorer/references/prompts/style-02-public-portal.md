# STYLE 02 — 공공데이터 포털 (매력 축: 신뢰)

매력 가설: 이 스타일의 매력은 "아무것도 숨기지 않음"이다.
고도화 = 모션 추가가 아니라 정보 설계의 정직함. 마스터 프롬프트 뒤에 붙여 사용한다.

```
[스타일 지시 — 공공데이터 포털 v2]

■ 무드
정부·공공기관 통계 시스템. 장식 없는 신뢰와 명료함.
흑백으로 인쇄해도 완전한 화면이어야 한다.

■ 팔레트
배경 #F5F6F8 / 기관 네이비 #00447C / 표머리 하늘 #E8F1F9 / 텍스트 #1A1A1A /
보더 #D5DBE1 / 성공 #1A7F3C / 문제 #C62828 / 대기 #9A6700

■ 타이포
Pretendard 단일 사용. 굵기 대비로만 위계 표현. 본문 13px — 작아도 명료하게.

■ 형태·질감
직각(radius 0), 격자 보더의 표, 최상단 기관 헤더 바(네이비),
좌측 4px 네이비 보더의 섹션 제목, 표 짝수행 줄무늬(#FAFBFC).

■ 히어로 연출
없음. 정적임이 곧 신뢰다. 로드 즉시 완성 화면.

■ 고도화 포인트 (모션 대신 정보 설계 — 전부 필수)
1) 모든 수치에 단위 명기, 금액·수량은 우측 정렬 + tabular-nums로 자릿수 정렬
2) 차트·표 하단에 "※ 출처 · 집계 기준(기간) · 갱신 주기" 각주
3) 표 아래 "전체 목록 보기 +" 등 다음 행동 링크
4) @media print 인쇄 CSS — 배경 제거, 보더 유지, 실제 출력 가능하게
5) 접근성: table에 caption, th에 scope 지정

■ 마이크로 인터랙션
링크·표 행 hover에 하늘색(#E8F1F9) 배경 정도만. 장식적 반응 금지.

■ 시그니처
각주와 출처 표기 — 관공서 문서의 문법을 정확하게 재현한 표.

■ 금지
그림자, 그라디언트, 이모지, 애니메이션, 라운드, 카운트업, 장식 아이콘,
배경 이미지, 두 가지 이상의 강조색 동시 사용

■ 자체 크리틱 루프
1) 흑백으로 인쇄해도 모든 정보가 구분되는가?
2) 단위·출처·기준일이 빠진 수치가 하나라도 있는가? → 보강
3) 장식으로만 존재하는 요소가 있는가? → 제거
4) 표의 정렬(텍스트 좌, 숫자 우, 상태 중앙)이 일관되는가?
```
