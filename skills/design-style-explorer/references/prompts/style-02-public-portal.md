# STYLE 02 — 공공데이터 포털 (매력 축: 신뢰)

매력 가설: 이 스타일의 매력은 "아무것도 숨기지 않음"이다.
고도화 = 모션 추가가 아니라 정보 설계의 정직함. 출처·기준일·단위·각주가 화면의 주인공이다.
네이비 액센트 하나로 위계를 세우고, 흑백 인쇄로도 완전한 문서가 된다. 마스터 프롬프트 뒤에 붙여 사용한다.

```
[스타일 지시 — 공공데이터 포털 v2]

■ 무드
정부·공공기관 통계 시스템. 장식 없는 신뢰와 명료함.
행정 문서의 문법 — 표, 각주, 출처 표기, 집계 기준일, 갱신 주기 — 을 정확히 재현한다.
흑백으로 인쇄해도 완전한 화면이어야 한다. 화면은 "읽는 것"이 아니라 "열람·인용·출력하는 것"이다.

■ 팔레트 (단색 기관색 원칙 — 강조는 오직 네이비 하나)
배경 #F5F6F8 / 기관 네이비 #00447C / 네이비 진하게(헤더·링크 hover 텍스트) #003060 /
표머리 하늘 #E8F1F9 / 행 hover 하늘 #EEF5FC / 텍스트 #1A1A1A / 보조 텍스트 #55606B /
보더 #D5DBE1 / 표 짝수행 줄무늬 #FAFBFC / 성공 #1A7F3C / 문제 #C62828 / 대기 #9A6700
→ 성공·문제·대기 3색은 상태 배지·수치 부호에만 국소 사용. 주액센트는 네이비 단일. 페이지 전체에서 두 강조색이 동시에 넓은 면을 차지하지 않는다.

■ 타이포 (본문 밀도 우선 — 초대형 헤드라인 없음)
Pretendard 단일 사용. 굵기 대비로만 위계 표현.
본문 13px, 표 셀 12.5px, 각주 11px — 작아도 명료하게. line-height 1.55로 다행(多行) 표를 촘촘히.
페이지 제목 20px / 섹션 제목 15px bold — 그 이상으로 키우지 않는다. 정보량이 크기보다 앞선다.
숫자는 tabular-nums 고정폭. 자릿수 콤마(1,234,567) 필수. 소수 자릿수 표 안에서 통일.

■ 형태·질감 (질서 있는 표 격자 — 노출형 대형 그리드 아님)
직각(radius 0), 1px 격자 보더의 표가 화면의 뼈대.
최상단 기관 헤더 바(네이비 배경 + 흰 로고자리 + 우측 "최종 갱신일"),
좌측 4px 네이비 보더의 섹션 제목, 표 짝수행 줄무늬(#FAFBFC), th에 하늘 배경(#E8F1F9).
카드가 필요하면 흰 배경 + 1px 보더 + 좌상단 라벨. 여백은 정보를 정렬하기 위한 최소치로만.

■ 히어로 연출
없음. 정적임이 곧 신뢰다. 로드 즉시 완성 화면.
첫 화면 최상단은 "이 데이터가 무엇인지 + 집계 기준일 + 출처 기관"을 한 줄로 명시하는 안내 밴드.

■ 고도화 포인트 (모션 대신 정보 설계 — 전부 필수)
1) 모든 수치에 단위 명기. 금액·수량은 우측 정렬 + tabular-nums로 자릿수 정렬. 증감은 ▲/▼ + 부호색(성공/문제)로만.
2) 차트·표 하단에 "※ 출처 기관 · 집계 기준(기간) · 갱신 주기 · 다음 갱신 예정일" 각주 4요소를 빠짐없이.
3) 표 아래 "전체 목록 보기 +", "원자료(CSV) 내려받기", "이전 회차 비교" 등 다음 행동 링크.
4) @media print 인쇄 CSS — 배경·hover 제거, 격자 보더 유지, 각주·페이지 번호 표시, 실제 출력 가능하게.
5) 접근성: table에 caption, th에 scope, 상태 배지에 텍스트 라벨 병기(색만으로 의미 전달 금지).
6) 데이터 없음/비공개 셀은 "–" 또는 "비공개" 명시. 빈칸을 남기지 않는다(누락도 정보다).

■ 마이크로 인터랙션
링크·표 행 hover에 하늘색(#EEF5FC) 배경 + 링크 텍스트 네이비 진하게(#003060) 정도만.
정렬 가능한 표머리에 ↕ 표식. 장식적 반응·이징 곡선·부드러운 슬라이드 금지.

■ 시그니처
각주와 출처 표기 — 관공서 통계표의 문법을 정확하게 재현한 표.
"집계 기준일 2026-06-30 · 출처 ○○청 · 매월 갱신 · 단위: 천 원" 같은 메타 한 줄이 화면마다 반드시 존재한다.

■ 금지 (일반)
그림자, 그라디언트, 이모지, 애니메이션, 카운트업, 라운드, 장식 아이콘,
배경 이미지, 두 가지 이상의 강조색이 동시에 넓은 면을 차지하는 것.

■ 금지 (클러스터메이트와의 분리 — 반드시 지킬 것)
· 07 미니멀 모노크롬처럼 보이지 말 것:
  - 정보를 덜어낸 극단 여백·큰 빈 공간 연출 금지. 여백은 정렬용 최소치이며, 남는 자리는 각주·메타·다음 링크로 채운다.
  - 무채색만으로 마감 금지 — 네이비 기관색이 헤더 바·섹션 보더·링크에 반드시 존재해야 한다.
  - "적을수록 좋다"의 절제미 금지. 이 스타일은 "빠짐없이 밝힌다"의 밀도미다.
· 18 스위스 인터내셔널처럼 보이지 말 것:
  - 빨강을 주액센트로 쓰지 말 것. 빨강(#C62828)은 오직 "문제" 상태 배지·음수 부호에만 국소 사용.
  - 비대칭 초대형 타이포·화면을 가로지르는 대형 헤드라인 금지. 제목은 20px를 넘기지 않는다.
  - 강한 격자선 노출·모듈러 그리드를 디자인 언어로 전시하는 것 금지. 격자는 오직 데이터 표의 셀 보더로만 존재한다.
  - Helvetica/Neue Haas류 스위스 조판 뉘앙스 금지. Pretendard 행정체 밀도를 유지한다.

■ 자체 크리틱 루프
1) 흑백으로 인쇄해도 모든 정보(상태·증감·위계)가 구분되는가? → 색 의존 발견 시 텍스트 라벨 병기
2) 단위·출처·집계 기준일·갱신 주기가 빠진 수치·표가 하나라도 있는가? → 보강
3) 장식으로만 존재하는 요소, 정보 없는 빈 여백이 있는가? → 제거하고 메타·각주로 대체
4) 표의 정렬(텍스트 좌, 숫자 우+tabular-nums, 상태 중앙)이 일관되는가?
5) 07처럼 무채색·극단 여백으로 흐르지 않았는가? → 네이비 기관색·정보 밀도 복원
6) 18처럼 빨강·초대형 타이포·격자 전시로 흐르지 않았는가? → 네이비 단일 액센트·표 격자·13px 밀도로 복원
```
