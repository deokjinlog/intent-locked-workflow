# STYLE 23 — 엔터프라이즈 B2B (매력 축: 실용 신뢰)

매력 가설: 화려함이 아니라 예측 가능함이 신뢰를 만든다. Atlassian·Salesforce가 검증한 문법 — 좌측 사이드바에 모든 것이 있고, 정보는 빽빽하되 구획선으로 정돈되어, 처음 로그인한 관리자도 5분 만에 길을 안다.
고도화 = 밀도를 유지하면서도 구획선·라벨·상태 뱃지로 스캔 가능성을 잃지 않는 정보 위계. 마스터 프롬프트 뒤에 붙여 사용한다.

```
[스타일 지시 — 엔터프라이즈 B2B v2]

■ 무드
Atlassian Jira / Salesforce Lightning 계열 업무용 SaaS. 하루 8시간 들여다보는 도구.
감탄이 아니라 신뢰 — 어제와 오늘이 똑같이 배치돼 있어 근육 기억으로 조작한다.
라이트 그레이 캔버스 위에 하얀 카드가 얇은 보더로 구획되고, 실용 네이비가
클릭 가능한 것만 가리킨다. 절제·정돈·정보 밀도. 장식이 아니라 기능.

■ 팔레트 — 라이트 그레이 캔버스 + 실용 네이비 + 뉴트럴
캔버스 #F4F5F7 / 카드·표면 #FFFFFF / 구획 보더 #DFE1E6 / 강한 구분선 #C1C7D0
Primary 네이비 #0052CC / hover #0747A6 / active #043584 / 링크 #0065FF
텍스트 강조 #172B4D / 본문 #42526E / 보조·캡션 #6B778C / 비활성 #97A0AF
서브틀 배경(hover row·zebra) #FAFBFC / 선택 행 #DEEBFF / 좌측 사이드바 #FFFFFF·활성항목 #DEEBFF
상태 뱃지: 성공 #00875A/배경 #E3FCEF · 진행 #0052CC/배경 #DEEBFF ·
경고 #FF8B00/배경 #FFFAE6 · 문제 #DE350B/배경 #FFEBE6 · 중립 #42526E/배경 #F4F5F7
차트 계열: 네이비 #0052CC → #4C9AFF → #B3D4FF 순 단계 + 뉴트럴 그레이 보조.
채도 낮은 유틸 컬러만 — 원색 축제 금지.

■ 타이포 — 산세리프 실용 스택
본문 한글 Pretendard 가정. 영문·숫자·라벨은 Inter (구글폰트) 명시 로드.
페이지 타이틀 24/28 600 · 섹션 헤더 20/24 600 · 카드 타이틀 16/20 600 ·
본문 14/20 400 · 조밀 테이블/메타 13/18 400 · 라벨·오버라인 11/16 600 대문자
letter-spacing +0.5px · KPI 수치 28/32 600 tabular-nums.
letter-spacing 본문 0, 라벨만 넓힘. 폰트 웨이트는 400/500/600 세 단만 — 300·800 금지.

■ 형태
radius 작게 — 카드·모달 4px, 버튼·인풋·뱃지 3px, 아바타·상태점만 원형.
보더 우선 위계: 카드는 1px #DFE1E6 보더 + 아주 얕은 그림자 0 1px 1px rgba(9,30,66,.08).
구획선이 주역 — 섹션 사이 1px 수평 보더, 사이드바와 본문 사이 1px 수직 보더.
간격 8px 그리드(4/8/12/16/24). 밀도 우선이라 카드 패딩 16px, 테이블 행 높이 36~40px.
좌측 사이드바 240px 고정 폭 + 접기 시 56px 아이콘 레일. 상단바 56px 고정.
버튼: primary는 네이비 솔리드, secondary는 #FFFFFF + #DFE1E6 보더, subtle은 보더 없이 텍스트만.

■ 히어로 연출 — 셸이 먼저, 콘텐츠가 채워짐 (로드 시퀀스, 절제된 속도)
0.0s 상단바 + 좌측 사이드바(셸)가 페이드 없이 즉시 렌더 — 뼈대는 항상 그 자리
0.1s 본문 영역에 KPI 카드 4장이 좌→우 순서로 y+4px→0 + 페이드 (각 160ms, 40ms stagger)
0.35s 차트 카드의 막대/라인이 baseline에서 320ms ease-out으로 한 번에 그려짐
0.45s 테이블이 헤더 먼저 뜬 뒤 행이 스켈레톤 그레이→실데이터로 상단부터 20ms 간격 교체
0.7s 사이드바 현재 메뉴에 네이비 좌측 3px 인디케이터 바가 위→아래로 짧게 슬라이드
연출 총량 1초 미만 — 요란한 등장 없음, "이미 준비돼 있었다"는 인상.

■ 마이크로 인터랙션 (이 스타일의 실체)
1) 테이블 행 hover: 배경 #FAFBFC로 즉시 전환 + 행 우측 끝에 인라인 액션 아이콘(편집·삭제·⋯)
   페이드인. 선택 시 #DEEBFF + 좌측 2px 네이비 마커. (transition 100ms)
2) 사이드바 항목 hover: 배경 #F4F5F7, active 항목은 #DEEBFF + 네이비 좌측 3px 바 상시 표시.
   접기 토글 시 라벨이 폭 0으로 접히고 툴팁으로 대체 (180ms).
3) 정렬 가능한 컬럼 헤더 클릭: ▲/▼ 아이콘 토글 + 헤더 텍스트 네이비화, 행 재정렬은 즉시.
4) 버튼: primary hover #0747A6로 어두워짐(150ms), active #043584 + 눌림 없음(그림자 유지).
   secondary hover 보더 #C1C7D0 + 배경 #FAFBFC.
5) 상태 뱃지·필터 칩: 클릭 시 배경 톤 유지한 채 1px 네이비 보더로 활성 표시, 제거 ✕ 인라인.

■ 시그니처
좌측 240px 네이비-악센트 사이드바(활성 항목 #DEEBFF + 3px 좌측 바) + 라이트 그레이
캔버스 위 얇은 보더로 구획된 고밀도 화이트 카드/테이블. 이 "사이드바 헤비 + 구획선 +
실용 파랑 링크"의 조합이 화면을 즉시 업무용 B2B 콘솔로 읽히게 하는 각인.

■ 접근성·안전장치
- prefers-reduced-motion: KPI stagger·차트 그리기·인디케이터 슬라이드 제거, 상태만 즉시 반영
- JS 실패 폴백: 테이블 정렬·필터 없이도 서버 렌더 순서로 읽히고, 행 hover/선택은 CSS만으로 동작
- focus-visible: #0052CC 2px 아웃라인 + offset 1px (라이트 배경이라 그림자 대신 명시 링)
- 대비: 보조 텍스트 #6B778C도 #FFFFFF 위 4.5:1, 상태 뱃지는 색+텍스트+아이콘 3중 표기(색맹 대응)
- 밀집 테이블 최소 행 높이 36px·터치 타겟 인라인 액션 32×32 히트영역, 컬럼 최소 폭 보장
- 사이드바 접기 시에도 aria-label 유지, 키보드 Tab 순서는 사이드바→상단바→본문

■ 금지 (정체성의 절반)
화려한 히어로 연출·큰 스케일 등장 애니메이션, 그라디언트 폭발·다색 그라디언트 배경,
큰 여백 낭비(hero 여백·거대 폰트로 카드 한두 개만 띄우는 랜딩식 레이아웃),
형광·네온 채도, 다크 배경 강제(라이트 그레이 캔버스 기본), backdrop-filter 글래스,
radius 12px 초과 둥근 카드, 세리프·손글씨 서체, 폰트 웨이트 300/800 극단,
장식용 일러스트·이모지 남발, 상단 가로 메가 내비만 쓰고 좌측 사이드바 생략,
그림자 두껍게(elevation 과장) 카드 띄우기 — 보더가 주역이고 그림자는 얕게.

■ 자체 크리틱 루프
1) 좌측 사이드바가 240px로 존재하며 활성 항목에 네이비 인디케이터가 붙어 있는가?
2) 카드·섹션·테이블이 1px 보더/구획선으로 명확히 나뉘고, 그림자에 의존하지 않는가?
3) 네이비(#0052CC)가 오직 클릭 가능/링크/활성 상태에만 쓰였는가 — 장식으로 새지 않았는가?
4) 정보 밀도가 충분한가(여백 낭비 없이) 그러면서도 라벨·뱃지·zebra로 스캔 가능한가? — 밀도 vs 가독성 균형 전수 검사
```
