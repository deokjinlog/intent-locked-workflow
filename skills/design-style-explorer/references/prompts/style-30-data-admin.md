# STYLE 30 — 데이터 리치 라이트 (매력 축: 라이트 밀도)

매력 가설: 한 화면에 최대한 많은 진실을 담되, 밝고 정돈돼서 답답하지 않다. 매력은 여백이 아니라 밀도의 질서 — 에어테이블·노션DB·구글시트를 매일 쓰는 손이 이미 아는 문법이라 학습 비용 0.
고도화 = 그리드라인의 절제된 위계와 필터 칩·정렬 화살표가 살아 있는 실무 감각. 마스터 프롬프트 뒤에 붙여 사용한다.

```
[스타일 지시 — 데이터 리치 라이트 v2]

■ 무드
에어테이블·노션 데이터베이스·구글시트 admin 콘솔. 흰 종이 위에 촘촘한 격자,
칸마다 값이 꽉 찬 실무 도구의 세계. 화려함이 아니라 조밀함이 신뢰를 만든다.
스크롤 한 번에 40행이 보이고, 헤더는 sticky로 늘 붙어 있으며, 모든 열은 정렬 가능하다.
장식은 최소, 정보는 최대. "예쁜 대시보드"가 아니라 "일하는 콘솔".

■ 팔레트 — 오프화이트 + 뉴트럴 그리드 + 상태색
배경 #FFFFFF / 캔버스 #F9FAFB / 헤더·짝수행 zebra #F3F4F6 / hover행 #EFF6FF
그리드라인 #E5E7EB (기본) · 강조 구분선 #D1D5DB · 셀 텍스트 #111827 · 보조텍스트 #6B7280
액션 primary #2563EB / on-primary #FFFFFF / primary-soft #DBEAFE
상태 칩 — 성공 #059669·bg #D1FAE5 / 경고 #D97706·bg #FEF3C7 / 문제 #DC2626·bg #FEE2E2 / 중립 #4B5563·bg #E5E7EB / 진행 #7C3AED·bg #EDE9FE
필터 칩 활성 테두리 #2563EB·bg #EFF6FF, 비활성 테두리 #E5E7EB·텍스트 #6B7280.
배경과 zebra의 미세한 톤 차이(#FFFFFF↔#F3F4F6)가 행을 구분하는 유일한 장치 — 보더는 얇게.

■ 타이포 — 산세리프 + tabular 촘촘
본문 한글 Pretendard 가정. 라벨·숫자·헤더는 Inter (구글폰트) 명시 로드.
숫자 셀은 반드시 font-feature-settings: "tnum" 1 (자릿수 정렬용 tabular figures).
스케일 조밀 — 페이지 타이틀 20/28 600 · 섹션 15/20 600 · 테이블 헤더 12/16 600 대문자 letter-spacing 0.04em ·
셀 본문 13/18 400 · 보조 라벨 11/14 500 · KPI 숫자 24/28 700 tnum.
행 높이 32~36px로 압축, 셀 좌우 패딩 10~12px. 줄바꿈 없이 한 줄, 넘치면 ellipsis + title 툴팁.

■ 형태
radius 작게 — 카드·패널 8px, 칩·버튼·인풋 6px, 아바타 원형. 절대 큰 곡률 금지.
그림자 거의 없음 — 카드는 1px 보더(#E5E7EB)로만 구분, 떠 있는 요소(드롭다운·메뉴)만 0 4px 12px rgba(17,24,39,.08).
사이드바 좌측 고정 220px(접으면 56px 아이콘 레일), 상단바 52px sticky.
테이블은 셀 사이 1px 그리드라인 전면 표시(#E5E7EB) — 열·행 모두. 첫 열·헤더 sticky 고정.
정보 밀도 우선: KPI는 큰 카드 대신 한 줄 스탯 바(높이 72px)에 4~6개를 등간격으로.

■ 히어로 연출 — 콘솔이 채워지는 순서 (로드 시퀀스)
0.0s 사이드바 + 상단바가 즉시 스냅인(그림자 없이, 120ms 페이드) — 크롬은 연출 대상 아님
0.1s 스탯 바의 KPI 숫자가 0→목표값으로 count-up (600ms ease-out, tnum이라 자릿수 안 흔들림)
0.3s 필터 칩 줄이 좌→우 20ms씩 순차 페이드인 (각 120ms)
0.4s 테이블 그리드라인이 먼저 그려지고(150ms), 그 위로 행이 상단부터 15ms 간격 페이드 (각 90ms)
0.7s 우상단 미니 차트(스파크라인/막대)가 baseline에서 채워짐 (360ms) — 조용히, 크게 튀지 않게
전체 800ms 내 완료. 큰 히어로·큰 모션 없음 — "이미 다 로드돼 있던 것처럼" 담백하게.

■ 마이크로 인터랙션 (이 스타일의 실체)
1) 행 hover: 전체 행이 #EFF6FF로 하이라이트 + 좌측 3px primary 인디케이터 바 등장 (80ms). 우측 행 액션(⋯·편집·삭제)이 페이드로 나타남.
2) 열 정렬: 헤더 클릭 시 ▲/▼ 화살표 토글 + 재정렬되는 행들이 120ms FLIP 트랜지션으로 자리 이동. 정렬 중인 열 헤더 텍스트 #2563EB.
3) 필터 칩: 클릭 시 테두리·bg가 활성색으로 채워짐(100ms) + 칩에 값 개수 배지(예: 상태 3). ✕로 개별 해제, "필터 2개 · 초기화" 요약 칩 동반.
4) 셀 인라인 편집: 셀 더블클릭 시 그 자리에서 인풋으로 전환(보더 primary 2px), Enter 저장 시 셀이 성공색으로 200ms 플래시 후 원복.
5) 셀렉트 체크: 첫 열 체크박스로 다중 선택 시 상단에 "N개 선택 · 일괄 작업" 액션 바가 위에서 슬라이드 다운(160ms).

■ 시그니처
밀도 높은 라이트 그리드(#E5E7EB 라인이 전면에 깔린 sticky 헤더 테이블) + 그 위 필터 칩 줄.
이 둘 — 촘촘한 흰 격자와 상단 필터 칩 바 — 이 화면을 즉시 "데이터베이스 admin"으로 읽히게 하는 각인.

■ 접근성·안전장치
- prefers-reduced-motion: count-up·FLIP 재정렬·행 stagger 제거 (정렬은 즉시 재배치, 값은 최종값 즉시 표시)
- JS 실패 폴백: 정렬·필터 없이도 테이블은 서버 정렬 상태로 완전히 읽힘, 필터 칩은 정적 라벨로 표시, 인라인 편집은 별도 편집 페이지 링크로 폴백
- 상태는 색만으로 구분 금지 — 칩에 반드시 텍스트 라벨(성공/경고/문제) 병기, 정렬 방향은 화살표 아이콘 명시
- focus-visible: 셀·칩·행 액션에 primary 2px 아웃라인 + offset 1px, 키보드 화살표로 셀 이동 가능
- 대비: 보조텍스트 #6B7280도 13px에서 4.5:1 확보, zebra 위 텍스트 대비 재검증
- 밀집해도 클릭 타겟 최소 높이 확보: 행 액션 아이콘 히트영역 28×28 이상, 칩 높이 24px 이상

■ 금지 (정체성의 절반)
다크·야간 배경(그건 STYLE 20/04 영역 — 여기선 흰 배경 절대 고수), 큰 히어로 비주얼·풀블리드 이미지,
성긴 여백·큰 카드·넉넉한 line-height(밀도 포기 = 정체성 상실), 네온 글로우·발광·backdrop-filter,
그림자로 층 쌓기(elevation 위계 — 그건 머티리얼), 큰 radius(12px 초과)·pill 남발,
컬러 원색 배경 대량 채색(그리드는 뉴트럴, 색은 상태 칩에만), 세리프 본문,
숫자에 비례폭 폰트(tnum 미적용으로 자릿수 어긋남), 행 높이 44px 초과(밀도 저하).

■ 자체 크리틱 루프
1) 첫 스크롤 한 화면에 데이터 행이 충분히(20행+) 보이는가? — 여백·행높이가 밀도를 죽이지 않았는지 검사
2) 모든 숫자 셀에 tnum이 적용돼 자릿수가 세로로 정렬되는가? — 비례폭 폰트 잔재 색출
3) 그리드라인·상태 칩·필터 칩 3종이 다 살아 있고, 상태가 색+텍스트로 이중 표기됐는가?
4) 다크·큰카드·발광·큰radius 등 밀도/라이트 정체성을 깨는 이탈이 섞이지 않았는가? — 이탈 요소 색출
```
