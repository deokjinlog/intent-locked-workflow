# STYLE 03 — 트렌디 SaaS (매력 축: 생동감)

매력 가설: "살아있네?" — 지금 이 순간의 데이터라는 감각.
고도화 = 로드 연출 + 숨쉬는 디테일. 마스터 프롬프트 뒤에 붙여 사용한다.

```
[스타일 지시 — 트렌디 SaaS v2]

■ 무드
2025년 실리콘밸리 스타트업. Linear/Vercel 계열의 가볍고 낙관적인 감성.
"예쁜 정적 화면"이 아니라 "살아있는 제품"처럼 느껴져야 한다.

■ 팔레트
배경 #FBFAFF / 바이올렛 #7C5CFF(주) / 핑크 #FF8FC0 / 민트 #5EE6C8 /
잉크 #17143A / 보조 #7B7897
그라디언트는 바이올렛→핑크 한 방향만. 무지개 그라디언트 금지.

■ 타이포 — 위계를 극단화할 것
- KPI 수치와 캡션의 크기비 4배 이상
- KPI 수치: 38px 이상, 굵기 800, 자간 -0.03em, tabular-nums
- 레이블·캡션: 11~12px, 보조색으로 물러나게
- 그라디언트 텍스트는 제품명 단 1곳에만

■ 깊이 시스템 (3층 구조를 명확히)
L0 배경: #FBFAFF + 블러 90px 파스텔 블롭 2개 (우상단 바이올렛, 좌하단 민트)
L1 표면: 흰 카드, radius 24~28px, 그림자 0 10px 30px rgba(23,20,58,.06)
L2 부유: hover된 카드만 -4px 떠오르며 그림자가 컬러 틴트로 깊어짐
       (0 20px 40px rgba(124,92,255,.15)) — 깊이 변화에는 반드시 색이 동반된다

■ 히어로 연출 (페이지 로드 시퀀스 — 총 1.2초 이내)
1) 0.0s: 헤더가 먼저 자리잡는다
2) 0.1s~: KPI 카드 4장이 60ms 간격 스태거로 아래에서 떠오른다 (translateY 12px→0 + fade)
3) 동시에 KPI 숫자가 0에서 목표값까지 카운트업 (JS, 800ms, ease-out)
4) 0.4s~: 차트 막대가 왼쪽부터 40ms 간격으로 차오른다
5) 마지막: 피크 막대만 차오른 뒤 살짝 바운스하며 그라디언트로 점등

■ 마이크로 인터랙션 (최소 4개, 각 150~250ms)
1) KPI 카드 hover: L2 부유
2) 차트 막대 hover: 그라디언트 점등 + 수치 라벨 볼드
3) "진행" 상태 태그와 헤더의 실시간 칩: 점(●)이 2초 주기로 숨쉬는 펄스
4) 테이블 행 hover: 연보라로 물들며 식별자가 진해짐
과한 효과 금지 — 모든 모션은 "반응했다"를 알리는 수준에서 멈춘다.

■ 시그니처
살아있는 숫자 — 카운트업 + 숨쉬는 실시간 인디케이터의 조합.

■ 접근성·안전장치
- prefers-reduced-motion: 전 애니메이션 제거, 최종 상태 즉시 렌더
- 카운트업 실패 시에도 최종 숫자가 HTML에 존재 (JS는 연출만 담당)
- hover 효과는 focus-visible에도 동일 적용

■ 금지
직각, 격자 보더 테이블, 다크 배경, 모노스페이스, 무지개 그라디언트,
0.3초 초과 개별 트랜지션, 무한 회전/이동 애니메이션

■ 자체 크리틱 루프
1) 로드 후 3초간 시선이 머무는 곳이 있는가? → 없으면 히어로 타이밍 재조정
2) 가장 밋밋한 곳 1곳은? → 디테일 1개 추가 (전체 균형 유지)
3) 가장 과한 곳 1곳은? → 빼라. 액세서리 하나를 덜어낸다
4) 팔레트·시퀀스 타이밍이 지시와 일치하는가?
```
