# STYLE 04 — 고밀도 터미널 (매력 축: 유능감)

매력 가설: 전문가가 된 기분. 밀도는 부담이 아니라 권능이다.
고도화 = 실시간감 + 밀도의 질서. 마스터 프롬프트 뒤에 붙여 사용한다.

```
[스타일 지시 — 고밀도 터미널 v2]

■ 무드
Bloomberg 단말기. 픽셀당 정보량 극대화 — 단, 무질서는 금지.
화려함 대신 "이 화면을 다루는 나는 전문가"라는 감각.

■ 팔레트
배경 #0C0E12 / 패널 #111419 / 앰버 #F5A623(강조) / 그린 #3DDC84 / 레드 #FF5252 /
블루 #4DA3FF / 텍스트 #D6DAE0 / 보조 #6B7482 — 유채색 5색 초과 금지

■ 타이포
IBM Plex Mono 11~12px 중심, 한글만 Pretendard, 레이블은 대문자+자간(.08em 이상).
큰 폰트 금지 — 티커 수치도 22px 이하.

■ 형태·질감
1px 보더 3열 분할(좌: 요약+로그 / 중: 차트 / 우: 피드), radius 0, 여백 최소,
도트 구분선, 수평 스트라이프 바 차트, 최하단 F1~F5 펑션키 바.

■ 히어로 연출 — 즉시성
화려한 오프닝 대신 로드 즉시 완성 화면. 단, 이벤트 로그의 최신 1줄만
타이핑되듯 나타나며 "방금까지 돌아가고 있었음"을 암시한다.

■ 앰비언트 — 실시간감 (이 스타일 고도화의 핵심)
1) 이벤트 로그에 8~15초 간격으로 새 줄이 위로 밀며 추가된다
   (JS: 도메인에 맞는 더미 로그 5줄을 순환, 로그는 최대 7줄 유지)
2) 상태바 시계가 실제로 초 단위로 흐른다
3) LIVE 도트가 2초 주기로 점멸한다

■ 마이크로 인터랙션
1) 테이블 행 hover: 배경 미세 변화(#151A22)
2) 로그 항목 hover: 관련된 피드 행이 함께 하이라이트 (데이터 연결감)
3) 패널 헤더에 [1][2][3] 인덱스 — 키보드 문법의 암시

■ 시그니처
흐르는 이벤트 로그 — 시스템이 지금 일하고 있다는 증거.

■ 접근성·안전장치
- prefers-reduced-motion: 로그 추가·점멸 정지, 정적 로그 5줄 표시
- 로그 JS 실패 시에도 초기 로그 5줄이 HTML에 존재

■ 금지
여백 많은 레이아웃, 큰 폰트, 라운드, 파스텔, 장식 그림자, 스태거 등장 연출,
이모지, 유채색 6색 이상

■ 자체 크리틱 루프
1) 정보가 많아도 시선 동선이 좌→우, 위→아래로 읽히는가?
2) 유채색이 5색을 넘는가? → 넘으면 실패, 통합하라
3) 로그가 멈춰 보이는가? → 주기·애니메이션 점검
4) "실수로" 넓은 여백이 있는가? → 채우거나 좁혀라
```
