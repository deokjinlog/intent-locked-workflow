# STYLE 04 — 고밀도 터미널 (매력 축: 유능감)

매력 가설: 전문가가 된 기분. 밀도는 부담이 아니라 권능이다.
고도화 = 실시간감 + 밀도의 질서. "지금 이 콘솔 앞에 앉은 나는 시스템을 직접 조종한다."
마스터 프롬프트 뒤에 붙여 사용한다.

```
[스타일 지시 — 고밀도 터미널 v2]

■ 무드
개발자 콘솔 / tmux 세션 / SSH 로 접속한 프로덕션 서버의 tail -f 화면.
Bloomberg 단말기의 밀도 + htop·journalctl·psql 프롬프트의 텍스트 질감.
픽셀당 정보량 극대화 — 단, 무질서는 금지.
화려한 위젯이 아니라 "명령을 치면 즉시 반응하는 텍스트 인터페이스"라는 감각.
모든 것이 문자로 표현된다. 그래픽 위젯이 텍스트를 대체하는 순간 이 스타일은 죽는다.

■ 팔레트
배경 #0A0C10 / 패널 #0E1116 / 프롬프트존 #0C0F14 /
앰버 #F5A623(강조·커서) / 그린 #3DDC84(정상·성공 로그) / 레드 #FF5252(에러 로그) /
블루 #4DA3FF(경로·링크) / 텍스트 #C8CDD4 / 딤 #5B6472(주석·타임스탬프) —
유채색 5색 초과 금지. 색은 "장식"이 아니라 "로그 레벨"이다
(INFO=딤 / OK=그린 / WARN=앰버 / ERROR=레드 — 의미 없는 색칠 금지).

■ 타이포
전면 모노스페이스 — IBM Plex Mono / JetBrains Mono 계열 12~13px 이 화면의 99%.
한글 본문만 Pretendard 로 폴백하되 숫자·기호·경로·명령어는 반드시 모노.
레이블은 대문자+자간(.08em 이상). 행간 1.35~1.45 로 촘촘하게.
큰 폰트 절대 금지 — 최대 강조 수치도 20px 이하, 헤딩도 굵기·대문자로만 위계를 준다.
탭·컬럼은 문자폭 기준으로 정렬(모노 특유의 세로줄 정렬감을 적극 활용).

■ 형태·질감
1px 보더 다분할 그리드(좌: 상태 요약+메트릭 라인 / 중: 흐르는 로그 스트림 / 우: 큐·피드),
radius 0(모든 모서리 직각), 여백 최소(패널 패딩 8~12px),
도트/대시 구분선, ASCII 스타일 구획선(───, ═══, │, ├─) 적극 사용,
수치는 텍스트 표(정렬된 컬럼)로 — 원형 게이지·도넛·막대 위젯 금지.
막대가 필요하면 문자 막대만(예 ███████░░░ 72%).
최상단 한 줄 상태바(host·uptime·load·시계), 최하단 F1~F5 펑션키 바.

■ 명령줄·프롬프트 (이 스타일의 정체성)
화면 어딘가에 항상 살아있는 프롬프트 라인 한 줄:
  user@host:~/domain$ █
앰버 블록 커서(█)가 1초 주기로 깜빡인다.
로그 스트림의 각 줄은 `[HH:MM:SS] LEVEL  message` 형식(타임스탬프=딤, 레벨=색, 본문=텍스트).
경로·파일명·엔드포인트는 블루, 밑줄 없이 모노로.

■ 히어로 연출 — 즉시성
화려한 오프닝 대신 로드 즉시 완성 화면(이미 접속돼 돌아가던 세션에 방금 들어온 느낌).
단, 로그 스트림의 최신 1~2줄만 문자가 타이핑되듯 좌→우로 나타나고
프롬프트 커서가 곧바로 깜빡이기 시작 → "방금까지 명령이 오가고 있었음"을 암시.

■ 앰비언트 — 실시간감 (이 스타일 고도화의 핵심)
1) 로그 스트림에 6~12초 간격으로 새 줄이 아래(또는 위)로 밀며 추가된다
   (JS: 도메인에 맞는 더미 로그 6줄을 레벨 섞어 순환, 스트림은 최대 8줄 유지,
    오래된 줄은 딤 처리되며 밀려 사라진다)
2) 상태바 시계가 실제로 초 단위로 흐른다(uptime 카운터도 함께 증가)
3) 프롬프트 커서(█)가 1초 주기로 점멸
4) load·req/s 같은 상단 메트릭 수치가 미세하게 흔들린다(±1~2, 살아있는 계기 느낌)

■ 마이크로 인터랙션
1) 테이블/로그 행 hover: 배경 미세 변화(#141922) + 좌측에 1px 앰버 마커
2) 로그 항목 hover: 같은 요청 ID·경로를 가진 우측 큐 행이 함께 하이라이트(데이터 연결감)
3) 패널 헤더에 [1][2][3] 인덱스 + 펑션키 바 → 키보드로 조종하는 인터페이스라는 암시
4) 클릭 가능한 요소는 hover 시 앰버 언더스코어(_) 프리픽스가 붙는다

■ 시그니처
흐르는 로그 스트림 + 깜빡이는 프롬프트 커서 — 시스템이 지금 이 순간 명령을 처리하고 있다는 증거.

■ 접근성·안전장치
- prefers-reduced-motion: 로그 추가·커서 점멸·수치 흔들림 정지, 정적 로그 6줄 + 정적 커서 표시
- 로그 JS 실패 시에도 초기 로그 6줄과 프롬프트 라인이 HTML 에 존재
- 앰버·레드·그린은 텍스트 레벨 접두어(OK/WARN/ERR)도 함께 표기해 색만으로 의미 전달 X

■ 금지 (클러스터메이트와 확실히 벌리는 핵심)
- 발광·글로우: 네온 텍스트 섀도, 발광 링/할로, box-shadow 로 빛나는 테두리 → 전면 금지
  (이건 01 SF HUD 의 영역. 여기 색은 "빛"이 아니라 "로그 레벨 잉크"다)
- HUD 코너 프레임: L 자 코너 브래킷, 스캔라인 오버레이, 조준선/레티클, 회전 링, 육각 격자 → 금지
  (역시 01 HUD 시그니처 — 텍스트 그리드로 대체)
- 큰 시각 차트 위젯: 도넛·원형 게이지·게이지 바늘·에어리어 차트·스파크라인을 "주역"으로 크게 배치 → 금지
  (이건 20 다크 대시보드의 영역. 여기서 수치는 정렬된 텍스트 표 + 문자 막대로만)
- 넓은 여백·큰 폰트·중앙 정렬 히어로 카드·라운드 코너·파스텔·장식 그림자·
  그라디언트 배경·스태거 등장 연출·이모지·유채색 6색 이상 → 금지
- 프롬프트 라인/커서/타임스탬프 없는 "그냥 어두운 대시보드" → 금지
  (커서와 로그 스트림이 빠지면 20 과 구분 불가 — 반드시 존재)

■ 자체 크리틱 루프
1) 화면의 99% 가 모노스페이스인가? 세리프/산세리프 본문이 새어들지 않았나?
2) 프롬프트 라인 + 깜빡이는 █ 커서가 화면에 실제로 존재하는가? → 없으면 실패
3) 로그가 `[HH:MM:SS] LEVEL msg` 형식으로 흐르고 있는가? 멈춰 보이는가? → 주기·애니메이션 점검
4) 원형/도넛/게이지/큰 차트 위젯이 하나라도 있는가? → 있으면 20 표절, 텍스트 표로 교체
5) 발광·글로우·HUD 코너 프레임이 있는가? → 있으면 01 표절, 제거
6) 유채색이 5색을 넘거나, 색이 로그 레벨 의미 없이 장식으로 쓰였는가? → 통합·정리
7) "실수로" 넓은 여백·큰 폰트·라운드가 있는가? → 채우고 좁히고 각지게
```
