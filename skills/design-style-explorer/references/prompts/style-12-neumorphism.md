# STYLE 12 — 뉴모피즘 소프트 (매력 축: 촉감)

매력 가설: 누르고 싶다. 매력은 시각이 아니라 촉각의 시뮬레이션.
고도화 = "눌림"의 물리학. 색이 아니라 그림자로만 형태가 태어난다. 마스터 프롬프트 뒤에 붙여 사용한다.

```
[스타일 지시 — 뉴모피즘 소프트 v2]

■ 무드
부드러운 실리콘 버튼의 물리 인터페이스. 화면 전체가 하나의 매트한 회색 실리콘 판이고,
그 판을 안에서 밀어올리거나 눌러 파낸 것처럼 요철만으로 UI 가 존재한다.
"미묘한 요철" — 눈에 확 띄는 게 아니라, 손끝으로 쓸면 만져지는 낮은 부조(low-relief).
투명해서 뒤가 비치는 유리도 아니고, 통통 튀는 파스텔 젤리도 아니다. 불투명·저채도·평탄한 회색.

■ 팔레트
배경 #E4E9F2 단일 — 카드도, 패널도, 태그도, 입력부도 전부 같은 #E4E9F2 (뉴모피즘의 절대 전제).
카드에 배경과 다른 색을 칠하는 순간 뉴모피즘은 죽는다. 색 차이가 아니라 그림자 차이로만 면을 나눈다.
텍스트 #3D4A5C / 서브텍스트 #6B7789 / 포인트 #4A7CFF / 문제 #E36868 / 성공 #4FB477
채도는 전 구간 낮게(저채도 monochrome 회색빛). 포인트 블루조차 넓은 면적엔 쓰지 않고 8px 안팎의 점·아웃라인·텍스트로만 소량 투입.
색 대비는 의도적으로 낮게 유지 — 대비는 색이 아니라 그림자가 만든다. 화면을 흑백 변환해도 위계가 살아 있어야 정상.

■ 타이포
Pretendard 600~700 — 흐린 저대비 배경 위 가독성 확보를 위해 굵게. 400 이하 얇은 굵기 금지(묻힌다).
자간은 표준, 크기 계층은 완만하게(24 / 18 / 15 / 13px). 화려한 타이포 애니메이션 없음 — 표면이 주인공이다.

■ 형태 — 요철의 문법 (광원은 항상 좌상단 135°, 전 요소 예외 없음)
볼록(raised): box-shadow: -8px -8px 16px #FFFFFF, 8px 8px 16px #C3CAD6
오목(inset):  box-shadow: inset -8px -8px 16px #FFFFFF, inset 8px 8px 16px #C3CAD6
큰 패널은 -12px/-12px/24px + 12px/12px/24px 로 스케일업, 작은 알약은 -4px/-4px/8px 로 스케일다운 — 요철 깊이도 위계다.
radius 는 16~20px 고정 범위. 20px 를 넘겨 28px+ 의 통통한 chunky radius 로 가지 않는다(그건 클레이 스타일 — 이 판은 얇고 낮은 부조여야 한다).
하이라이트 흰색은 순백 #FFFFFF 를 쓰되 blur 로 충분히 풀어 "회색빛 은은함" 유지 — 흰 테두리처럼 또렷하게 남으면 실패.
역할 배분: KPI 카드·버튼·상태 알약 = 볼록 / 차트 트랙·테이블 영역·입력창·검색바 = 오목
— 볼록은 "만질 수 있는 것", 오목은 "무언가를 담는 홈". 이 요철 대비가 곧 정보 위계다.

■ 히어로 연출 — 표면의 완성도
화려한 등장 없음(그게 이 스타일의 자존심). 로드 시 400ms 페이드인만.
차트 막대는 오목한 트랙 안에서 700ms ease-out 으로 차오른다 (홈 안을 소프트하게 채우는 감각).
배경은 완전 균일한 매트 회색 — 그라디언트 오브·오로라·블러 배경 절대 없음. 표면의 평탄함 자체가 연출이다.

■ 마이크로 인터랙션 — 눌림 (이 스타일의 전부)
1) 카드/버튼 hover: 그림자 거리 -8px→-10px 로 살짝 벌어지며 표면이 2px 떠오름
2) active(클릭·터치): 볼록 그림자가 그대로 inset 으로 반전되며 실제로 오목하게 눌린다 (150ms ease)
   — 이 요철 반전이 시그니처. 모든 인터랙티브 요소에 빠짐없이 적용. 하나라도 누락되면 이 스타일이 아니다.
3) 차트 막대 hover: 포인트 블루 #4A7CFF 점등(면 채색이 아니라 얇은 라인·점광으로)
4) 상태 태그: 작은 볼록 알약 — 문제 상태만 오목 + 붉은 텍스트 #E36868 (파여서 주의를 끄는 홈)
5) 토글/스위치: 노브가 볼록, 트랙이 오목 — 켜짐/꺼짐이 요철로 읽힌다

■ 시그니처
클릭하면 실제로 눌리는 표면 — 배경과 동일 색 위에서 이중 그림자가 emboss↔deboss 로 반전되는 순간.
색이 하나도 안 바뀌었는데 "눌렸다"고 느껴지는 저대비 촉각 착시가 이 스타일의 정체성이다.

■ 접근성·안전장치
- prefers-reduced-motion: 페이드인·차오름 제거 (눌림 반응은 상태 피드백이라 접근성상 유지)
- 낮은 대비 보완: 본문 텍스트는 반드시 #3D4A5C 이상의 진한 색 (배경 #E4E9F2 대비 4.5:1 확보)
- focus-visible: 포인트 블루 2px 아웃라인 (은은한 그림자만으로는 포커스 식별 불가 — 필수 보강)
- 오목/볼록만으로 상태를 전달하지 말 것 — 문제 상태엔 붉은 텍스트/아이콘을 병행(그림자 인지 어려운 사용자 배려)

■ 금지
강한 색 대비, 보더 라인(모든 경계는 그림자로만), 다크 배경, 플랫 카드,
배경과 다른 색의 카드/패널/태그(같은 #E4E9F2 강제 — 위반 시 뉴모피즘 소멸),
하드 섀도, 순백 하이라이트가 또렷하게 남는 것(반드시 blur 로 풀어 회색빛 유지), 광원 방향 불일치,
— 클러스터메이트(06 글래스) 오염 금지: 투명도(opacity<1)·backdrop-blur·frosted glass·뒤가 비치는 반투명 패널·유리 테두리 하이라이트 절대 금지. 이 판은 불투명 실리콘이다.
— 클러스터메이트(15 클레이) 오염 금지: 강한 파스텔 채도·통통한 젤리 볼륨·28px 이상 chunky radius·두껍고 부풀어 오른 3D 립(lip)·다색 파스텔 카드 절대 금지. 이 판은 얇고 낮은 저채도 부조다.
— 그라디언트 배경/오로라/컬러 오브·네온·글로우 등 발광 효과(요철은 반사광이지 발광이 아니다).

■ 자체 크리틱 루프
1) 광원이 전 요소에서 좌상단 135°로 일관되는가? → 그림자 방향 전수 검사
2) 볼록·오목 구분이 정보 위계와 일치하는가? (만질 것=볼록 / 담는 홈=오목)
3) 대비 부족으로 텍스트가 묻힌 곳(배경 대비 4.5:1 미달)은? → 진한 #3D4A5C 이상으로 보정
4) 눌림 반응(요철 반전)이 없는 인터랙티브 요소가 있는가? → 전수 추가
5) 배경과 다른 색으로 칠해진 면이 하나라도 있는가? → #E4E9F2 로 통일 (글래스/클레이 오염 검출)
6) 투명·blur·파스텔 채도·chunky radius 가 섞여 들어왔는가? → 클러스터메이트 오염, 즉시 제거
7) 화면을 흑백으로 변환했을 때 위계가 그대로 읽히는가? → 색 의존 위계는 실패
```
