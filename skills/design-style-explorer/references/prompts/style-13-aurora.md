# STYLE 13 — 오로라 그라디언트 (매력 축: 광채/미래감)

매력 가설: 화면이 스스로 빛나는 것 같다. 매력은 색이 아니라 어둠 속에서 번지는 광원.
고도화 = "은은함"의 절제 — 오로라는 크게 번지되 결코 소리치지 않는다. 마스터 프롬프트 뒤에 붙여 사용한다.

```
[스타일 지시 — 오로라 그라디언트 v2]

■ 무드
딥 네이비 어둠 위로 오로라가 느리게 흐르는 모던 스타트업 콘솔.
Linear·Stripe·Raycast 바이브 — 미래적이지만 차분하고, 광채는 배경 저 뒤에서 새어 나온다.
빛은 이벤트가 아니라 상시 상태다. 늘 은은히, 늘 움직인다.

■ 팔레트
배경 베이스 #0A0A12 (거의 검정에 가까운 딥 네이비) — 하드 블랙 금지, 미세한 파랑 기운 유지 /
표면(카드) #12121C~#16161F 반투명 (배경 위 rgba(255,255,255,0.03) 오버레이) /
텍스트 주 #E8E9F2 / 텍스트 보조 #8A8DA6 /
오로라 글로우 3색 — 보라 #7C5CFF / 시안 #2FD8E8 / 핑크 #FF5CC8
포인트(액션·KPI 강조) 보라→시안 그라디언트 / 문제·경고 #FF6B8A (원색 아닌 채도 낮춘 핑크레드)
규칙: 오로라 3색은 배경 글로우와 그라디언트 보더에만. 텍스트·아이콘 본체는 흰빛 계열 유지.

■ 타이포
영문·숫자 Inter (Google Fonts) 600~700 tight — letter-spacing -0.02em, KPI 수치는 tabular-nums.
한글 본문 Pretendard 400~600. 제목은 Inter/Pretendard 혼용 tight, 본문 대비 확실히.
수치는 크고 밝게(#E8E9F2), 라벨은 작고 흐리게(#8A8DA6) — 광채 위계는 밝기로.

■ 형태
radius 16~20px. 카드는 반투명 표면 + backdrop-blur(12px) — 배경 오로라가 카드 뒤로 비친다.
시그니처 = 그라디언트 보더: 1px, 보라→시안→핑크 conic/linear, 평소 opacity 0.25.
보더 안쪽에 미세한 inner glow(box-shadow: inset 0 1px 0 rgba(255,255,255,0.06)).
구분선은 rgba(255,255,255,0.08) 헤어라인. 두꺼운 검정 보더 절대 금지.

■ 히어로 연출 — 오로라의 점등 (로드 시퀀스)
0.0s: 순수 #0A0A12 어둠만.
0.0~1.2s: 배경 오로라 메시(보라·시안·핑크 3개의 큰 radial-gradient blob)가 blur(80px)로
         opacity 0 → 0.5 서서히 번진다 (ease-out, 각 blob 0.15s 스태거).
0.3~0.9s: 헤더·KPI 카드가 아래 12px + opacity 0 → 1 로 떠오른다 (60ms 스태거).
0.6~1.4s: 카드 그라디언트 보더가 opacity 0 → 0.25 로 점등.
1.0s~: 배경 오로라 blob이 무한 루프로 아주 느리게 이동/회전 (한 사이클 20~30s, ease-in-out).
차트 데이터는 800ms 동안 왼쪽→오른쪽으로 그려지며(line stroke-dashoffset), 라인 아래 그라디언트 fill 서서히 차오름.

■ 마이크로 인터랙션
1) 카드 hover: 그라디언트 보더 opacity 0.25 → 0.7 로 밝아지고, 카드 뒤 오로라 글로우가
   해당 카드 방향으로 살짝 몰린다 (box-shadow 확산, 250ms ease).
2) 마우스 이동: 배경 오로라 blob이 커서 위치에 아주 약하게 이끌린다 (parallax, 최대 20px,
   0.6s 지연 추종 — 즉각 반응 금지, 어디까지나 은은하게).
3) 스크롤: 배경 오로라가 스크롤 진행도에 따라 색상 hue를 천천히 이동(보라→시안 쪽으로),
   전경 콘텐츠와 다른 속도로 흘러 깊이감(퍼럴랙스) 형성.
4) 버튼 hover: 보라→시안 그라디언트가 좌우로 흐르고(background-position 이동 400ms) 미세 글로우 점등.
5) 테이블 행 hover: 행 좌측에 1px 시안 글로우 바 + 배경 rgba(255,255,255,0.03) 살짝 밝아짐.

■ 시그니처
스크롤·마우스에 은은히 흐르는 배경 오로라 + 카드 그라디언트 보더(글로우 엣지).
정지 화면에서도 배경은 미세하게 살아 움직인다 — "빛나는 어둠"이 정체성.

■ 접근성·안전장치
- prefers-reduced-motion: 오로라 이동/회전·마우스 추종·hue 스크롤 시프트 전부 정지
  (오로라 blob은 고정 위치로 표시 — 광채는 남기되 움직임만 제거), 차트/카드 등장은 즉시 표시.
- JS 실패 폴백: 마우스 추종·스크롤 hue 시프트는 JS 의존 → 실패 시 배경은 CSS 정적 그라디언트로 렌더,
  카드 그라디언트 보더는 순수 CSS라 항상 표시. 콘텐츠는 JS 없이 100% 판독 가능.
- 대비 확보: 본문 텍스트는 반드시 #E8E9F2(딥 배경 대비 4.5:1 이상). 흐린 보조색 #8A8DA6는
  라벨·메타에만, 본문 금지. 오로라 색 위에 텍스트 올리지 않는다(가독성 붕괴).
- focus-visible: 시안 #2FD8E8 2px 아웃라인 + 미세 글로우 (그라디언트 보더만으로 포커스 식별 불가).

■ 금지
- 하드 플랫 색면 (오로라 없는 단색 채움), 원색 채도 폭발(#FF00FF·#00FFFF 류 형광),
- 두꺼운 검정 보더, 밝은/흰 배경, 하드 드롭섀도(검정 40%+ 각진 그림자),
- 오로라 색을 텍스트·아이콘 본체에 사용, 오로라를 여러 개 강한 대비로 겹쳐 카니발처럼 만들기,
- 즉각 반응하는 마우스 추종(은은함 상실), blur 없는 날것의 그라디언트 blob(경계선 노출),
- 카드를 불투명 단색으로 채워 배경 오로라를 가리기, 회의록·사무 톤의 무광 그레이.

■ 자체 크리틱 루프
1) 배경이 정지 상태에서도 "빛나는 어둠"으로 살아 있는가? 아니면 그냥 어두운 화면인가? → 오로라 강도·blur 재조정
2) 오로라 3색이 텍스트/아이콘으로 새어 원색 폭발이 되지 않았는가? → 색 사용처 전수 검사(글로우·보더에만)
3) 모든 인터랙션이 "은은함" 선을 지키는가? 즉각·과격한 반응은 없는가? → 지연·감쇠 값 상향
4) 대비 미달로 묻힌 텍스트(4.5:1 미만)나 오로라 위에 올라탄 글자는 없는가? → 밝기·배치 보정
```
