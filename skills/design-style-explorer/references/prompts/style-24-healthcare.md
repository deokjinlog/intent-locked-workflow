# STYLE 24 — 헬스케어 웰니스 (매력 축: 안심/케어)

매력 가설: 숫자가 나빠도 화면이 나를 다그치지 않는다. 매력은 자극이 아니라 진정 — 넉넉한 여백과 부드러운 곡선이 "괜찮다, 천천히 보자"고 말해서 오래 머물러도 피로하지 않다.
고도화 = 여백의 리듬과 라운드의 일관성, 그리고 위험을 알리되 겁주지 않는 절제된 신호. 마스터 프롬프트 뒤에 붙여 사용한다.

```
[스타일 지시 — 헬스케어 웰니스 v2]

■ 무드
병원 로비와 건강 앱 사이의 감각. 밝고 청결한 화이트 공간에 하늘과 잎사귀 색이 옅게 스민다.
모든 것이 둥글고 부드럽고, 서두르지 않는다. 신뢰·돌봄·안정 — 데이터를 진단이 아니라 안내로 전한다.
사용자를 환자가 아니라 손님처럼 대하는 화면. 조용한 확신, 낮은 목소리.

■ 팔레트 — 화이트 + calm 스카이블루/세이지 + 웜 뉴트럴
배경 #FBFCFD / surface #FFFFFF / 카드 보더 #E8EDF1
Primary(calm 스카이블루) #4A90D9 / primary-hover #3A7BC0 / primary-soft #E3F0FB / on-primary #FFFFFF
Secondary(세이지그린) #7FB069 / secondary-soft #ECF4E7
웜 뉴트럴: 텍스트 강 #2E3A46 / 텍스트 중 #5A6B7A / 텍스트 약 #8A9AA8 / 웜 배경 틴트 #FAF8F5
상태색(채도 낮춘 파스텔): 정상 #7FB069 / 주의 #E8B04B / 위험 #E27D7D / 정보 #4A90D9
위험조차 파스텔로 — 붉은 경보가 아니라 차분한 신호. surface는 순백, 배경은 미세하게 웜.

■ 타이포 — 부드러운 산세리프
본문 한글 Pretendard 가정. 영문·숫자는 Nunito Sans (구글폰트) 명시 로드 — 둥근 끝단이 무드와 일치.
숫자 강조가 필요한 KPI는 Nunito (구글폰트) 700, 살짝 더 둥근 인상.
스케일: Display 32/42 700 · Title 22/30 600 · Section 18/26 600 ·
Body 15/24 400 · Small 13/20 400 · Label 12/18 600 (대문자 금지, letter-spacing 0).
자간은 0~0.2px로 여유롭게, 행간은 넉넉히(본문 1.6) — 촘촘함이 주는 압박을 없앤다.

■ 형태
radius 큼 — 카드 20px, 버튼·인풋 12px, 칩·뱃지 pill(999px), 아바타 원형, 모달 24px.
보더는 1px #E8EDF1 저대비, 구분선은 색보다 여백으로. 
그림자는 아주 부드럽게: 카드 0 2px 8px rgba(74,144,217,.06), 0 8px 24px rgba(46,58,70,.05) —
파랑 기가 살짝 도는 확산형, 절대 검정 하드섀도 금지.
아이콘은 stroke 1.75px 라운드 캡, 채움 대신 선. 넉넉한 padding(카드 내부 24~28px).

■ 히어로 연출 — 숨을 고르는 등장 (로드 시퀀스, 전체가 부드럽고 느리다)
0.0s 상단바가 opacity 0→1로 조용히 나타남 (400ms ease-out, 이동 없음 — 흔들리지 않는 인상)
0.2s 사이드바 아이콘이 위→아래로 60ms씩 순차 페이드 (각 300ms)
0.45s KPI 카드 4장이 y+12px→0 + 페이드로 왼→오 80ms stagger (각 380ms, 아주 부드러운 ease)
0.9s 라인 차트의 선이 왼쪽부터 900ms에 걸쳐 그려지고(stroke-dashoffset), 면적 틴트가 뒤따라 채워짐
1.3s 테이블 행이 위→아래 50ms 간격 페이드, 상태 뱃지는 각 행 등장 후 pill이 살짝 부풀었다 안착
전 구간 튀거나 튕기는 모션 없음 — 물 위에 떠오르듯.

■ 마이크로 인터랙션 (이 스타일의 실체)
1) 카드·버튼 hover: 그림자가 부드럽게 1.4배 확산 + y-2px 미세 부상 (220ms ease-out), 색 변화 없음
2) primary 버튼 press: primary-soft 링이 버튼 둘레에 250ms 퍼졌다 사라짐(리플 아님, 잔잔한 후광)
3) 토글·체크: track이 secondary-soft→primary로 채워지며 thumb가 300ms에 부드럽게 미끄러짐
4) 상태 뱃지 hover: 툴팁이 아래에서 y+6px→0 페이드로 떠오르며 "정상 범위: 70~110" 같은 안심 문구 표시
5) KPI 숫자: 최초 등장 시 0→목표값으로 800ms 카운트업(ease-out), 이후 값 변경 시에도 부드러운 트윈

■ 시그니처
calm 톤(채도 낮춘 스카이블루·세이지) + 넉넉한 여백(카드 간 24px, 내부 28px) + 부드러운 라운드(20px 카드/pill 뱃지).
이 세 가지가 겹쳐 화면 전체가 "숨 쉴 공간"처럼 읽힌다 — 위험 지표조차 파스텔로 감싸 겁주지 않는 것이 각인.

■ 접근성·안전장치
- prefers-reduced-motion: 카운트업·선 그리기·stagger 제거, 최종 상태만 즉시 표시(부상 hover도 색만 유지)
- 색맹 대응: 상태는 색+아이콘+텍스트 3중 표기(정상 체크 / 주의 느낌표 / 위험 삼각형) — 파스텔은 대비 낮아 색 단독 금지
- 대비: 파스텔 상태색 위 텍스트는 웜 뉴트럴 강(#2E3A46)으로, 본문 4.5:1 확보. primary 텍스트는 화이트 위에서만 사용
- focus-visible: primary 2px 아웃라인 + offset 2px + primary-soft 배경 힌트
- 큰 터치 타겟: 넉넉한 여백 유지하되 인터랙티브 히트영역 최소 44×44px

■ 금지 (정체성의 절반)
다크 배경·다크모드 강제, 강렬한 원색(순빨강 #FF0000·형광)·네온, 고밀도 촘촘 레이아웃(카드 간 여백 <16px),
날카로운 직각(radius 0~4px)·하드섀도(0 offset 검정 그림자), 세리프·모노스페이스 본문,
공격적 붉은 경보 배너, 굵은 1px 순검정 보더, 그라디언트 배경 남발, 대문자 라벨(SHOUTING),
elevation 과시용 짙은 그림자, 애니메이션의 튕김(bounce)·빠른 스냅(150ms 미만).

■ 자체 크리틱 루프
1) 여백이 충분한가? — 카드 간·내부 padding이 규정치 이상인지, 어디도 답답하지 않은지 전수 확인
2) 라운드가 일관된가? — 직각·작은 radius가 섞이지 않았는지, 뱃지는 pill인지 색출
3) 위험 신호가 안심 기조를 지키는가? — 상태색이 파스텔 범위인지, 붉은 경보·형광이 없는지, 색 단독 표기 아닌지
4) calm 톤을 벗어난 채도 높은 색이나 다크·하드섀도·세리프 잔재가 섞이지 않았는가? — 이탈 요소 색출
```
