# STYLE 18 — 스위스 인터내셔널 (매력 축: 격자/타이포 정밀)

매력 가설: 장식이 없어서 신뢰가 간다. 매력은 격자의 수학과 타이포의 정밀함.
고도화 = 눈에 보이는 모듈 그리드 + 단 하나의 빨강 + 왼쪽 기준선을 깨는 초대형 비대칭 헤드라인.
"여백만의 절제"가 아니라 "구조의 절제"다 — 격자가 보이고 타이포가 밀어붙인다. 마스터 프롬프트 뒤에 붙여 사용한다.

```
[스타일 지시 — 스위스 인터내셔널 v2]

■ 무드
International Typographic Style. 스위스 취리히·바젤 학파의 엄격한 질서.
객관성·명료함·환원. 장식은 소음이고, 격자는 진실이다.
포스터가 아니라 계기판이지만 규율은 포스터의 것 — Müller-Brockmann 의 그리드 시스템,
Akzidenz-Grotesk 시절의 냉정함, 빨강 하나로 지면을 지배하던 그 자신감.
"조용한 미니멀"이 아니라 "규율 있는 긴장" — 여백이 아니라 격자와 타이포가 주인공.

■ 팔레트
배경 #FFFFFF 순백 (오프화이트 아님 — 정직한 흰색, 회색빛 도는 미니멀 배경 금지) /
잉크 #111111 (거의 검정, 순흑 #000000 은 대형 헤드라인에만 제한적) /
액센트 #E2231A (스위스 레드 — 화면에 딱 1곳, 이것이 규율의 시험. 이 빨강이 정체성의 핵) /
그리드선 #E4E4E4 (희미한 회색, 모듈 경계를 눈에 보이게 — 숨기지 않는다) /
보조 회색 #6B6B6B (라벨·캡션·축 눈금 전용)
색은 4개면 충분하다. 빨강은 예산제 — 쓸 때마다 하나를 반납한다.
파랑은 이 스타일에 존재하지 않는 색이다 (신뢰는 격자가 만들지 파랑이 만들지 않는다).

■ 타이포
Helvetica Neue 우선, 없으면 'Neue Haas Grotesk Display', 대체로 Inter (Google Fonts) —
전부 grotesque sans. 한글 본문은 Pretendard (동일 grotesque 계열이라 이질감 없음).
- 초대형 헤드라인 Inter 800, letter-spacing -0.03em, line-height 0.9 (지면을 뚫을 듯 타이트하게)
  → 크기 과감하게 (clamp 로 최소 48px, 히어로는 72~120px). 작게 쓰면 스위스가 아니다.
- 섹션 라벨 Inter 500, letter-spacing 0.1em, 대문자, 11px (작고 넓게, 헤드라인과 극단 대비)
- 본문·수치 Inter 400~500, line-height 1.4
- KPI 대형 숫자 Inter 700, tabular-nums 필수 (자릿수 정렬)
flush-left 정렬이 절대 원칙. 모든 텍스트 블록은 왼쪽 기준선에서 시작한다.
헤드라인은 왼쪽 컬럼에 붙어 오른쪽으로 크게 흘러넘치는 비대칭 배치 — 가운데 놓지 않는다.
극단 대비가 핵심: 초대형 헤드라인 하나 옆에 11px 라벨, 그 사이 중간 크기는 최소화.

■ 형태
radius 0 — 예외 없음. 모든 모서리는 직각.
카드는 카드가 아니라 "격자 셀". 배경과 같은 흰색, 1px #E4E4E4 헤어라인으로만 구획.
12컬럼 모듈 그리드 (gutter 24px). baseline 그리드 8px 단위 — 모든 요소가 8의 배수 위에 앉는다.
격자선은 숨기지 않고 드러낸다 — 섹션 경계·컬럼 가이드가 #E4E4E4 헤어라인으로 실제 보인다.
KPI는 4셀 균등 분할이 아니라 비대칭 허용 (예: 5·3·2·2 컬럼 스팬) — 정보량이 폭을 정한다.
비대칭이 규율이다: 대칭 4등분·중앙 정렬 블록은 스위스가 아니라 관공서다.
빨강은 면이 아니라 선·점·1개 숫자로. 채우기보다 지시.

■ 히어로 연출 — 격자의 조립 (초 단위)
화려한 등장 금지. 격자가 "그려지는" 연출.
0.0s: 흰 화면 + 잉크색 초대형 헤드라인만 (flush-left, 왼쪽에 붙어 오른쪽으로 흘러넘침)
0.2s: 세로 컬럼 가이드선이 왼→오른쪽으로 240ms 그어짐 (#E4E4E4, 12컬럼이 눈에 그어진다)
0.5s: KPI 숫자가 각 셀에서 60ms 간격 stagger 로 페이드인 (아래→위 4px)
0.8s: 빨강 액센트 1곳만 마지막에 점등 (색이 규율을 완성하는 순간)
차트 바/라인은 좌측 기준선에서 오른쪽으로 500ms grow. easing 은 ease-out 만 (bounce 금지).

■ 마이크로 인터랙션
1) 행(row) hover: 배경 #FAFAFA 로 8ms 전환 + 좌측에 2px #E2231A 인디케이터 바 슬라이드인
   — 빨강이 "지금 여기"를 가리키는 유일한 순간
2) KPI 셀 hover: 하단 헤어라인이 1px→2px 잉크색으로 굵어짐 (격자가 반응)
3) 정렬 가능한 테이블 헤더 클릭: 라벨 옆 작은 삼각형(▲/▼) 회색→검정, 정렬 방향 즉시 반영
4) 차트 데이터포인트 hover: 얇은 crosshair 가이드선(1px #6B6B6B, 점선 아님 실선) + tabular-nums 툴팁
5) 탭/필터 선택: 활성 항목만 하단 2px #111111 언더라인 (빨강 아님 — 빨강은 데이터 강조 예산)

■ 시그니처
엄격한 모듈 그리드 위 flush-left 초대형 비대칭 타이포 + 빨강 포인트 단 1곳.
헤어라인으로 그어진 12컬럼이 눈에 보이고, 그 위에 헤드라인이 왼쪽 기준선에 붙어
오른쪽으로 흘러넘치며 규율 있게 앉는다. 화면을 덮으면 즉시 "취리히 포스터의 계기판" 이라 읽혀야 한다.
"파랑 표 관공서"(공공)도 아니고 "여백뿐인 무채색"(미니멀)도 아니다 — 빨강·격자·초대형 타이포가 셋 다 보인다.

■ 접근성·안전장치
- prefers-reduced-motion: 격자 조립·stagger·grow 전부 제거, 즉시 최종 상태. hover 색 전환만 유지
- JS 실패 폴백: 정렬·탭·차트 애니메이션 없이도 전체 데이터가 정적으로 완전히 보여야 함 (grid 는 CSS 만으로 성립)
- focus-visible: 2px #E2231A 아웃라인 + 2px offset (직각 유지, 라운드 금지) — 빨강 예산의 유일한 예외 허용
- 대비: 본문 #111111 on #FFFFFF (16:1), 캡션 #6B6B6B (4.6:1 확보) — 회색 라벨이 너무 옅어지지 않게

■ 금지 (정체성의 절반)
- box-shadow 일체 (그림자로 띄우지 않는다 — 위계는 타이포·격자가 만든다)
- gradient (선형·방사형 전부), 반투명 오버레이
- border-radius > 0 (모든 라운드 금지, 알약 버튼·둥근 아바타 포함)
- 장식 아이콘·이모지·일러스트 (아이콘 쓰면 단색 선형 기능 아이콘만, 그것도 최소)
- serif 남발 (Helvetica/Inter grotesque 이외 서체 금지, 강조용 이탤릭 세리프 없음)
- 파랑을 주액센트로 쓰기 (← 02 공공포털의 신뢰-파랑 지분. 이 스타일의 신뢰는 격자가 만든다.
  파랑·초록·남색 등 제3의 색 자체가 금지. 액센트는 오직 스위스 레드 #E2231A 하나)
- 중앙정렬 레이아웃 (← 공공·문서 톤. flush-left 절대 원칙 — 센터정렬은 페이지 최상단 로고 정도만.
  헤드라인·KPI·섹션 제목을 가운데 놓는 순간 스위스가 아니다)
- 표+각주 위주 구성 (← 02 공공의 서식 문서 톤. 표는 격자 셀의 한 종류일 뿐,
  화면 전체가 라벨-값 표·각주 나열로 채워지면 안 됨. 초대형 타이포·비대칭 KPI 스팬이 주도해야 함)
- 여백만으로 절제한 척하기 (← 07 미니멀 모노크롬의 방식. 눈에 보이는 12컬럼 격자선·8px baseline·
  초대형 헤드라인 없이 "흰 여백 + 작은 무채색 텍스트"만 깔면 미니멀과 구분 불가 = 회귀.
  스위스는 여백이 아니라 드러난 격자와 타이포 구조로 절제한다)
- 대칭 4등분 균등 KPI (비대칭 컬럼 스팬이 규율 — 균등 grid 는 관공서 대시보드 톤)
- 빨강 2곳 이상 (액센트 예산 초과 = 규율 붕괴, 하나로 환원)
- 카드에 배경색 채우기 (셀은 흰색, 헤어라인으로만 구획), 컬러 배지 채우기

■ 자체 크리틱 루프
1) 빨강(#E2231A)이 화면에 정확히 1곳(hover·focus 순간 제외)인가? → 2곳 이상이면 규율 붕괴, 하나로 환원
2) 파랑·초록·남색 등 제3의 색이 한 픽셀이라도 있는가? → 있으면 공공포털 톤 오염, 전부 잉크·회색·레드로 환원
3) 모든 요소가 8px baseline·12컬럼 격자에 정렬되었는가? → 어긋난 요소 전수 검사
4) 12컬럼 가이드선·격자가 실제로 눈에 보이는가? → 안 보이면 미니멀과 구분 불가, 헤어라인으로 격자 드러내기
5) 헤드라인이 초대형이고 flush-left 로 왼쪽에 붙어 흘러넘치는가? → 작거나 중앙이면 스위스 실패, 크게·왼쪽으로
6) 화면이 표+각주 나열로만 채워졌는가? → 비대칭 KPI 스팬·대형 타이포로 위계 재구성
7) 그림자·라운드·그라디언트가 한 곳이라도 살아있는가? → 발견 즉시 제거 (금지 목록 위반)
8) 텍스트가 flush-left 인가, 무심코 중앙정렬한 블록은 없는가? → 좌측 기준선 재정렬
```
