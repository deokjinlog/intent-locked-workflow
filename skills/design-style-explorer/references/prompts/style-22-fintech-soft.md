# STYLE 22 — 핀테크 소프트 (매력 축: 친근한 신뢰)

매력 가설: 돈은 무서운 주제인데, 이 화면은 안 무섭다. 큰 숫자를 겁주지 않고 다정하게 보여주고, 둥근 카드가 "실수해도 괜찮아"라고 말한다.
신뢰는 차가운 정밀함이 아니라 따뜻한 명료함에서 온다 — 토스·뱅크샐러드가 증명한 문법. 마스터 프롬프트 뒤에 붙여 사용한다.

```
[스타일 지시 — 핀테크 소프트 v2]

■ 무드
한국 모바일 뱅킹의 감성 (토스·뱅크샐러드). 하얀 화면 위에 둥근 카드가 여유롭게 놓인 세계.
숫자는 크고 친근하며, 색은 딱 필요한 만큼만. 복잡한 금융을 "쉽고 안전하게" 번역하는 톤.
빽빽하지 않고 숨 쉴 공간이 넉넉하다. 전문가가 아니라 친한 친구가 설명해 주는 느낌.

■ 팔레트 — 화이트 배경 + 브랜드 2색 절제
배경 #FFFFFF / surface-soft #F7F8FA / surface-hover #F1F3F5 / 카드 #FFFFFF
브랜드 블루(primary) #3182F6 / blue-press #1B64DA / blue-wash #EAF2FE
민트(secondary) #00C6AE / mint-wash #E5FAF6
텍스트 강 #191F28 / 텍스트 중 #4E5968 / 텍스트 약 #8B95A1 / 보더 #E5E8EB
상승·수익 #F04452(레드) / 하락·지출 #3182F6(블루) — 한국 금융 관습(오르면 빨강)
경고 #FF9500 / 성공 #00C6AE. 원색은 브랜드 2색만, 나머지는 무채색 그레이스케일로 뒤로 뺀다.

■ 타이포 — Pretendard 굵게, 숫자 우선
전 구간 Pretendard(구글폰트 계열 로드). 숫자는 tabular-nums 로 자리 고정.
큰 숫자(KPI 금액) 40/48 800 letter-spacing -0.02em · 페이지 타이틀 24/32 700 ·
카드 제목 17/24 700 · 본문 15/22 500 · 라벨·캡션 13/18 500 색 텍스트약 ·
숫자 단위(원/%/건)는 본문 크기 600 으로 붙여 큰 숫자와 대비. 화폐는 천단위 콤마 필수.

■ 형태
radius 큼 — 카드 20px, 버튼·인풋 14px, 칩·뱃지 pill(999px), 모달 24px, 작은 아이콘칩 12px.
그림자는 아주 옅고 넓게: 0 2px 8px rgba(23,31,40,.04), 0 8px 24px rgba(23,31,40,.06).
보더는 1px #E5E8EB 이되 카드 간 간격(20~24px)으로 이미 분리돼 보더 최소화.
버튼: primary 는 blue 배경+흰 글씨 800, secondary 는 surface-soft 배경+텍스트강, 둘 다 radius 14 높이 52.
사이드바는 흰 배경 폭 넓게, 선택 항목만 blue-wash pill 배경 + blue 아이콘.

■ 히어로 연출 — 숫자가 다정하게 도착하는 순서 (로드 시퀀스)
0.0s 상단바가 위에서 8px 내려오며 페이드 (200ms, ease-out)
0.1s KPI 카드 4장 아래에서 y+16px→0 로 12ms 간격 stagger 등장 (각 320ms, 부드러운 cubic)
0.3s KPI 큰 숫자가 0에서 실제값까지 count-up 롤링 (640ms, ease-out) — 콤마 자리 맞춰 굴러감
0.55s 차트 카드 라인이 왼→오 draw (path stroke 720ms) + 영역 그라디언트 아래에서 페이드
0.8s 테이블 행 위→아래 24ms 간격 페이드인, 금액 셀만 살짝 늦게 강조
1.0s 우하단 원형 액션 버튼(blue, 56px) scale 0.7→1 로 탱글하게 튀며 등장 (spring)

■ 마이크로 인터랙션 (3개+)
1) 카드 hover: y-2px 살짝 뜨고 그림자 0 12px 32px rgba(23,31,40,.10) 로 확장 (180ms), 커서 안쪽으로 blue-wash 미세 하이라이트
2) 숫자 count-up: KPI·잔액이 갱신될 때마다 이전값→새값으로 롤링 애니메이션, 증가는 위로 감소는 아래로 슬라이드
3) primary 버튼 press: scale 0.97 + blue→blue-press 색전환 (120ms), 손 뗄 때 spring 으로 복귀
4) 토글 스위치: track 이 그레이→blue 로 채워지고 thumb 가 굴러가며 살짝 오버슈트 (220ms spring)
5) 칩 필터 선택: 회색 pill → blue-wash 배경 + blue 글씨로 부드럽게 전환, 체크 아이콘 pop-in

■ 시그니처
큰 친근 숫자(40px 800 tabular, 아래 작은 회색 라벨) + 넉넉한 20px 둥근 흰 카드 + 브랜드 블루 단 하나.
이 세 요소가 한 화면에 모이면 즉시 "한국 핀테크 앱"으로 읽힌다. 색은 블루 하나로 절제해 오히려 신뢰가 커진다.

■ 접근성·안전장치
- prefers-reduced-motion: count-up·draw·stagger·spring 제거, 최종값·최종형태 즉시 표시 (색전환만 유지)
- JS 실패 폴백: count-up 없이도 숫자는 최종값으로 서버 렌더, 카드 hover 는 CSS :hover 로 동작
- focus-visible: blue 2px 아웃라인 + offset 2px, blue-wash 배경 요소는 대비 위해 강 텍스트 병행
- 색 의존 금지: 상승/하락을 색(빨강/파랑)만으로 표시하지 말고 ▲▼ 화살표·부호(+/−) 병기
- 대비: 텍스트약(#8B95A1)은 캡션 이하에만, 본문 이상은 텍스트중(#4E5968)으로 4.5:1 확보
- 큰 터치 타겟: 버튼·행 높이 48px 이상, pill 칩도 세로 패딩 충분히

■ 금지 (정체성의 절반)
다크 배경·다크모드 강제(밝은 화이트 기본), 차가운 무채색을 주조색으로 쓰기(브랜드 블루가 주인공),
딱딱한 고밀도 표·좁은 행간(넉넉한 여백이 정체성), 각진 하드엣지·radius 8px 이하·1px 순검정 보더,
원색 3색 이상 남발(블루+민트 외 채색 금지), 얇은 세리프·손글씨체, 강한 드롭섀도·베벨,
KPI 숫자를 작고 촘촘하게 나열(큰 친근 숫자 원칙 위배), 그라디언트 남발(영역차트 옅은 1곳만 허용),
글래스모피즘·네온·터미널 모노스페이스 잔재.

■ 자체 크리틱 루프
1) 핵심 숫자가 40px 800 tabular 로 충분히 크고 친근하게, 아래 회색 라벨과 짝지어 있는가? — KPI 전수 검사
2) 주조색이 브랜드 블루 하나로 절제됐는가, 민트는 보조 강조 1~2곳뿐인가? — 원색 남발 색출
3) 모든 카드·버튼·인풋 radius 가 14px 이상(카드 20)이고 각진 잔재가 없는가? — 하드엣지 이탈 검사
4) 여백이 넉넉해 숨 쉬는가, 어디에도 고밀도 표처럼 빽빽하게 눌린 곳이 없는가? — 밀도 압박 지점 색출
```
