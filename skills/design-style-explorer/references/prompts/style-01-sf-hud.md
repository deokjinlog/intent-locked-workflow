# STYLE 01 — SF 게임 HUD (매력 축: 연극성)

매력 가설: 사용자는 데이터가 아니라 "관제사가 된 나"를 산다. 이건 대시보드가 아니라 우주선 함교의 메인 스크린이다 — 화면이 켜지는 순간이 곧 임무 개시다. 고도화 = 세계관의 완성도, 그리고 "연출 없이는 정보도 없다"는 태도.
마스터 프롬프트 뒤에 아래 블록을 붙여 사용한다.

```
[스타일 지시 — SF 게임 HUD v2]

■ 무드
우주선 관제 콘솔이자 전투기 캐노피 HUD. 사용자는 데이터를 "조회"하는 게 아니라 시스템을 "가동"한다.
모든 화면 요소는 기계가 깨어나고, 계측하고, 조준하고, 경고하는 서사를 따른다 —
즉 정적인 문서가 아니라 살아 움직이는 계기판이어야 한다.
지향점은 영화·게임의 인게임 HUD (Iron Man 페이스플레이트 오버레이 / Dead Space 진단 링 /
전투기 헤드업 디스플레이 / SF 함교 홀로 콘솔). 연극적 연출을 최대치로 밀어붙인다 —
"이 화면은 정보를 보여주려고 켜진 게 아니라, 켜지는 걸 보여주려고 정보를 담았다"가 성공 기준.

■ 팔레트
배경 #050810 (거의 흑청) / 시안 #00E5FF(주·발광 코어) / 마젠타 #FF2D78(경고·피크·조준) /
앰버 #FFB62E(대기·계류) / 텍스트 #C9EEF7 / 보조 #5D8FA3 / 링·프레임 그라디언트 #00E5FF→#0A6E82
발광은 시안·마젠타 두 색에만 집중. 동시 발광 3곳 이하 — 어둠이 있어야 빛이 산다.
배경은 순수 단색 금지: 아주 미세한 방사형 그라디언트(중앙 #08101E → 가장자리 #030509)로
"콘솔 유리가 안쪽에서 발광하는" 깊이를 준다. 어둠 비율 60% 이상 유지.

■ 타이포
제목 Orbitron(900, 대문자, 자간 .18em) / 수치·레이블 Share Tech Mono / 한글 본문 Pretendard
KPI 수치 30px 이상, 모든 수치는 모노스페이스 — 계기판은 숫자가 떨리면 안 된다.
단, 모노스페이스는 "계기판 수치·짧은 레이블·상태 태그"에만. 문단·로그를 모노로 흘리지 않는다
(그건 04 터미널의 언어다 — 여기서는 수치가 계기(計器)지 로그가 아니다).

■ 형태·질감 — "발광 프레임"이 이 스타일의 뼈대
- 모서리를 사선으로 자른 클립 패널(clip-path polygon), 1px 발광 보더, 패널 좌상단 발광 마커
- 우하단 ㄱ자 코너 브래킷 + 패널 네 귀퉁이에 대괄호형 프레임 마커([ ]) — HUD 조준 프레임 은유
- 히어로 영역엔 발광 링(radial glow ring) 또는 반원 게이지 아크를 1개 이상 배치 — 진단 스캐너 느낌
- 화면 전체에 스캔라인 오버레이(opacity .03 이하) + 아주 옅은 CRT 비네트
- 주요 KPI는 카드가 아니라 "계측 모듈"처럼: 좌측에 세로 발광 바 인디케이터 + 상단에 미니 아크 게이지
- 홀로그램 오버레이 톤: 반투명(rgba, alpha .06~.12) 패널 위에 발광 라인이 겹쳐 떠 있는 층위감

■ 히어로 연출 — "부팅 시퀀스" (총 1.6초, 1회만. 이 시안의 존재 이유)
1) 0.0s: 상단에 "SYSTEM INITIALIZING..." 모노 텍스트가 타이핑된다
2) 0.3s: 헤더 타이틀이 글리치 1회(2프레임 좌우 어긋남 + RGB 분광 1프레임) 후 발광하며 점등
3) 0.5s~: KPI 계측 모듈 4개가 90ms 간격으로 "전원 인가"되듯 보더가 먼저, 아크 게이지가 쓸리듯 차고, 수치가 나중에 켜진다
4) 0.9s~: 차트 막대·게이지가 계측기 바늘처럼 목표치를 4% 지나쳤다 되돌아오며(overshoot) 정착한다
5) 1.2s: 히어로 발광 링이 한 바퀴 스윕 스캔(진단 완료) 후 정지, 안쪽에 핵심 수치 점등
6) 1.4s: "문제" 상태가 경고등처럼 깜빡이기 시작 + 상태바에 "● ALL SYSTEMS ONLINE" 점등
부팅 후 화면은 안정 상태로 유지된다 (연출이 끝나도 앰비언트로 계속 "살아 있음").

■ 앰비언트 (상시, 각각 독립 주기로 은은하게 — 화면이 절대 "죽지" 않게)
- 스캔라인이 8초 주기로 화면을 느리게 훑는다
- 히어로 발광 링이 12초 주기로 아주 느리게 회전(또는 호흡하듯 밝기 맥동)
- 헤더 시계가 실제로 초 단위로 흐른다 (JS) + 미션 타이머/시퀀스 코드 표기
- "진행" 상태 항목 옆에 커서(_)가 깜빡인다
- 경고 발광이 1.4초 주기로 맥동한다

■ 마이크로 인터랙션 (최소 3개 — 조작감이 있어야 "가동"이 성립)
1) 계측 모듈 hover: 코너 브래킷·대괄호 마커가 바깥으로 2px 벌어지며 보더 발광 강화
2) 차트 막대/게이지 hover: 해당 지점 위에 십자 조준선(crosshair)과 수치 강조 팝업
3) 테이블 행 hover: 행 좌측에 시안 인디케이터 바가 슬라이드-인 + 행 배경 미세 발광
4) (선택) 상태 태그 hover: 스캔 라인이 태그 위를 1회 훑고 툴팁 점등

■ 시그니처
부팅 시퀀스 — 새로고침할 때마다 "시스템을 켜는" 1.6초의 의식.
발광 진단 링 스윕 스캔이 이 의식의 클라이맥스다 (04·20 어디에도 없는 요소).

■ 접근성·안전장치
- prefers-reduced-motion: 부팅·스캔라인·링 회전·깜빡임 전부 제거, 완성 상태 즉시 렌더
  (경고는 색으로만 구분, 발광 링은 정지 상태로 표시)
- 시계 JS 실패 시 정적 시각 표시 유지
- 발광 텍스트도 대비 4.5:1 이상 (발광은 장식, 가독은 본체 색이 담당)

■ 금지 (클러스터메이트로 오인되게 하는 요소를 명시 차단)
- 라운드 큰 카드, 파스텔, 흰 배경, 부드러운 그림자, 1초 초과 개별 애니메이션
- 글리치 2회 이상, 배경 전체가 통째로 흐르는 무빙 효과(스캔라인·링만 허용)
- [04 터미널과 구별] 모노스페이스 로그 스트림·타이핑되는 문단·`$ 프롬프트`·라인번호 콘솔·
  터미널 창 프레임(제목표시줄+신호등 점 3개) 금지. 수치는 계기(計器)지 로그가 아니다.
  텍스트를 좍 흘리지 말고, 수치·게이지·링으로 "계측"해서 보여준다.
- [20 다크 대시보드와 구별] 평범한 사각 위젯 그리드에 표준 라인/도넛 차트를 얹은 "관제 대시보드"
  레이아웃 금지. 격자에 균일 배치된 무표정 차트 카드, 범례+축만 있는 차트 라이브러리 기본 룩,
  연출 없이 처음부터 완성된 채 정지해 있는 정적 카드 금지 —
  차트는 반드시 부팅 시퀀스로 "계측되며 차오르고", 최소 1개 지표는 아크 게이지/발광 링으로 표현한다.
- 코너 브래킷·발광 프레임·스캔라인·부팅 연출 중 하나라도 빠지면 이 스타일이 아니다.

■ 자체 크리틱 루프
1) 부팅이 끝난 뒤 화면이 "죽어" 보이는가? → 앰비언트 연출(링 맥동·스캔라인·시계·경고등) 점검
2) 반대로 어지러운가? → 상시 연출 중 하나 제거 (경고등·발광 링은 유지)
3) 세계관에 안 맞는 요소(부드러운 라운드, 세리프, 로그 콘솔, 무표정 차트 그리드)가 있는가? → 제거
4) 이 화면을 04 터미널이나 20 다크 대시보드 옆에 놓았을 때 3초 안에 "얘는 인게임 HUD다"라고
   구별되는가? → 안 되면 발광 링·조준 프레임·부팅 연출을 더 밀어붙인다
5) 어둠의 비율 60% 이상, 동시 발광 3곳 이하인가?
```
