# STYLE 01 — SF 게임 HUD (매력 축: 연극성)

매력 가설: 사용자는 데이터가 아니라 "관제사가 된 나"를 산다. 고도화 = 세계관의 완성도.
마스터 프롬프트 뒤에 아래 블록을 붙여 사용한다.

```
[스타일 지시 — SF 게임 HUD v2]

■ 무드
우주선 관제 콘솔. 사용자는 데이터를 "조회"하는 게 아니라 시스템을 "가동"한다.
모든 연출은 기계가 깨어나고, 계측하고, 경고하는 서사를 따른다.

■ 팔레트
배경 #050810 / 시안 #00E5FF(주) / 마젠타 #FF2D78(경고·피크) / 앰버 #FFB62E(대기) /
텍스트 #C9EEF7 / 보조 #5D8FA3
발광은 시안·마젠타 두 색에만. 동시 발광 3곳 이하 — 어둠이 있어야 빛이 산다.

■ 타이포
제목 Orbitron(900, 대문자, 자간 .18em) / 수치·레이블 Share Tech Mono / 한글 본문 Pretendard
KPI 수치 30px 이상, 모든 수치는 모노스페이스 — 계기판은 숫자가 떨리면 안 된다.

■ 형태·질감
모서리를 사선으로 자른 클립 패널(clip-path), 1px 발광 보더, 패널 좌상단 발광 마커,
우하단 ㄱ자 코너 브래킷, 화면 전체에 스캔라인 오버레이(opacity .03 이하).

■ 히어로 연출 — "부팅 시퀀스" (총 1.6초, 1회만. 이 시안의 존재 이유)
1) 0.0s: 상단에 "SYSTEM INITIALIZING..." 모노 텍스트가 타이핑된다
2) 0.3s: 헤더 타이틀이 글리치 1회(2프레임 좌우 어긋남) 후 발광하며 점등
3) 0.5s~: KPI 패널 4개가 90ms 간격으로 "전원 인가"되듯 보더가 먼저, 내용이 나중에 켜진다
4) 0.9s~: 차트 막대가 계측기 바늘처럼 목표치를 4% 지나쳤다 돌아오며 차오른다
5) 1.4s: "문제" 상태가 경고등처럼 깜빡이기 시작 + 상태바에 "● ALL SYSTEMS ONLINE"
부팅 후 화면은 안정 상태로 유지된다.

■ 앰비언트 (상시, 각각 독립 주기로 은은하게)
- 스캔라인이 8초 주기로 화면을 느리게 훑는다
- 헤더 시계가 실제로 초 단위로 흐른다 (JS)
- "진행" 상태 항목 옆에 커서(_)가 깜빡인다
- 경고 발광이 1.4초 주기로 맥동한다

■ 마이크로 인터랙션 (최소 3개)
1) 패널 hover: 코너 브래킷이 바깥으로 2px 벌어지며 보더 발광 강화
2) 차트 막대 hover: 해당 시간대 위에 십자 조준선과 수치 강조
3) 테이블 행 hover: 행 좌측에 시안 인디케이터 바가 나타남

■ 시그니처
부팅 시퀀스 — 새로고침할 때마다 "시스템을 켜는" 1.6초의 의식.

■ 접근성·안전장치
- prefers-reduced-motion: 부팅·스캔라인·깜빡임 전부 제거, 완성 상태 즉시 렌더
  (경고는 색으로만 구분)
- 시계 JS 실패 시 정적 시각 표시 유지
- 발광 텍스트도 대비 4.5:1 이상 (발광은 장식, 가독은 본체 색이 담당)

■ 금지
라운드 큰 카드, 파스텔, 흰 배경, 부드러운 그림자, 1초 초과 개별 애니메이션,
글리치 2회 이상, 배경 전체가 움직이는 효과

■ 자체 크리틱 루프
1) 부팅이 끝난 뒤 화면이 "죽어" 보이는가? → 앰비언트 연출 점검
2) 반대로 어지러운가? → 상시 연출 중 하나 제거 (경고등은 유지)
3) 세계관에 안 맞는 요소(부드러운 라운드, 세리프 등)가 있는가? → 제거
4) 어둠의 비율 60% 이상, 동시 발광 3곳 이하인가?
```
