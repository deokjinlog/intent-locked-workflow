# STYLE 14 — 벤토 그리드 (매력 축: 조합/리듬)

매력 가설: 한눈에 다 보인다. 매력은 정보의 나열이 아니라 크기의 리듬.
고도화 = 비대칭의 균형. 마스터 프롬프트 뒤에 붙여 사용한다.

```
[스타일 지시 — 벤토 그리드 v2]

■ 무드
잘 짜인 도시락. 크기 다른 라운드 카드가 빈틈없이 맞물려 하나의 판을 이룬다.
Apple 키노트 요약 슬라이드 · Vercel 대시보드 · Toss 홈의 감각.
각 칸이 독립된 미니 위젯 — 큰 숫자 하나, 미니 차트 하나, 리스트 하나.

■ 팔레트
배경 #F4F4F5 (밝은 그레이) — 카드 사이 gap 으로 배경이 격자처럼 드러난다 /
카드 #FFFFFF 기본, 강조 카드 1~2개만 은은한 톤 (#FAFAFA 또는 포인트 4% 틴트) /
텍스트 #18181B / 보조 텍스트 #71717A / 포인트 #6366F1 / 문제 #EF4444 / 양호 #10B981
카드 경계는 보더가 아니라 gap + 그림자로 만든다.

■ 타이포
Pretendard — 큰 숫자 카드는 700~800 (36~56px), 카드 제목 라벨 600 (13px, 자간 넓게),
본문·리스트 400~500. 숫자와 라벨의 굵기 대비가 위젯 리듬을 만든다.

■ 형태 — 벤토의 문법 (비대칭이 곧 정체성)
CSS grid-template-areas 로 비대칭 배치. 4열 그리드 예시:
  "big   big   mid   kpi"
  "big   big   list  kpi"
  "chart chart list  small"
큰 카드 1 (2×2) + 중간 2~3 + 작은 여러 개. radius 20~24px, gap 16px.
카드 padding 24px, 그림자 0 1px 3px rgba(0,0,0,.06) — 얕게, 배경에 얹힌 느낌.
KPI 4개는 균일 나열이 아니라 크기·비중을 다르게 (하나는 세로 긴 카드 등).

■ 히어로 연출 — 판이 맞물리는 순간
로드 시 카드가 순차 stagger 로 등장.
0.0s 배경 렌더 → 0.1s 큰 카드부터 opacity 0→1 + translateY 12px→0 (400ms)
→ 각 카드 60ms 간격으로 좌상단→우하단 순서로 이어짐 (읽는 순서와 일치).
큰 숫자 카드는 등장 후 count-up (0 → 실제값, 600ms).
미니 차트 카드는 막대·라인이 등장 직후 차오른다 (500ms).

■ 마이크로 인터랙션
1) 카드 hover: translateY -4px + 그림자 0 8px 24px rgba(0,0,0,.10)
   — 판에서 그 칸만 살짝 떠오른다 (시그니처). transition 200ms ease-out
2) 큰 숫자 카드 hover: 숫자 옆 추세 화살표(▲/▼) 색 점등 (양호 그린 / 문제 레드)
3) 리스트 카드 항목 hover: 행 배경 #F4F4F5 로 살짝 채워짐 + 우측 chevron 4px 이동
4) 미니 차트 hover: 해당 막대 포인트 컬러 강조 + 툴팁 값 노출

■ 시그니처
크기 다른 카드의 빈틈없는 비대칭 배열 + hover 시 그 칸만 떠오르는 부양.

■ 접근성·안전장치
- prefers-reduced-motion: stagger·count-up·차오름·부양 이동 제거,
  hover 는 그림자 변화만 유지 (translate 제거)
- JS 실패 폴백: count-up 미동작 시 최종 숫자 그대로 표시,
  grid-template-areas 는 CSS만으로 성립 (JS 의존 X)
- 좁은 화면(<640px): 벤토 해제 대신 2열 유지 + 큰 카드만 전폭 (1열 스택 금지)
- focus-visible: 카드 포커스 시 포인트 인디고 2px 아웃라인 + offset 2px

■ 금지
다 같은 크기의 균일 카드 그리드, 카드 1열 세로 스택,
카드 사이 죽은 넓은 여백(gap 은 리듬용으로만 좁게), 배경과 카드가 같은 톤이라
경계가 사라지는 평면, 카드 없는 풀블리드 섹션, 보더 라인으로 칸 나누기,
큰 카드 없이 중간 카드만 반복(위계 실종), 세로로 길기만 한 화면.

■ 자체 크리틱 루프
1) 카드 크기가 최소 3단계(큰/중간/작은)로 나뉘고 비대칭인가? → 균일하면 재배치
2) 각 카드가 독립 위젯으로 읽히는가(숫자/차트/리스트 역할 분리)? → 중복이면 통합
3) gap 이 배경 격자로 기능하는가, 아니면 죽은 여백인가? → 넓으면 좁힌다
4) hover 부양이 없는 카드나, 균일 그리드로 회귀한 영역이 있는가? → 보정
```
