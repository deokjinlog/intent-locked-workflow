# STYLE 25 — 커머스 리테일 (매력 축: 활기/구매욕)

매력 가설: 셀러가 대시보드를 여는 순간 "오늘 뭐 팔렸지"가 궁금해서 스크롤하게 만든다. 매력은 정보가 아니라 상품이 살아 있다는 감각 — 큰 썸네일과 볼드한 브랜드 색이 매출을 "물건"으로 되돌린다.
고도화 = 상품 카드의 존재감과 프로모 배지가 화면을 얼마나 장터처럼 들썩이게 하는가. 마스터 프롬프트 뒤에 붙여 사용한다.

```
[스타일 지시 — 커머스 리테일 v2]

■ 무드
셀러 어드민 감성 — 쇼핑몰 관리자가 매일 아침 여는 매출 대시보드.
화이트 배경 위에 볼드한 브랜드 색이 팡 터지고, 큰 상품 썸네일이 격자로 늘어선다.
차분한 분석 툴이 아니라 물건이 팔리는 활기찬 장터. 숫자마다 "지금 잘 나간다"는 온도가 있다.
모던하고 볼드하되 경박하지 않게 — 브랜드 색은 강하지만 배경은 깨끗해서 상품이 주인공.

■ 팔레트 — 화이트 배경 + 볼드 듀오톤 브랜드
배경 #FFFFFF / 서피스 #FFFFFF / 캔버스틴트 #FFF7F7(코랄 5%) / 보더 #F0E9F5 / 라인 #ECECF2
코랄 primary #FF5A5F / 코랄-강 #E8484D / 코랄-연 #FFE7E8(배지·칩 배경) / on-코랄 #FFFFFF
퍼플 secondary #7B2FF7 / 퍼플-강 #6522D6 / 퍼플-연 #EFE4FF(배지·칩 배경) / on-퍼플 #FFFFFF
텍스트 #1B1533 / 서브 #6B6480 / 힌트 #A29CB5
프로모/세일 #FF5A5F · 신규 #7B2FF7 · 재고임박 #F5A623 · 매출↑성공 #17B26A · 반품/문제 #E8484D
KPI 상승은 성공 초록, 하락은 문제 빨강 — 화살표+색 동시. 코랄↔퍼플은 히어로 그라디언트에서만 섞는다.

■ 타이포 — 볼드 헤드라인 + 또렷한 숫자
본문 한글은 Pretendard 가정. 영문·숫자·라벨은 Manrope (구글폰트) 명시 로드.
매출 숫자·KPI 값은 Manrope 800 tabular-nums, letter-spacing -0.02em.
Display 34/40 800 · 페이지 타이틀 26/32 800 · 카드 타이틀 18/24 700 ·
상품명 15/20 600 · 본문 14/20 500 · 배지·라벨 12/16 700 대문자 letter-spacing 0.04em ·
캡션 12/16 500. 브랜드 로고 워드마크는 코랄+퍼플 2색 조합, 숫자는 절대 얇게 쓰지 않는다.

■ 형태
radius 큼 — 카드 16px, 상품 썸네일 14px, 버튼·인풋 12px, 배지·칩 pill(999px), 모달 20px.
상품 썸네일은 정사각 64~72px, 이니셜 1~2자를 브랜드 색 그라디언트 배경 위 화이트 800으로.
카드 보더는 1px #F0E9F5 + 그림자 0 2px 8px rgba(27,21,51,.06), hover 시 0 8px 24px rgba(255,90,95,.14).
프로모 배지는 pill, 코랄/퍼플 solid 배경 + 화이트 텍스트, 카드 우상단에 살짝 겹쳐 얹힌다.
primary 버튼은 코랄 solid, secondary는 퍼플 outline. 여백은 촘촘하게 — 상품이 많아 보이도록.

■ 히어로 연출 — 매대에 물건이 깔리는 순서 (로드 시퀀스)
0.0s 상단바가 화이트로 고정, 좌측 브랜드 워드마크가 코랄→퍼플 그라디언트로 0.4s 좌→우 스윕
0.15s KPI 4장이 왼→오 50ms stagger, y+12px→0 + 페이드, 숫자는 0에서 목표값까지 700ms 카운트업
0.5s 상품 카드 그리드가 썸네일부터 pop-in(scale 0.9→1, 40ms 간격), 이니셜 배경 그라디언트 선명해짐
0.7s 프로모 배지가 카드 우상단에서 scale 0.6→1 + 살짝 튀는 백(overshoot 1.08→1)으로 톡 붙음
0.9s 매출 차트 막대가 baseline에서 코랄 그라디언트로 550ms ease-out 차오름, 최고 막대만 퍼플 하이라이트
1.1s 테이블(주문 목록) 행이 상단부터 30ms 간격 페이드인, 재고임박 행은 앰버 좌측 바 깜빡 1회

■ 마이크로 인터랙션 (이 스타일의 실체)
1) 상품 카드 hover: y-4px 부상 + 코랄 그림자 확산 + 썸네일 미세 zoom(scale 1.04), "판매관리" 액션 버튼이 하단에서 슬라이드업
2) KPI 카운트업: 로드·필터 변경 시 값이 이전 수에서 새 수로 500ms 굴러 오르고, 상승이면 초록 화살표가 위로 튕김
3) 프로모 배지 hover: 살짝 흔들(1° 좌우 wiggle 300ms) + 툴팁으로 프로모 기간 노출 — 지금 이벤트 중임을 각인
4) 장바구니/찜 토글: 하트·태그 아이콘이 코랄로 채워지며 pop(scale 1.3→1) + 작은 파티클 2~3개 튐
5) 필터 칩 선택: pill이 코랄 solid로 채워지며 화이트 텍스트, 해제 시 연코랄 배경으로 부드럽게 복귀
6) 매출 목표 게이지: 목표 달성 시 게이지가 코랄→퍼플 그라디언트로 채워지고 100% 순간 콘페티 한 번

■ 시그니처
큰 이니셜 썸네일을 얹은 상품 카드 + 코랄·퍼플 볼드 듀오톤 브랜드 + 카드 우상단에 겹쳐 붙은 pill 프로모 배지.
이 셋(상품 카드 · 볼드 브랜드 · 프로모 배지)이 화면을 즉시 "파는 곳"으로 읽히게 하는 각인 — 분석 툴이 아니라 매대.

■ 접근성·안전장치
- prefers-reduced-motion: 카운트업·pop-in·wiggle·콘페티·파티클 제거, 값은 즉시 최종 표시(색 상태만 유지)
- JS 실패 폴백: 카드 hover 액션 버튼은 CSS :hover만으로 노출, 카운트업 없이 최종 숫자 정적 렌더
- focus-visible: 코랄 3px 아웃라인 + offset 2px, 배지·칩·카드 모두 키보드 포커스 링 보장
- 대비: 코랄/퍼플 solid 위 화이트 텍스트 4.5:1 확보, 연배경 배지는 강한 텍스트색 사용, 상태는 색+아이콘 병행(색맹 대비)
- 프로모 배지 다발 제한: 카드당 배지 최대 2개, 화면 전체가 세일 딱지로 뒤덮이지 않게 우선순위 정렬

■ 금지 (정체성의 절반)
칙칙한 무채색 그레이스케일 대시보드, 과한 절제·미니멀 여백(상품이 듬성듬성 보이면 안 됨),
다크 배경 강제(화이트 기본 — 다크는 옵션조차 이 스타일 아님), 밋밋한 단색 플랫 톤,
세리프 서체, 얇은 숫자(라이트/씬 웨이트), 글래스모피즘·backdrop-filter, 브루탈리즘식 하드엣지 순검정 보더,
radius 0 또는 4px 이하 각진 카드, 그림자 전무한 완전 플랫, 코랄·퍼플을 회색으로 죽여 쓰기,
프로모 배지 생략, 이니셜 썸네일 없이 텍스트만 나열한 상품 목록(매대감 상실).

■ 자체 크리틱 루프
1) 상품 카드에 큰 이니셜 썸네일·상품명·프로모 배지가 갖춰져 "매대"로 읽히는가? — 밋밋한 행 나열로 퇴화 안 했는지 검사
2) 코랄·퍼플 볼드 듀오톤이 화이트 배경 위에서 활기 있게 살아 있는가? — 무채색·저채도로 죽지 않았는지 색 전수 점검
3) KPI 숫자가 볼드 tabular이고 상승/하락이 화살표+색으로 즉시 읽히는가? — 얇은 숫자·색 없는 지표 색출
4) 프로모 배지가 존재하되 카드당 2개 이하로 절제되어 매출 데이터를 가리지 않는가? — 배지 과잉·세일 딱지 도배 이탈 검사
```
