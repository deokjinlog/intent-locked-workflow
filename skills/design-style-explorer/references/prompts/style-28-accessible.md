# STYLE 28 — 하이컨트라스트 접근성 (매력 축: 명료/포용)

매력 가설: 아무도 배제하지 않는 화면은 그 자체로 신뢰를 준다. 매력은 화려함이 아니라 확신 — 저시력·색약·노안·집중장애 누구에게나 "여기가 눌러야 할 곳"이 즉시 읽힌다.
고도화 = WCAG AAA를 넘어서는 여백·굵기·터치 타겟의 자신감. 마스터 프롬프트 뒤에 붙여 사용한다.

```
[스타일 지시 — 하이컨트라스트 접근성 v2]

■ 무드
접근성 우선(WCAG AAA)의 공적 신뢰감. 순백 위에 진한 잉크로 또박또박 새긴 게시판.
장식을 덜어내고 대비와 굵기로만 위계를 세운다. 흐릿함·모호함이 없는 세계 —
모든 상태는 반드시 색과 텍스트와 아이콘 셋으로 동시에 말한다. 명료함이 곧 배려.

■ 팔레트 — 순백 배경 + 고대비 잉크/액센트
배경 #FFFFFF / 카드 #FFFFFF / 옅은 존 구분 #F4F5F7 (테두리로만 구분, 채우기 의존 X)
잉크(본문·헤딩) #111111 / 보조 텍스트 #2A2A2A (회색 아님 — AAA 유지 위해 진하게)
보더·구분선 #111111 (기본 2px) / 포커스 링 #0B5FFF
액센트 진파랑 #0B5FFF (링크·주요 액션·선택) / on-blue #FFFFFF
액센트 진초록 #067647 (성공·정상) / on-green #FFFFFF
경고 #8A5A00 (텍스트+아이콘 병행) / 문제 #B00020 (텍스트+아이콘 병행)
회색 텍스트·파스텔은 팔레트에 없음 — 대비 4.5:1 미만 색 전면 금지.

■ 타이포 — 명료 산세리프, 크게
본문 한글 Pretendard 가정. 영문·숫자·라벨은 Inter (구글폰트) 명시 로드.
본문 최소 16px(1rem) — 그 이하 없음. 헤딩 굵게(700~800), 본문 500, 라벨 600.
Display 40/48 800 · H1 30/38 700 · H2 24/32 700 · Title 20/28 700 ·
Body 16/26 500 · Label 15/20 600 · 최소 캡션도 14/20 500 이상.
line-height 넉넉히(본문 1.6), letter-spacing 0 기준(가독 우선, 자간 압축 금지).

■ 형태
radius 8~10px(과한 곡률 X, 각진 명료함 유지). 모든 카드·입력·버튼에 2px solid #111 보더.
선택·활성 요소는 보더 3px로 승격. 그림자 최소 — 위계는 보더 굵기와 여백으로 표현.
큰 터치 타겟: 모든 인터랙티브 요소 최소 48×48px, 버튼 padding 14×20 이상.
입력 필드 높이 48px+, 라벨은 항상 필드 위 좌측 정렬 표시(placeholder-only 금지).

■ 히어로 연출 — 또렷하게 자리잡는 순서 (로드 시퀀스, 절제된 모션)
0.0s 상단바가 2px 하단 보더와 함께 즉시 표시 (페이드 120ms, 이동 없음)
0.15s KPI 카드 4장이 좌→우 50ms씩 stagger, opacity 0→1 (각 160ms, 이동 최소 4px 이내)
0.4s 각 KPI의 큰 숫자가 0에서 목표값까지 카운트업 (600ms, 감속) + 상태 배지 함께 표시
0.6s 차트 막대가 baseline에서 위로 차오름 (450ms) — 각 막대에 값 라벨 텍스트 동시 노출
0.8s 테이블 헤더 고정, 본문 행이 상단부터 40ms 간격 페이드 (이동 없이 opacity만)
모든 등장은 이동 폭을 4px 이내로 억제 — 흔들림 없이 또렷하게 "박히는" 느낌.

■ 마이크로 인터랙션 (이 스타일의 실체)
1) 포커스 이동: Tab 이동 시 대상에 3px #0B5FFF 아웃라인 + offset 3px가 즉시 나타남
   (마우스 클릭엔 억제, 키보드 포커스에만 focus-visible로 강조) — 절대 outline:none 없음
2) 버튼 hover: 배경/글자 색이 반전(진파랑↔흰색)되며 보더 2px→3px, 100ms 즉각 반응
3) 행 hover: 행 전체에 #F4F5F7 배경 + 좌측 4px 진파랑 마커 바 등장, 커서 pointer
4) 상태 배지: 정상/경고/오류가 색+텍스트+아이콘 3중 표기, hover 시 툴팁으로 전체 설명 노출
5) 토글·체크: on 상태가 진초록 채움 + 체크 아이콘 + "켜짐" 텍스트로 동시 확정(색만 변화 금지)

■ 시그니처
순백 위 2px 잉크 보더로 구획된 카드들 + 색·텍스트·아이콘 3중으로 말하는 상태 배지 +
어디를 눌러도 큼직한 48px 터치 타겟과 3px 진파랑 포커스 링. 이 조합이 화면을 즉시
"모두를 위한 화면"으로 각인시킨다.

■ 접근성·안전장치
- prefers-reduced-motion: 카운트업·차오름·stagger 제거, 최종 상태 즉시 표시(대비는 그대로)
- JS 실패 폴백: 상태 배지·라벨·포커스 링 모두 CSS·정적 마크업만으로 완전 동작
- focus-visible: 전 인터랙티브 요소 3px #0B5FFF 아웃라인 + offset, outline 제거 절대 금지
- 대비: 본문·라벨 전부 7:1(AAA) 목표, 큰 텍스트도 4.5:1 이상 — 회색 본문 사용 금지
- 상태 이중·삼중화: 색만으로 정보 전달 금지, 반드시 텍스트+아이콘 병행(색약 안전)
- 큰 터치 타겟: 밀집 테이블도 행 높이 48px+, 인접 버튼 간 8px 이상 간격 확보

■ 금지 (정체성의 절반)
저대비 얇은 회색 텍스트(#999·#AAA류), 색만으로 구분하는 상태 표기,
16px 미만 본문·작은 캡션 남용, 흐린 파스텔·반투명 위계, outline:none·포커스 링 제거,
placeholder-only 라벨, 48px 미만 터치 타겟, 자간 압축(letter-spacing 음수),
장식용 그라디언트·글로우·그림자 위계, 보더 없는 완전 플랫 카드, 세리프 본문 남용,
얇은 1px 연회색 구분선 의존(2px 잉크 보더가 기본).

■ 자체 크리틱 루프
1) 모든 상태(정상·경고·오류·선택)가 색+텍스트+아이콘 3중으로 표기되어 색약에도 구별되는가?
2) 전 인터랙티브 요소에 3px 포커스 링이 살아 있고 outline:none 잔재가 없는가? — 전수 검사
3) 본문·라벨 대비가 7:1(AAA)를 만족하고 회색 저대비 텍스트가 섞이지 않았는가? — 이탈 색 색출
4) 모든 버튼·행·입력의 터치 타겟이 48×48px 이상이고 본문이 16px 이상인가? — 최소치 위반 색출
```
