# STYLE 15 — 클레이모피즘 (매력 축: 장난기/말랑)

매력 가설: 만지면 말랑할 것 같다. 매력은 정보가 아니라 손끝의 촉감 예감 — 통통하게 부푼 점토 덩어리를 엄지로 꾹 눌러 자국을 남기고 싶은 충동. 화면이 "표면"이 아니라 "부피를 가진 반죽"으로 읽혀야 한다.
고도화 = "점토의 물성"(두께·눌림·복원·탄성)의 시뮬레이션. 유리(투명)도 아니고 저대비 요철(뉴모)도 아닌, 채도 있는 파스텔 반죽이 실제로 3D로 부풀어 오른 상태. 마스터 프롬프트 뒤에 붙여 사용한다.

```
[스타일 지시 — 클레이모피즘 v2]

■ 무드
채도 있는 파스텔 점토로 조몰락조몰락 빚어낸 친근한 앱. 키즈 앱·핀테크 온보딩·이모지 스티커처럼 무해하고 통통하고 말랑하다.
모든 요소가 "면"이 아니라 "덩어리"다 — 두껍게 부풀어 있고, 손대면 쏙 눌렸다 천천히 되돌아올 것 같은 반죽의 물성. 화면 전체가 손으로 빚은 점토 장난감 트레이처럼 보여야 한다.
결정적 대비 축: 이 스타일은 불투명(opaque)한 색 반죽이다. 비쳐 보이는 유리도, 배경과 거의 같은 색의 은은한 요철도 아니다. 색이 꽉 찬 통통한 3D 덩어리.

■ 팔레트 (채도 있는 파스텔이 정체성의 뿌리 — 무채색·탁색 진입 시 즉시 회귀)
배경 #F4F1FF (라벤더 화이트) — 밝고 채도 있는 파스텔이 전제. 회색빛(#EEE 계열) 절대 금지.
카드 표면 #FFFFFF~#FBF9FF (완전 불투명) / 라벤더 #B8B3F5 / 민트 #A8E6CF / 피치 #FFC7A8 / 소프트옐로 #FFE7A0 / 스카이 #A8D8F5
포인트(주요 버튼·강조) #B8B3F5 (라벤더) — 채운 통통 알약. hover 시 #A49EF0 로 진해짐.
문제·경고 #FF9AA2 (붉은 파스텔, 채도 유지 — 절대 회색 경고 금지) / 성공 #A8E6CF (민트).
텍스트 #4A4266 (진한 라벤더-그레이 — 무채색 #666/#333 금지, 반드시 배경 색 계열의 어두운 톤이라 점토 세계 안에 속함).
KPI 4장은 라벤더 #B8B3F5 · 민트 #A8E6CF · 피치 #FFC7A8 · 소프트옐로 #FFE7A0 순환 배정 — 색 자체로 구획을 나눈다. 4장이 전부 흰색이면 회귀(색 반죽의 다양성 상실).
그림자 색도 채도를 가진다: 검정 반투명이 아니라 라벤더 톤 rgba(120,110,180,·) — 점토 세계 안의 색 그림자.

■ 타이포 (통통·둥긂 = 점토와 공명, 얇음은 정체성 파괴)
Quicksand 700 (헤더·KPI 숫자) — 둥근 산세리프의 통통함이 점토와 공명. 획 끝이 둥글어야 반죽과 한 몸.
한글 본문은 Pretendard 600~700 굵게 — 둥근 라틴과 무게·둥근맛 맞춤.
숫자·라벨 자간 살짝 넓혀(letter-spacing +0.3~0.5px) 통통한 여백 유지. font-weight 500 미만 얇은 웨이트 전면 금지 — 얇은 글자는 점토 위에 얹힌 종잇조각처럼 이물감을 준다.
제목은 살짝 크게(fluid, clamp) 잡아 덩어리감 강조. 헤더도 부푼 캡슐 배경 위에 얹을 수 있다.

■ 형태 — 점토의 문법 (두께 = 이중 소프트 섀도, radius = 극단적 둥긂)
모든 카드·버튼·아이콘 칩: 아주 큰 radius — KPI 카드 32px, 일반 카드 28px 이상, 버튼 24px, 알약 태그·상태 칩 999px(완전 캡슐), 입력창 20px 이상. 16px 이하 작은 곡률·직각 모서리 전면 금지.
두께는 반드시 이중 소프트 섀도로 만든다 (광원 고정: 좌상단):
  바깥 진한 그림자(덩어리를 바닥에서 띄움): box-shadow: 12px 12px 24px rgba(120,110,180,0.28)
  안쪽 밝은 하이라이트(윗면이 볼록하게 부풀어 빛 받는 느낌): inset -6px -6px 12px rgba(255,255,255,0.9)
  선택적 3겹째(더 통통하게): inset 4px 4px 10px rgba(120,110,180,0.15) 로 아래쪽 살짝 그늘.
  → 바깥 진한 그림자 + 안쪽 흰 하이라이트, 이 두 겹이 반드시 동시에 있어야 "부풀어 오른 점토". 한 겹만이면 즉시 회귀. 그림자 거리(12px+)와 blur(24px)가 충분히 커야 "얇은 요철"이 아니라 "통통한 부피"로 읽힌다 — 거리 4px 이하 미묘한 그림자는 뉴모피즘 회귀.
카드 배경은 100% 불투명 색면. 반투명(rgba alpha<1)·backdrop-filter·blur 로 뒤가 비치는 표현 전면 금지 — 그건 유리이지 점토가 아니다.
카드끼리 넉넉한 gap(24px+) — 각 덩어리가 트레이 위 개별 점토 조각으로 또렷이 떨어져 읽히게. 카드끼리 붙거나 겹쳐 경계가 뭉개지면 회귀.
아이콘은 채운 둥근 실루엣(rounded-fill), 픽토그램도 두툼하게 부풀린다. 1px 라인아트 아이콘 금지.
보더는 쓰지 않는다 — 경계는 오직 그림자와 색면 차이로. 1px 얇은 구분선·테두리는 점토 세계에 존재하지 않는다.

■ 히어로 연출 — 부풀어 오르는 로드 (초 단위, 탄성이 주인공)
0.0s: 모든 카드 scale(0.9) + opacity 0, 살짝 아래(translateY 12px).
0.0~0.5s: 카드가 순차(각 80ms stagger)로 scale(1)까지 통통 튀며 등장
         — cubic-bezier(0.34, 1.56, 0.64, 1) 오버슈트(1.0을 살짝 넘어 부풀었다 복원)로 말랑함 표현. 이 오버슈트가 점토 탄성의 핵심.
0.4~1.1s: KPI 숫자 0에서 목표값까지 카운트업(둥근 숫자가 굴러오듯).
0.5~1.3s: 차트 막대가 아래에서 통통 튀며 차오름(끝에서 살짝 오버슈트 후 정착).
과한 파티클·회전·플래시 없음 — 등장의 리듬과 오버슈트 자체가 점토의 탄성이다. 딱딱한 선형 ease·페이드만으로 등장하면 회귀.

■ 마이크로 인터랙션 — 눌림과 복원 (이 스타일의 심장, 촉각의 전부)
1) 버튼/카드 hover: scale(1.03) + 바깥 그림자 거리 확대(16px 16px) — 손을 대면 반죽이 더 부푼다.
2) active(클릭·터치): scale(0.94) + 바깥 그림자 축소(6px) + inset 그림자 강화로 엄지에 꾹 눌린 점토
   — 100ms 눌림 후 cubic-bezier(0.34,1.56,0.64,1)로 천천히 복원(180ms). 모든 인터랙티브 요소 필수. 눌렸을 때 안쪽 그림자가 깊어지고 바깥 그림자가 얕아지는 "반전"이 핵심 촉각.
3) 아이콘 칩 hover: 8도 살짝 기울며 통 튀는 wobble(150ms) — 젤리처럼.
4) 상태 태그 hover: 통통한 캡슐이 살짝 부풀며 색이 한 단계 진해짐(파스텔→살짝 채도 up).
5) 차트 막대 hover: 해당 막대만 포인트색 점등 + 2px 부풀기(막대도 둥근 반죽 덩어리).
색·투명도만 바뀌고 형태(scale·그림자)가 안 움직이면 회귀 — 점토는 반드시 물리적으로 눌리고 부푼다.

■ 시그니처
통통한 클레이 버튼·아이콘 — 누르면 scale(0.94)로 실제로 쏙 들어갔다가 오버슈트 곡선으로 말랑 되돌아오는 촉각 반응. 이중 섀도(바깥 진함 + 안쪽 흰 하이라이트)가 눌림 순간 반전되어, 화면을 만지면 손끝에 반죽의 저항과 복원이 느껴지는 착각을 준다. 채도 있는 색 덩어리들이 트레이 위에 통통 떨어져 있는 장난감 상자 같은 첫인상.

■ 접근성·안전장치
- prefers-reduced-motion: 부풀어오름·튐·카운트업·wobble·오버슈트 제거. 눌림 반응은 scale 없이 그림자 축소(바깥↓ 안쪽↑)만 유지해 피드백 보존.
- JS 실패 폴백: 카운트업·stagger가 JS 의존이면 최종 상태(카드 opacity 1, 숫자 완성값, 막대 목표 높이)를 CSS 기본값으로 둬 no-JS에서도 완성 화면.
- focus-visible: 포인트 라벤더 3px 아웃라인 + 2px offset — 소프트 섀도만으로는 포커스 식별 불가하므로 명시 링 필수(그림자로 대체 금지).
- 파스텔 저대비 보완: 본문 텍스트 #4A4266 이상 진하게(대비 4.5:1 확보). 흰/파스텔 배경 위 파스텔 텍스트 금지. 채도는 유지하되 명도만 낮춰 가독성 확보.

■ 금지 (정체성의 절반 — 특히 클러스터메이트와의 경계)
[유리(06 글래스모피즘)로 새는 것 차단]
반투명 카드(rgba alpha<1)·backdrop-filter·blur·frosted glass / 뒤 배경이 비쳐 보이는 표현 / 얇은 반투명 보더 / 유리 광택 하이라이트. — 점토는 불투명한 색 덩어리다. 절대 비치지 않는다.
[뉴모(12 뉴모피즘)로 새는 것 차단]
배경과 거의 같은 색의 저대비 미묘한 요철 / 얇고 은은한 soft-UI 음영(그림자 거리 4px 이하·blur 8px 이하) / 표면에 살짝 파묻힌 듯한 눌린 홈 / 무채색 회색 팔레트(#E0E0E0·#EEE 계열) / 색 없는 미묘한 명암만의 입체. — 점토는 채도 있는 색이 크고 진한 이중 그림자로 크게 부풀어야 한다. 미묘함은 이 스타일의 반대말.
[공통 금지]
다크·무채색 배경 / 낮은 채도·탁한 회색·비즈니스 정장 톤 / 작은 radius(16px 이하)·하드 엣지·직각 모서리 / 세리프체·font-weight 500 미만 얇은 산세리프 / 1px 얇은 보더 라인·구분선 / 플랫 카드(그림자 없음) 또는 한 겹 그림자만 / 하드 드롭섀도(검은 반투명 단일) / 채도 없는 회색 그림자 / 배경과 같은 흰 카드를 그림자 없이 겹치기 / 날카로운 아이콘 라인아트(outline) / 색·투명도만 바뀌고 형태는 안 움직이는 인터랙션.

■ 자체 크리틱 루프
1) 모든 카드·버튼에 이중 섀도(바깥 진한 라벤더 그림자 12px+ 거리 + 안쪽 밝은 흰 하이라이트)가 다 있는가? → 한 겹뿐이거나 그림자 거리 4px 이하인 요소 전수 보정(뉴모 회귀 색출).
2) 반투명·blur·backdrop-filter로 뒤가 비치는 요소가 하나라도 있는가? → 전부 불투명 색면으로 교체(글래스 회귀 색출).
3) radius가 어디선가 28px 미만으로 샜거나(태그·칩 제외) 직각이 남았는가? → 극단적 둥긂으로 보정.
4) 팔레트가 채도를 잃고 회색·탁색으로 흘렀는가? KPI 4장이 전부 흰색인가? → 라벤더·민트·피치·옐로 색 반죽으로 되돌림.
5) 눌림+복원(scale 0.94 + 그림자 반전) 반응이 없는 인터랙티브 요소가 있는가? → active 오버슈트 추가.
6) 파스텔 저대비로 텍스트가 묻힌 곳(4.5:1 미달)은? → 텍스트 톤 진하게 보정(명도만, 채도는 유지).
```
