# STYLE 15 — 클레이모피즘 (매력 축: 장난기/말랑)

매력 가설: 만지면 말랑할 것 같다. 매력은 정보가 아니라 손끝의 촉감 예감 — 통통한 점토를 꾹 누르고 싶은 충동.
고도화 = "점토의 물성"(두께·눌림·복원)의 시뮬레이션. 마스터 프롬프트 뒤에 붙여 사용한다.

```
[스타일 지시 — 클레이모피즘 v2]

■ 무드
파스텔 점토로 빚은 친근한 앱. 키즈 앱·핀테크 온보딩처럼 무해하고 말랑하다.
모든 요소가 두껍게 부풀어 있고, 손대면 눌렸다 천천히 되돌아올 것 같은 물성.

■ 팔레트
배경 #F4F1FF (라벤더 화이트) — 밝고 채도 있는 파스텔이 전제.
카드 표면 #FFFFFF~#FBF9FF / 라벤더 #B8B3F5 / 민트 #A8E6CF / 피치 #FFC7A8
포인트(주요 버튼·강조) #B8B3F5 / 문제·경고 #FF9AA2 (붉은 파스텔, 채도 유지)
텍스트 #4A4266 (진한 라벤더-그레이 — 무채색 금지, 배경과 같은 색 계열의 어두운 톤).
KPI 4장은 라벤더·민트·피치·소프트옐로 #FFE7A0 순환 배정 — 색으로 구획을 나눈다.

■ 타이포
Quicksand 700 (헤더·KPI 숫자) — 둥근 산세리프의 통통함이 점토와 공명.
한글 본문은 Pretendard 600~700 굵게 — 둥근 라틴과 무게 맞춤.
숫자·라벨 자간 살짝 넓혀 통통한 여백 유지. 얇은 웨이트 금지.

■ 형태 — 점토의 문법 (두께 = 이중 소프트 섀도)
모든 카드·버튼·아이콘 칩: 큰 radius 28px 이상 (KPI 카드 32px, 버튼 24px, 알약 태그 999px).
이중 소프트 섀도로 두께를 만든다 (광원 좌상단):
  바깥 진한 그림자: box-shadow: 12px 12px 24px rgba(120,110,180,0.28)
  안쪽 밝은 하이라이트: inset -6px -6px 12px rgba(255,255,255,0.9)
  → 이 두 겹이 동시에 있어야 "부풀어 오른 점토". 한 겹만 쓰면 회귀.
카드끼리 넉넉한 gap(24px+) — 각 덩어리가 개별 점토 조각으로 읽히게.
아이콘은 채운 둥근 실루엣(rounded-fill), 픽토그램도 두툼하게.

■ 히어로 연출 — 부풀어 오르는 로드 (초 단위)
0.0s: 모든 카드 scale(0.9) + opacity 0, 살짝 아래(translateY 12px).
0.0~0.5s: 카드가 순차(각 80ms stagger)로 scale(1)까지 통통 튀며 등장
         — cubic-bezier(0.34, 1.56, 0.64, 1) 오버슈트(살짝 부풀었다 복원)로 말랑함 표현.
0.4~1.1s: KPI 숫자 0에서 목표값까지 카운트업.
0.5~1.3s: 차트 막대가 아래에서 통통 튀며 차오름(끝에서 살짝 오버슈트).
과한 파티클·회전 없음 — 등장의 리듬 자체가 점토의 탄성.

■ 마이크로 인터랙션 — 눌림과 복원 (이 스타일의 심장)
1) 버튼/카드 hover: scale(1.03) + 바깥 그림자 거리 확대(16px) — 손을 대면 더 부푼다.
2) active(클릭·터치): scale(0.94) + 바깥 그림자 축소 + inset 그림자 강화로 꾹 눌린 점토
   — 100ms 눌림 후 cubic-bezier(0.34,1.56,0.64,1)로 천천히 복원(180ms). 모든 인터랙티브 요소 필수.
3) 아이콘 칩 hover: 8도 살짝 기울며 통 튀는 wobble(150ms).
4) 상태 태그 hover: 통통한 알약이 살짝 부풀며 색이 진해짐.
5) 차트 막대 hover: 해당 막대만 포인트색 점등 + 2px 부풀기.

■ 시그니처
통통한 클레이 버튼·아이콘 — 누르면 scale(0.94)로 실제로 쏙 들어갔다가
오버슈트 곡선으로 말랑 되돌아오는 촉각 반응. 이중 섀도가 눌림에 반전.

■ 접근성·안전장치
- prefers-reduced-motion: 부풀어오름·튐·카운트업·wobble 제거. 눌림 반응은 scale 없이 그림자 축소만 유지(피드백 보존).
- JS 실패 폴백: 카운트업·stagger가 JS 의존이면 최종 상태(카드 opacity 1, 숫자 완성값, 막대 목표 높이)를 CSS 기본값으로 둬 no-JS에서도 완성 화면.
- focus-visible: 포인트 라벤더 3px 아웃라인 + 2px offset — 소프트 섀도만으로 포커스 식별 불가하므로 명시 링 필수.
- 파스텔 저대비 보완: 본문 텍스트 #4A4266 이상 진하게(4.5:1 확보). 흰 배경 위 파스텔 텍스트 금지.

■ 금지 (정체성의 절반)
다크·무채색 배경 / 낮은 채도·탁한 회색 / 작은 radius(16px 이하)·하드 엣지·직각 모서리 /
세리프체·얇은 산세리프 웨이트 / 1px 얇은 보더 라인·구분선 / 플랫 카드(그림자 없음 또는 한 겹만) /
하드 드롭섀도(검은 반투명 단일) / 배경과 같은 흰 카드를 그림자 없이 겹치기 /
날카로운 아이콘 라인아트(outline) / 채도 없는 텍스트·비즈니스 정장 톤.

■ 자체 크리틱 루프
1) 모든 카드·버튼에 이중 섀도(바깥 진한 그림자 + 안쪽 밝은 하이라이트)가 다 있는가? → 한 겹뿐인 요소 전수 보정.
2) radius가 어디선가 28px 미만으로 새지 않았는가? → 직각·작은 곡률 색출.
3) 눌림+복원 반응이 없는 인터랙티브 요소가 있는가? → active 오버슈트 추가.
4) 파스텔 저대비로 텍스트가 묻힌 곳(4.5:1 미달)은? → 텍스트 톤 진하게 보정.
```
