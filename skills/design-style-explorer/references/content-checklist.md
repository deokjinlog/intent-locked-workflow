# 콘텐츠 견고성 체크리스트 — 모양 말고 안의 내용

디자인 시안을 볼 때 **겉모양뿐 아니라 "내 최악의 콘텐츠"를 견디는지** 확인하는 리스트.
생성 직전(품질 기준)과 사용자 리뷰 때 함께 본다. style-lint.py(금지 검사)와 **직교**하는 축이다.

> 핵심 원칙: **템플릿은 베스트 케이스(짧은 제목·예쁜 숫자)를 팔고, 나는 워스트 케이스를 산다.**
> 예시 데이터가 아니라 **최악의 콘텐츠 기준으로 판단**한다.

## 컴포넌트 커버리지 (역량 시연 — research 기반)

디자인 역량은 컴포넌트 다양성 × 상태 처리 × 정보 위계로 판단된다. 한 화면에 아래를 담되 **요약 먼저(progressive disclosure)** 로 정리한다 (과밀 금지):

- [ ] **상단바** — 검색 인풋 · 알림 벨(뱃지) · 아바타 · 주요/보조 버튼
- [ ] **탭 + 필터 드롭다운** — 선택 상태가 뚜렷한가
- [ ] **알림 배너** — info/경고 피드백 컴포넌트
- [ ] **KPI + 미니 스파크라인 + 증감(+/−)**
- [ ] **차트 2종+** — 추이(라인/영역) + 구성비(도넛/파이) + 목표 진행률(바/게이지)
- [ ] **테이블** — 정렬 표시 · 상태 · 아바타/썸네일 · 행 액션 + 엣지케이스
- [ ] **활동 피드 또는 빈 상태** — 빈 상태는 헤드라인 → 설명 → CTA

> 시각 위계(size·contrast·color·spacing·alignment)로 "먼저 볼 것"이 3초 안에 잡히는가 — 이게 역량의 핵심.

## 공통 — 모든 시안 (스타일 무관)

- [ ] **큰 숫자 안 깨짐** — 억/조·10자리(1,234,567,890)도 `tabular-nums`로 정렬 유지, 카드 밖으로 안 넘침
- [ ] **긴 제목/라벨** — 2줄로 wrap 되거나 말줄임(ellipsis) 처리, 옆 요소 안 밀침
- [ ] **테이블 견고성** — 아주 긴 셀 텍스트(말줄임), 결측값 `—`, 행이 늘어도 정렬·간격 유지
- [ ] **빈·제로·에러 상태** — 값이 0이거나 "데이터 없음"일 때도 화면이 자연스러움 (빈 칸 방치 X)
- [ ] **음수/감소** — 증감이 마이너스일 때 색·부호가 올바름
- [ ] **모바일(900px 이하)** — 그리드가 1~2열로 접히고 안 깨짐
- [ ] **속도** — self-contained(외부 이미지·라이브러리 X), 무거운 연출로 로딩 지연 X
- [ ] **JS 실패 폴백** — JS 꺼도 완성된 화면 (연출만 사라짐)
- [ ] **버튼/탭** — 주요·보조 버튼, 탭 선택 상태가 스타일에 맞게 구분되나 (인터랙티브도 스타일 판단 대상)
- [ ] **배경은 body 에** — 래퍼에만 두지 않기 (흰 여백·로드 플래시 방지)

## 접근성 (색만으로 정보 전달 금지)

- [ ] **상태를 색+텍스트/아이콘 병행** — 색맹도 구분 가능 ("문제" 배지에 빨강 + "문제" 글자)
- [ ] **본문 대비 4.5:1 이상** — 흐린 배경·저대비 스타일(글래스·뉴모·오로라)에서 특히 주의
- [ ] **focus-visible** — 키보드 포커스가 보임
- [ ] **prefers-reduced-motion** — 모션 끄면 완성 상태 즉시 렌더

## 차트 (데이터 왜곡 금지)

- [ ] **세로축 0부터** — 잘라서 차이 과장하지 않기 (3D·기울인 축 금지)
- [ ] **색 외 구분** — 라벨·패턴·직접 표기 병행 (색만으로 계열 구분 X)
- [ ] **피크 명시** — 지정한 피크 1곳이 실제로 강조됨

## 스타일별로 특히 볼 것

| 스타일 계열 | 견고성 취약점 |
|---|---|
| 글래스·뉴모·오로라·클레이 (저대비) | 흐린 배경 위 **텍스트 대비 4.5:1** 지키는가 |
| 벤토·맥시멀 (밀집·겹침) | 큰 숫자·긴 라벨 넣어도 **카드 조합이 안 무너지나** |
| 브루탈·Y2K·맥시멀 (과감) | 과감함에 밀려 **가독성·상태 구분**이 죽지 않았나 |
| 다크 대시보드·터미널 (고밀도) | 행·위젯 늘어도 **정렬·간격**이 버티나 |
| 미니멀·스위스 (절제) | 절제 때문에 **상태 4종 구분**이 약해지지 않았나 |

## 언제 쓰나

1. **생성 직전** — master-prompt 품질 기준에서 이 리스트로 자체 점검 후 출력
2. **사용자 리뷰** — 갤러리에서 "이 스타일이 내 최악 콘텐츠를 견디나"로 판단
3. **확정 전** — 역추출(디자인 시스템) 하기 전에 이 견고성이 시스템 규칙에 반영됐는지

> 요약: 예쁜 건 3초 무드로 판단하고(style-lint), **오래 쓸지는 최악 콘텐츠로 판단**한다(이 체크리스트). 둘 다 통과해야 진짜 쓸 만한 시안.
