# 결정 프레임워크 — 여러 시안에서 하나 고르기

30개를 평평하게 비교하면 결정 마비(선택지 5개↑면 결정률 급락). **"선택지가 아니라 지도"** 로 만든다.

## 1. 보기 전에 — 결정 기준 (rubric)
사업/주제별로 **미리 3~4개** 기준을 정한다. 느낌 대신 기준으로 점수 → 다음 날 안 뒤집힘.
- 브랜드 성격 (신뢰? 활기? 프리미엄?)
- 타겟 (B2B 실무자? 2030 소비자?)
- **1순위 목적** (전환 / 신뢰 / 차별화)
- 제약 (다크모드·접근성·모바일 비중 등)

## 2. 생성 — 무작위 X, 축 설계 O
30을 그냥 다 다르게 뽑으면 지도가 없다. **6~8 방향 × 3~5 변주**로 채운다.
- 방향(예): 신뢰 · 모던 · 다크 · 절제 · 부드러움 · 개성
- 변주: 강도(100/70%) · 라이트/다크
→ 30이 **2D 지도** 위에. "더 어둡게 + 더 절제" = 한 칸으로 이동(스캔 아님).

## 3. 한 변수만 — 결정 단계는 콘텐츠 고정
콘텐츠·데이터·화면 동일, **스타일만 변수**. 사과 vs 사과라야 순수 비교.
(콘텐츠도 다양한 세트는 "영감 탐색"이지 "결정"이 아니다 — 섞으면 결정 불가)

## 4. 좁히기 — 퍼널 (한 번에 5개까지만)
- **1라운드 방향**: 패밀리 대표 6~8개, 3초 컷 → 방향 1~2개 선택
- **2라운드 후보**: 고른 패밀리 안 3~5개 정독
- **3라운드 결승**: 1~2개 + 변주(강도·색·타이포)
→ 30-way 비교가 **5-way 3번**으로.

## 5. 방법 — 떨어뜨리기 먼저 (elimination-first)
"이 20개는 아니다"가 "이 1개다"보다 훨씬 빠르다. 3초 직감 탈락 → 남은 8개만 정독.

## 6. 도구 — 결정 갤러리 (gallery.html)
- ⭐ 별점/즐겨찾기 · ✕ 탈락(흐리게) · 📝 메모 · ⚖️ 2개 나란히 비교 · 필터(즐겨찾기만/탈락 숨김)
- localStorage 로 유지 (새로고침해도 남음)
- 같은 콘텐츠·같은 크기·실제 맥락(브라우저 프레임)에서 비교

## 7. 확정 — 하이브리드 후 역추출
결정은 1개 승자가 아니라 **합성**일 때가 많다: "A 레이아웃 + B 컬러". 병합 후 디자인 시스템 프롬프트로 역추출(5단계).

> 요약: **기준 먼저 · 축으로 설계 · 한 변수 · 퍼널로 좁히고 · 탈락 먼저 · 결정 갤러리로.** "예쁜 30"이 아니라 "**좁혀지는 30**"을 만든다.
