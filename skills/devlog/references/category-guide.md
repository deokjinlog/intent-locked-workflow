# 티스토리 카테고리 가이드

devlog **발행(모드 2) 시 카테고리를 추천**할 때 참조. 카테고리는 **주제**로 나눈다 (글의 결이 아니라).

## 핵심: 카테고리(주제) ⟂ 아키타입(글쓰기 결)

두 축은 **직교**한다. 헷갈리지 말 것:

- **카테고리 = 주제** — 무엇에 관한 글인가 (AI·클로드 / CS·프론트엔드 / 회고)
- **아키타입 = 글쓰기 결** — 어떻게 쓰는가 (개념 정리형 / 삽질기형 — `blog-craft.md` 참조)

예:
- "브라우저 저장소 정리" → **CS·프론트엔드** 카테고리 + **개념 정리형** 아키타입
- "디자인 스킬 만들기" → **AI·클로드** 카테고리 + **삽질기형** 아키타입

같은 카테고리 안에 두 아키타입이 섞여 있어도 정상이다.

## 카테고리 3개 (현재)

| 카테고리 | 무엇 | 예 |
|---|---|---|
| **AI·클로드** | Claude Code·스킬·프롬프트·AI 자동화. 지금 가장 뜨는 주제 = 검색 유입 | 디자인 스킬 만들기, devlog 스킬, 프롬프트 |
| **CS·프론트엔드** | 브라우저·네트워크·React·언어 원리 등 개념 학습 | 브라우저 저장소, React 상태, 캐시 계층 |
| **회고** | 분기·프로젝트·러닝 돌아보기 | 월간 회고, 프로젝트 후기 |

## 카테고리 고르는 법 (발행 시)

1. 이 글의 **주제**가 무엇인가 → 위 표에서 매칭.
2. 애매하면 (예: "Claude 로 React 개념을 배운 글") → **주된 주제** 하나로. 독자가 검색할 키워드 쪽으로 (지금은 AI·클로드가 유입 유리).
3. 3개 중 어디에도 안 맞는 새 주제가 **반복**되면 → 카테고리 추가를 사용자에게 제안.

## 성장 시 (참고)

글이 쌓이면 세분한다. 참고 블로그 youngsuk-dev 는 단어형 주제로 세분돼 있다 (회고 / 서버 / 디비 / 자바 / 자바스크립트 / 학습법 / 스타트업):

- **CS·프론트엔드** → CS / 프론트엔드 분리
- **AI·클로드** → 클로드 스킬 / 프롬프트 분리

지금은 시작 단계라 3개로 심플하게. 카테고리 이름은 **짧은 주제어**로 (참고 블로그 스타일).

> 요약: **카테고리는 주제**(AI·클로드 / CS·프론트엔드 / 회고), **글쓰기 결은 아키타입**(개념정리형/삽질기형)으로 따로. 발행 시 주제로 카테고리를, 내용으로 아키타입을 고른다.
