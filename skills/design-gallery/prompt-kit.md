# Design Gallery — 프롬프트 키트 (Master + 12 Style Blocks)

`design-gallery` 스킬이 실제로 사용하는 프롬프트 **저장소**. 사용자가 **도메인(만들 화면/콘텐츠)** 만 주면:

1. [PART A 마스터]의 `{{DOMAIN}}` 슬롯을 그 도메인으로 채우고
2. [PART B] 12개 스타일 블록 각각과 조합해 **병렬 생성**한다.

> 스타일 블록은 **v1(기본) → v2(고도화)** 로 점진 업그레이드. 사용자가 특정 스타일의 v2 프롬프트를 주면 해당 `### STYLE NN` 섹션을 통째로 교체한다. v2 는 SKILL.md 의 **"고도화 품질 레이어"**(로드 시퀀스·깊이 3층·마이크로 인터랙션·크리틱 루프·안전장치)를 그 스타일 언어로 녹인 버전.

---

## [PART A] 마스터 프롬프트 (도메인 슬롯)

```
당신은 시니어 프로덕트 디자이너입니다. 아래 [고정 콘텐츠]를 담은 화면을 단일 HTML 파일로
제작하세요. 콘텐츠·데이터·구성은 절대 바꾸지 말고, 이어지는 [스타일 지시]의 시각 언어만으로
디자인하세요.

[고정 콘텐츠 — 수정 금지]
{{DOMAIN}}
  ← 사용자가 지정한 화면/제품/섹션/데이터로 채운다. 모든 스타일이 이 동일 콘텐츠를 공유.
    (예시: "FleetOps 배송 관제 대시보드 — 헤더(제품명+갱신시각) / KPI 4개 / 시간대별 차트 /
     배송 현황 테이블" + 실제 수치·행 데이터)

[기술 요건]
- 단일 .html 파일, 순수 HTML/CSS (폰트 CDN만 허용)
- 한글 본문은 Pretendard CDN: https://cdn.jsdelivr.net/gh/orioncactus/pretendard@v1.3.9/dist/web/static/pretendard.min.css
- 스타일 지시의 디스플레이 폰트는 Google Fonts로 로드
- 반응형(900px 이하 1~2열), tabular-nums, 상태값은 색 구분 뱃지
- 차트는 CSS/인라인 SVG로 직접 구현 (차트 라이브러리 금지)

[품질 기준 — 고도화 레이어 필수]
SKILL.md 의 "고도화 품질 레이어" 6항목을 스타일 언어에 맞게 반드시 구현한다:
① 로드 시퀀스(히어로 연출 ~1.2초, 수치 카운트업) ② 깊이 3층(컬러 틴트 그림자)
③ 마이크로 인터랙션 3+ ④ 타이포 위계 4배+ ⑤ 자체 크리틱 루프(출력 전) ⑥ 안전장치
(reduced-motion / JS 실패해도 최종 데이터 존재 / focus-visible)
파일 하단 푸터에 "STYLE {번호} · {스타일명}" 표기.
```

### STYLE 03 — 트렌디 SaaS  `(v2 — 고도화 완료 ✅)`
```
[스타일 지시 — 트렌디 SaaS v2]

■ 무드
2025년 실리콘밸리 스타트업. Linear/Vercel 계열의 가볍고 낙관적인 감성.
"예쁜 정적 화면"이 아니라 "살아있는 제품"처럼 느껴져야 한다.

■ 팔레트
배경 #FBFAFF / 바이올렛 #7C5CFF(주) / 핑크 #FF8FC0 / 민트 #5EE6C8 / 잉크 #17143A / 보조 #7B7897
그라디언트는 바이올렛→핑크 한 방향만. 3색 이상 무지개 그라디언트 금지.

■ 타이포 — 위계 극단화
- 최대 텍스트(KPI 수치) vs 최소(캡션) 크기비 4배 이상
- KPI 수치: 40px+, 굵기 800, 자간 -0.03em, tabular-nums
- 레이블·캡션: 11~12px, 보조색으로 물러나게
- 그라디언트 텍스트는 제품명 단 1곳

■ 깊이 시스템 (3층)
L0 배경: #FBFAFF + 블러 90px 파스텔 블롭 2개(우상단 바이올렛, 좌하단 민트)
L1 표면: 흰 카드, radius 24~28px, 그림자 0 10px 30px rgba(23,20,58,.06)
L2 부유: hover된 카드만 -4px 떠오르며 그림자가 컬러 틴트로(0 20px 40px rgba(124,92,255,.15))
        — 깊이 변화엔 반드시 색이 동반

■ 히어로 연출 (로드 시퀀스, 총 1.2초 이내)
1) 0.0s 헤더 자리잡음
2) 0.1s~ KPI 4장이 60ms 스태거로 아래에서 떠오름(translateY 12px→0 + fade)
3) 동시에 KPI 숫자 0→목표 카운트업(JS 800ms ease-out)
4) 0.4s~ 차트 막대 왼쪽부터 40ms 간격으로 차오름
5) 마지막 피크 막대(예: 최댓값)만 바운스하며 그라디언트 점등

■ 마이크로 인터랙션 (4개, 각 150~250ms)
1) KPI 카드 hover: L2 부유  2) 차트 막대 hover: 그라디언트 점등+라벨 볼드
3) 진행중 상태 태그/실시간 칩 점(●) 2초 주기 숨쉬기  4) 테이블 행 hover: 연보라 물들기+ID 진해짐
과한 효과 금지 — "반응했다" 수준에서 멈춘다.

■ 시그니처
살아있는 숫자 — 카운트업 + 숨쉬는 실시간 인디케이터로 "지금 이 순간의 데이터" 감각.

■ 접근성·안전
prefers-reduced-motion: 모든 애니메이션 제거+최종상태 즉시 / JS 실패해도 최종 숫자가 HTML에 존재
(JS는 연출만) / hover는 focus-visible에도 동일.

■ 금지
직각, 격자 보더 테이블, 다크 배경, 모노스페이스, 무지개 그라디언트,
0.3초 넘는 개별 트랜지션, 무한 회전/이동.

■ 자체 크리틱 루프 (출력 전 필수)
① 로드 후 3초간 시선 머무는 곳 있나? 없으면 타이밍 재조정
② 가장 밋밋한 1곳 → 디테일 1개 추가  ③ 가장 과한 1곳 → 제거
④ 모든 색이 팔레트 안? 시퀀스 타이밍이 지시와 일치?
```
