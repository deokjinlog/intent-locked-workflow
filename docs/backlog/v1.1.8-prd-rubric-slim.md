## 배경

v1.1.7 dogfood 직후 사용자가 brainstorming 의 PRD adaptive rubric 을 보고 "✅필수 가 너무 많다" 고 신고. 현재 rubric 의 ✅필수 분포:

| 카테고리 | ✅필수 개수 / 6 |
|---|---|
| (a) 외부 사용자향 | 6/6 (전부) |
| (b) 내부 도구 | 3/6 |
| (c) 수정/리팩터 | 3/6 + 조건 1 |
| (d) 인프라 | 4/6 |

특히 (a) 외부향이 6/6 으로 baseline 대비 0 절감. 모든 카테고리에서 §1·§3·§6 (배경/FR/AC) 3개는 무조건 ✅필수로 박혀 있어서 "최소 PRD" 라기엔 짙다.

## 결정: 안 B (표준 슬림) 채택

### 신규 rubric

| 섹션 | (a) 외부향 | (b) 내부 도구 | (c) 수정/리팩터 | (d) 인프라 |
|---|---|---|---|---|
| 1. 배경/목적 | ✅ 필수 | ✅ 필수 | ✅ 필수 | ✅ 필수 |
| 2. 사용자 스토리 | ➖ 간소 | ⏭ 스킵 | ⏭ 스킵 | ⏭ 스킵 |
| 3. FR | ✅ 필수 | ✅ 필수 | ✅ 필수 | ✅ 필수 |
| 4. NFR | ➖ 간소 | ⏭ 스킵 | ⏭ 스킵 | ➖ 간소 |
| 5. 범위 밖 | ➖ 간소 | ➖ 간소 | ➖ 간소 | ➖ 간소 |
| 6. 수용 기준 | ➖ 간소* | ➖ 간소* | ➖ 간소* | ➖ 간소* |
| **✅필수 합** | **2** | **2** | **2** | **2** |

\* 사용자가 게이트에서 "수용 기준 풀로 가자" 명시 시만 ✅ 로 승격.

baseline 대비 ✅필수 16 → 8 = **-50%**.

### 변경 원칙

- §1 배경/목적 + §3 FR — 모든 카테고리에서 항상 ✅ 필수 (최소한의 plan ↔ 코드 매핑 단위)
- §6 수용 기준 — ➖간소 디폴트, 자동화 테스트가 있는 피처면 사용자가 게이트에서 ✅로 승격
- §2 사용자 스토리 — (a) 만 ➖간소, 나머지 ⏭ 스킵 (외부향 외에는 의미 적음)
- §4 NFR — (a)(d) 만 ➖간소, (b)(c) 는 ⏭ 스킵 (내부도구·리팩터는 NFR 대부분 무관)
- §5 범위 밖 — 모든 카테고리 ➖ 간소 (제외 항목은 대화 중 누적 consolidation 으로 처리)

### 안전망

verify-spec gate (단계 종료 직전 자동 발화) 가 §1/§3/§6 매핑 누락을 catch:
- FR 없으면 design 단계에서 "FR 매핑 부족" gap
- AC 모호하면 verify-spec §A. Consistency 가 "측정 불가" warning
