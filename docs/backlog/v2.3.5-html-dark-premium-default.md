# v2.3.5 후보 — generating-html 다크 프리미엄 default + 옵셔널 4-요소

## 문제 (사용자 신고)

`/Users/goldenplanet/Desktop/sa-dashboard-overhaul-tech-design.html` 검토 시:

> "지금 배경의 흰색인데.. 그 글에 회색깔로 칠해진 부분때문에 정말 사람이 알아보기 힘들 지경이야"

신고된 html 의 스타일 변수 발췌:
```css
:root {
  --bg: #f8f9fa;
  --text: #212529;
  --text-muted: #6c757d;   /* ← 한글 본문 muted, 가독성 ↓ */
}
@media (prefers-color-scheme: dark) {
  :root { --bg: #0d1117; ... }   /* dark 분기는 OS 설정 의존 */
}
```

→ default 가 light 모드. `#6c757d` (회색) on `#f8f9fa` (거의 흰색) 의 한글 가독성이 특히 떨어짐. 사용자 OS 가 light 면 회색 글자가 그대로 노출.

## 회귀 원인 — v2.2.4 톤 자유 룰의 부작용

v2.2.4 에서:
- 6 톤 inspiration 자유 선택 (docs portal / dashboard / **editorial** / playful / brutalist / experimental)
- "variety = feature" — 매 호출 다르게

→ subagent B 가 **editorial / docs portal** 톤을 골라 흰 배경 + 회색 muted 본문 패턴을 적용하는 경우 발생. 디자인 다양성은 가치 있으나 **가독성 baseline 이 깨지면 무용지물**.

## 해결 — 다크 프리미엄 default + 4-요소 옵셔널

### 룰 1: 비주얼 테마 = "다크 프리미엄" default

**모든** generating-html 산출물의 default 테마:

- **배경 (dark base)**: 깊은 어두운 톤 — `#0a0b0f` / `#0d1117` / `#111218` 류
- **텍스트 (high contrast)**: 밝은 톤 — `#e6edf3` / `#f0f6fc` / `#e8eaf0`. **muted 도 contrast 충분히** (`#8b949e` 보다 어둡지 않게)
- **악센트 (premium)**: 보라/시안/핑크 그라데이션 또는 단색 강조 — 차분하면서 시각 임팩트
- **카드 / 표면**: 반투명 흰색 overlay (`rgba(255,255,255,0.04~0.07)`) — 깊이감
- **그림자 / glow**: subtle drop shadow + accent glow (`rgba(124,109,255,0.25)` 류)
- **타이포**: heading bold, body 1.6~1.7 line-height, 한글 친화 폰트 스택

**다크 프리미엄 = "Premium dashboard / 고급 분석 도구" 톤**. 사무 / 회의록 / 흰 배경 editorial 톤 금지.

라이트 모드는 **`prefers-color-scheme: light`** 분기로만 제공 (default 가 dark, 명시적 라이트 선택은 OS 설정 따라감).

### 룰 2: 4-요소 옵셔널 — LLM 자율 판단

산출물에 포함 **가능** 한 요소 (모두 옵셔널):

| 요소 | 적용 케이스 (예시) |
|---|---|
| **데이터 차트/그래프** | 숫자 비교 / 분포 / 시계열 / 비율 데이터가 본문에 있을 때 |
| **인터랙티브 요소** | TOC / dark-mode toggle / copy 버튼 / smooth scroll / 보조 disclosure (primary 본문 X) |
| **다이어그램 / 플로우** | 아키텍처 / 데이터 흐름 / DAG / 컴포넌트 관계 |
| **핵심 요약 카드** | FR / 결정 / 위험 등 묶음 형태로 일관 표현 가능할 때 |

**룰**: 본문 콘텐츠가 위 요소에 맞지 않으면 **사용하지 마라**. 4 요소 모두 적용해야 한다는 강제 X. LLM 이 콘텐츠 보고 판단.

### 룰 3: v2.3.3 룰과의 정합

v2.3.3 의 "primary content default-visible" 룰 그대로:
- FR / AC / 배경 / 결정 등 primary 콘텐츠는 default-visible
- 인터랙티브 요소는 secondary 에만 (TOC / 변경이력 / dark-mode toggle 등)

## Anti-Patterns (html-companion-prompt.md 본문에 추가)

| 안티 패턴 | 이유 |
|---|---|
| Default 흰 배경 (`#fff` / `#f8f9fa` 류) + 회색 muted 본문 | 신고 사례 직격. 한글 가독성 망가짐. |
| `prefers-color-scheme: dark` 만 dark 분기 (default light) | OS 설정 의존성 위험. default 가 dark 여야 함. |
| 6 톤 (editorial / playful / brutalist 등) 자유 선택 | v2.2.4 룰 약화. 다크 프리미엄 baseline 강제. |
| 4 요소 모두 의무 사용 (차트 / 인터랙티브 / 다이어그램 / 카드) | 4 요소는 옵셔널. 콘텐츠와 안 맞으면 사용 X. |
| Muted 텍스트가 contrast 4.5:1 미만 (WCAG AA) | 가독성 baseline. dark 배경에서도 muted 는 어두운 회색 X. |

## Non-goals (이번 버전 안 함)

- 흰 배경 / 라이트 모드 완전 금지 — `prefers-color-scheme: light` 분기는 OS 선호 따라 제공 OK (default 만 dark)
- 6 톤 inspiration 자체 삭제 — 다크 프리미엄 안에서 variety 보존
- 차트 라이브러리 도입 (inline SVG / CSS 만, self-contained 보존)
- 디자인 시스템 토큰 표준화 (CSS custom property 이름 / 팔레트 등 명시 강제)
- 모바일 반응형 강제 (이미 v2.2.3+ 에 일부 룰)
