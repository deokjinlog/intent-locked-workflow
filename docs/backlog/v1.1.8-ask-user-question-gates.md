## 배경

v1.1.7 brainstorming → design → write-plan → execute 사이클을 dogfood 한 직후 사용자가:

> "그 yes / no 받을때 뭔가.. 그 도구호출은 하지않고 그냥 'cycle3-tech-design.md is finalized. Proceed to the writing-plans (구현계획서, step-by-step plan) stage now? — yes / no' 이렇게 질문하니까 뭐랄까.. 사용자가 인지하기가 힘들어"

현재 게이트는 산문 한 줄로 묻기 때문에 본문(특히 verify 보고서가 길게 노출된 직후) 과 시각적으로 안 구분돼서 사용자가 게이트 위치/응답 형식을 놓치기 쉬움.

## 우려 / 제약

- **harness 호환성**: AskUserQuestion 은 deferred tool — Claude Code 의 plugin tool registry 에서 schema fetch 필요. 다른 harness (codex/cursor/gemini) 에서도 동작해야 함. 호환 안 되는 harness 면 fallback 으로 prose prompt 유지 필요.
- **fix/partial 응답 follow-up**: enum 응답 후 "어디 고쳐달라?" 자유 텍스트는 AskUserQuestion 후 일반 메시지로 받으면 됨 — 도구 호출 1회 + 메시지 1회로 분리.
- **테스트 커버리지**: 8개 게이트마다 fixture 또는 dogfood 검증 필요. 자동화 어려우면 v1.1.8 plan 에서 수동 검증 task 로.
