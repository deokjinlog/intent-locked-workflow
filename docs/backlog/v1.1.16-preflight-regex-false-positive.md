# v1.1.16 후보 — `scripts/preflight._has_changelog_entries` regex false positive

> 작성: 2026-05-10
> 우선순위: 중간 — UX 마찰 (사용자 게이트 발화 false). v1.1.15 dogfood 중 catch.

## 문제

`scripts/preflight._has_changelog_entries` 가 markdown code block 안의 리터럴 `## 변경이력` 헤더를 진짜 footer 로 false positive 매치.

### 재현

```python
from scripts.preflight import code_pretty_check
from pathlib import Path

# implementation-plan.md 의 §1 task 본문에 markdown code block 으로
# 다른 doc 예시를 보여주는 경우:
#
#     ```markdown
#     # 요구사항: test
#     ## 변경이력
#     ### [...] [요구사항-수정]
#     ```
#
# 위 code block 안의 `## 변경이력` 리터럴을 helper 가 진짜 footer 로 인식.

result = code_pretty_check(Path('implementation-plan-with-fixture-example.md'))
# ok=False reason="변경이력 footer not empty (doc is live)"
# 실제로는 진짜 footer 는 비어있음.
```

### 근본 원인

`scripts/preflight.py:30-34`:

```python
def _has_changelog_entries(text: str) -> bool:
    if "## 변경이력" not in text:
        return False
    footer = text.split("## 변경이력", 1)[1]  # ← FIRST occurrence split
    return _CHANGELOG_ENTRY.search(footer) is not None
```

`text.split("## 변경이력", 1)` 가 **첫 번째** occurrence 에서 split. markdown code block 안에 `## 변경이력` 리터럴이 있으면 그것이 첫 occurrence 가 되어 진짜 footer 보다 먼저 매치.

## 해결안

1. **fix**: 진짜 footer 만 매치하도록 정규식 + 마지막 occurrence 사용. code block 인식이 어려우면 최소한 LAST occurrence 사용.

```python
def _has_changelog_entries(text: str) -> bool:
    # Use the LAST `## 변경이력` occurrence (real footer, not code-block literal)
    parts = text.rsplit("## 변경이력", 1)
    if len(parts) < 2:
        return False
    footer = parts[1]
    return _CHANGELOG_ENTRY.search(footer) is not None
```

또는 더 robust 하게: code block 영역 (` ```...``` `) 을 strip 후 검사.

2. **추가 unit test**:

```python
def test_has_changelog_entries_ignores_code_block_literal(tmp_path):
    p = tmp_path / "x-implementation-plan.md"
    p.write_text("""# title
## 1. tasks

```markdown
# 요구사항: test
## 변경이력
### [2026-05-10] [요구사항-수정]
- ...
```
End of fixture.
---
## 변경이력
<!-- empty footer -->
""", encoding="utf-8")
    from scripts.preflight import code_pretty_check
    r = code_pretty_check(p)
    assert r.ok is True  # ← 현재 False, fix 후 True
```
