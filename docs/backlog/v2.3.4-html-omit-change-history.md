# v2.3.4 후보 — generating-html 의 `.html` 에서 변경이력 UI 제거 허용

> 작성: 2026-05-22
> 우선순위: 낮음 — 작은 룰 변경, 1줄 패치 수준.

## 문제

현재 `html-companion-prompt.md` 룰: `.md` 의 모든 섹션을 `.html` 에 1:1 매핑. 즉 `## 변경이력` footer 도 `.html` 안에 렌더링됨.

그러나 변경이력 footer 는:
- AI 가 추적용으로 쓰는 **audit log** (CH-id / 이유 / 영향범위)
- `.html` 은 사람용 빠른 훑기 surface
- 사람이 `.html` 에서 변경이력 entries 를 읽을 일이 거의 없음 (보통 git log / change-history skill 로 확인)
- 시각적으로 길고 hierarchy 평탄 — `.html` 의 핵심 가독성을 깎음

→ `.html` 에서 변경이력 UI 는 **선택적으로 생략 가능** 으로 룰 완화.

## 해결 — `html-companion-prompt.md` 본문 룰 1개 추가

### 추가 룰

`## 변경이력` 섹션 (또는 동등 footer) 은 `.html` 에서 **생략 OK** (사람 가독성 우선). 단:

- 생략 시 `.html` 푸터 한 줄 안내: "변경이력은 `.md` 원본 참조" (또는 동등 문구)
- 생략 vs 노출은 subagent B 의 자율 판단 (정보 hierarchy 기준)

### 기존 의미 1:1 보존 룰과의 관계

- 본 룰은 **변경이력 footer 한정 예외**
- FR / AC / 배경 / 결정 / NFR / Non-goals / 미정 등 본문 콘텐츠는 그대로 1:1 보존 (v2.3.3 의 primary visibility 룰과 정합)
- v2.3.3 의 "secondary content 만 progressive disclosure" 룰에서 변경이력은 secondary 범주였음 — 이번 룰은 한 발 더 나아가 **생략 자체** 허용

## Anti-Patterns (prompt 본문에 추가)

| 안티 패턴 | 이유 |
|---|---|
| 변경이력 entry 본문을 `.html` 에 그대로 1:1 렌더 (강제) | 사람 가독성 ↓. 본 룰로 생략 허용. |
| 변경이력 footer 만 의미 변형 / 요약 | 생략은 OK지만 요약 변형은 X. 의미 1:1 룰 보존. |
| 변경이력 외 본문 콘텐츠도 같이 생략 | 본 룰은 변경이력 footer 한정. FR / AC 등 본문 생략은 v2.3.3 룰 위반. |

## 보존 (변경 X)

- v2.2.3 / v2.2.4 / v2.3.3 룰 모두 그대로
- 의미 1:1 룰 — 변경이력 footer 외 본문에는 그대로 적용
- self-contained / offline 렌더 / header·code 검증 / fire-and-forget boundary — 그대로
- `.md` 원본은 그대로 (변경이력 footer 는 `.md` 안에 그대로 존재)

## 영향 범위

- `skills/generating-html/html-companion-prompt.md` 본문에 룰 1개 + Anti-Patterns 3개 추가
- 다른 skill / command 영향 0
- `/sync-html` / `/pretty-md` / `/audit-risk` 영향 0
- auto-* / og-* 영향 0
- preflight.py 영향 0

## 검증

- 변경이력 footer 가 5개 이상 entry 인 `.md` 로 `.html` 생성 → 생략 또는 압축 노출 확인
- 변경이력 footer 가 0개 entry 인 `.md` (live doc 진입 전) → 생략 자연
- 생략 시 푸터 안내 한 줄 노출 확인
- FR / AC 본문은 여전히 default-visible (v2.3.3 룰 보존)

## Non-goals

- 변경이력 자동 요약 (생략만 허용, 요약은 의미 변형 위험)
- `.md` 에서 변경이력 제거 (`.html` 만 룰 변경, `.md` 는 그대로)
- 변경이력 외 footer 류 (e.g. metadata) 의 생략 룰 확장

## 미정

- 푸터 안내 정확한 문구 ("변경이력은 .md 원본 참조" 가 default?)
- entry 개수 임계 (예: 5+ 면 생략 권장, 미만이면 노출?) — 룰 강제 X, subagent 자율

## 비용

- prompt 본문 1줄 + Anti-Patterns 3개 — 호출 비용 변동 X
- 변경이력 footer 가 긴 경우 `.html` size ↓
