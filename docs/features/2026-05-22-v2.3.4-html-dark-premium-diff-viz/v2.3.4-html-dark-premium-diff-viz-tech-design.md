### 2.2 Anti-Patterns 15개

**A 항목 (3)**

| 안티 패턴 | 이유 |
|---|---|
| 변경이력 entry 본문을 `.html` 에 그대로 1:1 렌더 강제 | 사람 가독성 ↓. A-1 룰로 생략 허용. |
| 변경이력 footer 만 의미 변형 / 요약 | 생략 OK / 변형 X. 의미 1:1 룰 보존. |
| 변경이력 외 본문 콘텐츠도 같이 생략 | A-1 은 변경이력 footer 한정. FR/AC 등 본문 생략은 v2.3.3 룰 위반. |

**B 항목 (5)**

| 안티 패턴 | 이유 |
|---|---|
| Default 흰 배경 (`#fff` / `#f8f9fa` 류) + 회색 muted 본문 | 신고 사례 직격. 한글 가독성 망가짐. |
| `prefers-color-scheme: dark` 만 dark 분기 (default light) | OS 설정 의존. default 가 dark 여야 함. |
| 6 톤 (editorial / playful / brutalist 등) 자유 baseline 선택 | v2.2.4 룰 보정. 다크 프리미엄 1 톤 baseline 강제. |
| 4 요소 모두 의무 사용 (차트 / 인터랙티브 / 다이어그램 / 카드) | 4 요소는 옵셔널. 콘텐츠와 안 맞으면 사용 X. |
| muted 텍스트 contrast 4.5:1 미만 (WCAG AA) | dark 배경 위 너무 어두운 회색 (`#6c757d` 류) 금지. |

**C 항목 (7)**

| 안티 패턴 | 이유 |
|---|---|
| 원본 / 수정본 두 블록 별개 렌더 (diff 시각화 X) | C-1 룰 위반. 검토 효율 ↓. |
| diff 시각화에 외부 라이브러리 의존 (e.g. diff2html CDN) | self-contained 룰 위반. inline CSS / SVG 만. |
| word-level / character-level diff 적용 | 의역/오류 위험. line-level 단순만. |
| semantic diff (의미 변경 요약 / hint 텍스트) | 의미 1:1 룰 위반. 해석은 plan 본문에 맡김. |
| diff 라인 색상 contrast 4.5:1 미만 | B WCAG AA 와 충돌. |
| context 라인 collapse 강제 (전부 hide) | 변경 라인 ±N 은 default-visible 보장 (C-2). |
| 신규/삭제 파일도 억지 unified diff 형식 | C-4 fallback 위반. 단순 코드 블록 OK. |
