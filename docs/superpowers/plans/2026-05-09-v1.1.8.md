# v1.1.8 Implementation Plan — PRD Slim + AskUserQuestion Gates

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or js-super:og-executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** PRD adaptive rubric 슬림화 (✅필수 16→8, 안 B) + 8개 HARD-GATE 의 prose prompt 를 `AskUserQuestion` 도구 호출로 교체해 사용자 게이트 인지성 향상.

**Architecture:** 두 변경은 독립적이지만 둘 다 brainstorming/designing-direction/writing-plans skill 본문 텍스트 수정이라 한 release 묶음. PRD slim 은 rubric 표 1개 + plan 메시지 템플릿 1개 교체. AskUserQuestion 변환은 8 gate × prose-prompt 교체로 skill 본문 약 8 위치 수정. 도구 호출 형식은 다른 harness (codex/cursor/gemini) 호환을 위해 prose fallback 도 유지 (도구 미지원 시 기존 prose 사용).

**Tech Stack:** Markdown skill 본문 + AskUserQuestion 도구 schema (Claude Code 전용) + bash bump-version.

## Asking the User a Gate Question (v1.1.8+)

For any HARD-GATE asking enum/binary response (yes/no, yes/fix/partial, Inline/Subagent, Merge/PR/Cleanup), use the `AskUserQuestion` tool with this schema:

```json
{
  "question": "<short question, e.g. 'Approve <slug>-tech-design.md and proceed?'>",
  "context": "<optional 1-line context — what was just shown>",
  "choices": [
    {"value": "yes", "label": "Yes — approve and proceed"},
    {"value": "fix", "label": "Fix — needs revision"},
    {"value": "partial", "label": "Partial — revise specific sections"}
  ]
}
```

Harness fallback (when AskUserQuestion is unavailable, e.g. codex/cursor/gemini):

```markdown
**Approve and proceed?** — `yes` / `fix` / `partial`
```

Always prefer the tool when available. Detect availability via the harness tool registry; on miss, fall back to the prose form. The skill body shows BOTH forms; the executor picks based on environment.
```
