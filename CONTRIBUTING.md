# 기여 안내

intent-locked-workflow 에 관심 가져주셔서 고맙습니다.

이 저장소는 **기획 의도를 고정한 채, 설계와 계획과 코드가 그 의도대로만 흐르게 하는** Claude Code 워크플로 플러그인입니다. 스킬, 슬래시 명령, 훅, 헬퍼 스크립트로 이루어져 있어요.

아래 원칙을 지켜서 함께 다듬어 주세요.

## 기여하기 전에

이 프로젝트는 두 가지 성격을 같이 가집니다.

- **동작을 바꾸는 코드입니다.** 스킬 본문은 에이전트의 행동을 실제로 바꾸는 프롬프트예요. 읽기 좋은 글이 아니라 코드에 가깝습니다.
- **의도를 지키는 도구입니다.** 빨리 가는 게 아니라, 처음 이해한 기획 의도가 코드에 닿을 때까지 흔들리지 않게 지킵니다.

그래서 "말이 되게 고쳤다" 만으로는 부족합니다. 그 변경이 **실제로 더 나은 결과를 낸다는 근거**가 필요합니다.

## 기여할 수 있는 내용

- 버그 제보 — 게이트가 안 열리거나, 스킬이 엉뚱하게 발동하거나, 문서가 깨지는 경우
- 스킬 개선 — 기존 스킬의 흐름을 더 안전하거나 더 분명하게
- 새 스킬 제안 — 네 단계 흐름과 게이트 철학을 해치지 않는 선에서
- 문서 개선 — README, 안내문, 예시의 표현과 구조
- 새 하네스 지원 — 다른 편집기나 CLI 에서도 스킬이 자동으로 발동하도록
- 오탈자 수정, 설명 보강, 링크 정리

## 지켜야 할 원칙

### 의도 고정 철학을 흐리지 말아 주세요

이 도구의 핵심은 "게이트는 사람만 연다" 입니다. 사람 확인을 건너뛰거나, 위험한 결정을 자동으로 통과시키는 변경은 받지 않습니다.

### 스킬은 근거와 함께 고쳐 주세요

스킬 본문은 에이전트의 행동을 바꿉니다. 표현만 다듬는 변경이라도, 그 변경이 실제 세션에서 더 나은지 바꾸기 전과 후를 함께 보여 주세요.

특히 Red Flags 표나 "human partner" 같은 원작자(superpowers)의 다듬어진 표현은 근거 없이 바꾸지 말아 주세요.

### 결합된 스킬은 묶어서 고쳐 주세요

일부 스킬은 서로 신호를 주고받습니다. 한쪽만 고치면 다른 쪽에서 회귀가 납니다. 어떤 스킬이 서로 묶여 있는지는 `CLAUDE.md` 의 결합 메모에 적혀 있으니 먼저 읽어 주세요.

### upstream 원본은 건드리지 말아 주세요

`og-` 로 시작하는 스킬은 superpowers 원본을 그대로 옮긴 사본입니다. 본문을 바꾸지 말아 주세요.

### 한 번에 한 문제만 다뤄 주세요

여러 문제를 한 요청에 섞지 말아 주세요. 하나를 깊게 이해하고, 그 하나만 고쳐 주세요.

### 문체를 지켜 주세요

- 자연스러운 한국어 존댓말을 유지해 주세요.
- 한자는 쓰지 말아 주세요.
- 도구·함수·파일 이름 같은 영어 식별자는 그대로 두되, 사용자에게 보이는 안내문은 한국어를 우선해 주세요.

## 스킬을 고칠 때 (특히 주의)

스킬은 읽는 글이 아니라 행동을 바꾸는 코드입니다.

- 여러 세션에서 눌러보며 시험하고, 바꾸기 전과 후의 결과를 함께 적어 주세요.
- 이 저장소가 왜 지금처럼 생겼는지 먼저 이해해 주세요. 목소리나 구조를 통째로 바꾸는 변경은 받기 어렵습니다.
- 변경 하나가 여러 스킬에 걸쳐 있으면, 걸린 파일을 함께 고쳐서 한 묶음으로 올려 주세요.

## 문서 작성 가이드

- 제목과 소제목을 분명하게 나눠 주세요.
- 문단은 짧게, 한 문단에 하나의 핵심만 담아 주세요.
- 처음 보는 분도 바로 따라 할 수 있는 쉬운 표현을 써 주세요.
- 핵심 표현은 일관되게 써 주세요: 의도, 잠금, 게이트, 단계, 스킬, 문서, 변경이력, 위험 주석

## 기여 절차

1. 변경 목적을 한두 문장으로 정리합니다.
2. 같은 문제를 다룬 이슈나 요청이 이미 있는지 찾아봅니다.
3. 관련 파일을 고칩니다. 결합된 스킬이면 함께 고칩니다.
4. 스킬을 고쳤다면, 실제로 눌러보며 바꾸기 전과 후를 확인합니다.
5. 변경 전후를 비교해 의도 고정 철학이 그대로 살아 있는지 점검합니다.

버그 제보와 제안, 기여는 GitHub Issues 로 받습니다.
https://github.com/deokjinlog/intent-locked-workflow/issues

## 버전을 올릴 때

스킬 · 명령 · 매니페스트를 고친 뒤 새 버전을 Claude Code 에 반영하는 절차는 [docs/plugin-update.md](docs/plugin-update.md) 에 정리돼 있습니다. 어떤 변경이 patch / minor / major 인지도 거기 표로 있어요.

## 검토 기준

- 처음 보는 사람도 이해할 수 있는가
- 게이트와 의도 고정 철학이 그대로 살아 있는가
- 스킬 변경이라면, 더 나아졌다는 근거가 있는가
- 결합된 스킬을 함께 고쳤는가
- 한 요청이 한 문제만 다루는가

함께 다듬어 주시는 모든 개선이, 더 많은 분들이 의도를 잃지 않고 프로젝트를 완성하도록 돕습니다.
